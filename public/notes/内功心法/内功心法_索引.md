# 内功心法

> [!quote] 这个文件夹是干什么的
> 对**工具本质、技术架构、工作流设计**的深度拆解。不是"怎么用"，是"为什么这么设计"、"背后的原理是什么"、"能不能做得更好"。
>
> **一句话：外功教你怎么做，内功教你怎么想。** 和 [[Skills介绍]] 互为表里。

## 收录标准

放什么：
- 工具/框架的底层架构分析
- 工作流的方法论提炼
- 技术选型的权衡与推演
- 精确/模糊分工、人/AI 协作模式的思考
- 可复用的设计原则

不放什么：
- 纯操作指南 → 去 `Skills介绍`
- 会心一笑的轻松内容 → 去 [[案头随感_索引|案头随感]]
- 项目文档 → 留在各自的项目目录

## 笔记间的暗线

这五篇虽然话题不同，但都在循环同一个母题：**"人和 AI 各自擅长什么，怎么组合最好。"**

```
Obsidian   →  数据格式的选择（Markdown = 人机可读的中间层）
PPT 方案   →  精准 vs 模糊的分工（代码 = 骨架，AI = 皮肤）
GSAP 技能包 →  知识如何正确喂给 AI（官方教科书替代垃圾教程）
自动化管线  →  AI 覆盖采集→编译→通知，人只做精炼
Scrapling  →  工具的自适应能力（网页改版爬虫不死）
```

合在一起，是一套**"AI 时代技术工作者的内功修炼指南"**——不是工具书，是思维模型。
