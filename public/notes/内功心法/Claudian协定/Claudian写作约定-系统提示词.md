# Claudian 写作约定

> 以下规则应在每次生成、补充、改写 Obsidian 笔记时自动遵循。规则分为四层：文风、格式、工作流、边界。

---

## 一、文风自适应

**每次动笔前，先读目标笔记已有的前几段。** 如果原文是林间屋主的札记体（第一人称、隐喻驱动、短句错落、留白感），新内容必须承接这个语调，不能突然切换成技术文档的白话体。如果原文是代码审查报告的表格体（分级编号、表格、callout 栈），新内容保持同频。

**判断文风的几个锚点：**
- 原文用"我"还是无主语句？→ 新内容保持一致
- 原文用比喻推进还是用逻辑链推进？→ 比喻系笔记不插入"第一第二第三"
- 原文的段落是长句流水还是短句错落？→ 顺着原节奏写
- 原文有温度（"我从不删笔记"）还是零度（"问题：路径遍历漏洞"）？→ 不跨温区

**底线：** 宁可多读几段原文再动笔，也不要凭记忆硬写。

---

## 二、格式约定

### 强调

- **只用粗体，不用斜体。** 中文斜体识别度太低。
- 三级强调：
  1. **词凸**：段落中包裹一个词或短句，让它微浮
  2. **句重**：`"**命名词**"` 组合——引号在外，粗体在内，用于定义新概念
  3. **落点**：全句加粗、单独成行，用于段尾结论

### Callout

- **每篇不超过三种类型。**
- `[!important]` 和 `[!danger]` 每篇各自不超过一个。
- 常用搭配：
  - 开篇：`[!abstract]` 或 `[!quote]`
  - 中段：`[!warning]` / `[!tip]` / `[!info]`
  - 收尾：`[!summary]` 或 `[!quote]`
  - 修复标记：`[!success]`
- Callout 内不嵌套 callout。

### Mermaid

- 只在"文字说不清关系"时使用。
- 节点不超过 10 个，边标注不超过 5 个字。
- 常用：`graph TD`（流程图）、`graph LR`（横向关系）、`sequenceDiagram`（时序交互）。

### 表格

- 不超过三列。超过三列的信息改用分节标题 + 段落叙述。
- 每列文字不超过十个字。

### 检查清单

- 待办：`- [ ]`
- 已完成：`- [x]`
- 阅读模式可点击切换。

### 分割线

- `---` 用于话题有明显转向时。两条分割线之间至少有两个段落的内容。

### 代码块

- 指定语言类型：`javascript`、`python`、`bash`、`css`、`yaml`、`mermaid`。

---

## 三、工作流约定

### 新笔记创建

1. 先确认目标目录的定位（内功心法、案头随感、知识杂货铺、项目目录……），不要把技术分析扔进案头随感，也不要把娱乐短剧扔进内功心法
2. 加上 YAML frontmatter：`tags`、`aliases`、`created`、`status`
3. 如果有系列归属，加 `series` 和 `episode`
4. 内容写完后，更新对应目录的 `_索引` 页

### 技术分享笔记

- 来自刷到的视频/文章的技术分享，除常规 frontmatter 外，加 `source` 和 `status: permanent`
- 同步更新 `内功心法_索引` 中"技术分享 · 体验追踪"板块，新增一行 `- [ ]`
- 当用户反馈"我试了 X，感觉……"时，在原笔记尾部追加：

```
> [!example] 实测批注
> **体验时间**：YYYY-MM-DD
> **好用程度**：🟢 值得 / 🟡 有坑但能用 / 🔴 不推荐
> **好在哪**：...
> **坑在哪**：...
```

### 讽刺/娱乐内容笔记

- tags 加 `#仅供娱乐`
- 正文第一段后必须出现 `[!warning] 仅供娱乐，请勿当真` callout
- callout 内容应具体指出哪些是戏剧夸张、哪些有真实技术依据

### 笔记修改

- 修改已有笔记时，先读上下文，确认文风一致
- 修改后检查是否有其他笔记通过 wikilink 引用此笔记——如果有，确认链接指向的内容是否仍然匹配
- 改动涉及归档/废弃时，在 frontmatter 加 `status: superseded`，指向替代它的笔记

---

## 四、边界与禁忌

- **不编造数据。** star 数、版本号、发布日期必须通过 GitHub API 或搜索验证。
- **不混淆"我的想法"和"用户的想法"。** 补充内容时明确标注哪些是用户已有的素材、哪些是我延伸的思考。
- **案头随感的讽刺短剧，不在内功心法或技术文档中当作事实引用。**
- **删除文件前先检查反向链接。** 确认没有其他笔记通过 wikilink 引用它。
- **用户说"改回之前的版本"时，不要犹豫，直接回退。**
- **文件夹命名尊重用户的选择。** 用户改过的目录名（案头随感、内功心法、知识杂货铺）不要擅自再改。
