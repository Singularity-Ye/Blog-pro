# 树莓派实验六附录：手搓像素笑脸

> 仅供娱乐。用 0/1 矩阵逐像素硬编码一个 16×16 笑脸，替代 `ellipse` + `arc` 方案。

---

## 像素矩阵

```python
def draw_pixel_smile(draw, x0, y0):
    smile = [
        "0000111111110000",
        "0011000000001100",
        "0100000000000010",
        "1000000000000001",
        "1000010000100001",
        "1000010000100001",
        "1000000000000001",
        "1000000000000001",
        "1000100000010001",
        "1000010000100001",
        "1000001111000001",
        "1000000000000001",
        "0100000000000010",
        "0011000000001100",
        "0000111111110000",
        "0000000000000000",
    ]

    for y, row in enumerate(smile):
        for x, pixel in enumerate(row):
            if pixel == "1":
                draw.point((x0 + x, y0 + y), fill=255)
```

调用方式：

```python
draw_pixel_smile(draw, 104, 0)
```

## 与 ellipse+arc 方案对比

| 方案 | 优点 | 缺点 |
|------|------|------|
| `ellipse` + `arc` | 代码简洁，参数可调 | 需要手动调坐标 |
| 手搓像素矩阵 | 像素级精确控制 | 代码冗长，改起来麻烦 |

> 正式报告用 `ellipse` + `arc`，这里纯属玩票。
