# TEC-2 矩阵指令微操作样例表

> 从 MADD（矩阵加法）和 MMOV（矩阵传送）中提炼出的**已验证微操作积木**。
> 设计新微程序时，优先从本表中找现成微操作，不要靠字段递增规律硬猜。

## 二、矩阵指令中的寄存器分工

### MADD（矩阵加法）

| 寄存器 | 用途 |
|--------|------|
| R0 | ADDR1 当前指针（矩阵 A） |
| R1 | ADDR2 当前指针（矩阵 B） |
| R2 | ADDR3 当前指针（矩阵 C） |
| R3 | 列数备份 |
| R8 | 行计数器 |
| R9 | 状态位聚合寄存器 |
| R12 | 当前行列计数器 |
| Q | 暂存 A 元素和加法结果 |

### MMOV（矩阵传送）

| 寄存器 | 用途 |
|--------|------|
| R0 | 源矩阵当前指针 |
| R1 | 目标矩阵当前指针 |
| Q | 暂存源元素 |

## 三、循环体对比

### MADD 循环体

```text
R0 → AR, R0++        ; AR 指向 A 当前元素
MEM → Q              ; Q ← A[i]
R1 → AR, R1++        ; AR 指向 B 当前元素
MEM + Q → Q          ; Q ← A[i] + B[i]
STATUS OR R9 → R9    ; 累计状态
R2 → AR, R2++        ; AR 指向 C 当前元素
Q → MEM              ; C[i] ← Q
```

### MMOV 循环体

```text
R0 → AR, R0++        ; AR 指向源矩阵当前元素
MEM → Q              ; Q ← 源元素
R1 → AR, R1++        ; AR 指向目标矩阵当前元素
Q → MEM              ; 目标元素 ← Q
```

> MMOV 是 MADD 的"子集"——少了一路源矩阵和加法运算。

## 四、踩坑记录：寄存器选择字段不能靠直觉递增

### 错误推测

最初误以为修改 `30F0` 的低位即可选择目标寄存器：

```text
30F0 = MEM → R0
31F0 = MEM → R1
32F0 = MEM → R2
33F0 = MEM → R3
```

### 实测结果

修改为 `31F0`、`33F0` 后，**所有测试均表现为 R0 被写入**，R1/R2/R3 保持原值。

### 正确写法

根据完整 MADD 微码确认，目标寄存器由**第四个字段的高位**决定：

```text
30F0 0000  → MEM → R0
30F0 1000  → MEM → R1
30F0 2000  → MEM → R2
```

### 结论

TEC-2 微指令中的寄存器选择由 56 位微指令中的**多个字段共同决定**，不能只看单个十六进制字段的表面变化。设计新微程序时，**必须参考已验证的微码样例**，不要靠肉眼猜规律。

## 五、MAPROM 入口地址（矩阵指令专用）

| 操作码 | MAPROM 入口 | 用途 |
|:------:|:-----------:|------|
| D4 | 100H | MADD 矩阵加法 |
| E4 | 140H | MMOV 矩阵传送 |

测试指令格式：

```text
D4DR 0A00 0A10 0A20    ; MADD: C ← A + B
E4DR 0A00 0A20         ; MMOV: 目标 ← 源
```
