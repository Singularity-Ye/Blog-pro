# TEC-2矩阵加法指令：从单元素原型到完整循环版

> 本节记录 TEC-2 课程设计中矩阵加法指令的探索过程，从单元素原型逐步演进到完整矩阵循环版。

---

## 一、背景

矩阵加法题目要求实现：

```text
[ADDR3]矩阵 ← [ADDR1]矩阵 + [ADDR2]矩阵
```

三个矩阵均位于主存，ADDR1、ADDR2、ADDR3 分别为两个源矩阵和结果矩阵的首地址。矩阵行数和列数由指令中的 SR、DR 字段指定。

调试过程中形成两个方案：

| 方案 | 定位 | 特点 |
|------|------|------|
| **A：单元素原型** | 验证核心数据通路 | 只用 R0 + Q，简单可靠 |
| **B：完整循环版 MADD** | 真正的矩阵加法指令 | R0/R1/R2 三指针 + R8/R12 双循环 + R9 状态聚合 |

二者不是替代关系，而是**从简单原型到完整指令的逐步演进**。

---

## 二、方案 A：单元素矩阵加法原型

### 2.1 指令格式

临时借用 D500 入口，四字指令：

```text
D500
ADDR1
ADDR2
ADDR3
```

功能：`[ADDR3] ← [ADDR1] + [ADDR2]`

### 2.2 测试数据

```text
D500 0A00 0A10 0A20

[0A00] = 0001
[0A10] = 0010
[0A20] = 0000

预期：[0A20] = 0011
```

### 2.3 微操作流程

只使用已验证可靠的数据通路：R0 暂存地址参数，Q 暂存操作数和结果。

```text
 1. PC → AR, PC + 1 → PC      ; 取 ADDR1
 2. MEM → R0                  ; R0 ← ADDR1
 3. R0 → AR                   ; AR ← ADDR1
 4. MEM → Q                   ; Q ← [ADDR1]

 5. PC → AR, PC + 1 → PC      ; 取 ADDR2
 6. MEM → R0                  ; R0 ← ADDR2
 7. R0 → AR                   ; AR ← ADDR2
 8. MEM + Q → Q               ; Q ← [ADDR1] + [ADDR2]

 9. PC → AR, PC + 1 → PC      ; 取 ADDR3
10. MEM → R0                  ; R0 ← ADDR3
11. R0 → AR                   ; AR ← ADDR3
12. Q → MEM                   ; [ADDR3] ← Q
```

### 2.4 实测结果

执行 `D500 0A00 0A10 0A20`，查看 DA20：

```text
0A20  0011 0000 0000 ...
```

`[0A20] = 0001 + 0010 = 0011` ✅ 单元素原型成功。

### 2.5 方案 A 的意义与不足

**意义：** 验证了矩阵加法最核心的单次元素运算 `C[i] ← A[i] + B[i]`，可视为完整矩阵加法的"循环体原型"。

**优点：**

- 只依赖 R0 和 Q，数据通路简单
- 不需要处理多寄存器选择问题
- 便于调试，能清楚验证三地址加法逻辑

**不足：**

- 一条指令只能处理一个矩阵元素
- 没有地址自增、行列循环、C/V 状态位聚合
- 完成 2×2 矩阵需要连续调用 4 次

```text
D500 0A00 0A10 0A20
D500 0A01 0A11 0A21
D500 0A02 0A12 0A22
D500 0A03 0A13 0A23
```

## 三、调试中发现的寄存器选择问题

最初推测：

```text
30F0 = MEM → R0
31F0 = MEM → R1
32F0 = MEM → R2
```

但实测发现修改为 31F0、33F0 并不能正确写入 R1、R3。所有测试结果均表现为 R0 被写入，R1/R2/R3 保持原值。

后续根据完整 MADD 微码发现，**正确方式不是修改 30F0 本身，而是修改微指令后面的寄存器选择字段**：

```text
30F0 0000  → MEM → R0
30F0 1000  → MEM → R1
30F0 2000  → MEM → R2
```

寄存器编号由 56 位微指令中的寄存器选择字段共同决定。这一发现解释了前期多寄存器方案失败的原因。

## 四、方案 B：完整矩阵循环版 MADD

### 4.1 指令格式

四字格式：

```text
第 1 字：D4 DR SR
第 2 字：ADDR1
第 3 字：ADDR2
第 4 字：ADDR3
```

功能：`MEM[ADDR3+i] ← MEM[ADDR1+i] + MEM[ADDR2+i]`

| 字段 | 含义 |
|------|------|
| ADDR1 | 源矩阵 A 首地址 |
| ADDR2 | 源矩阵 B 首地址 |
| ADDR3 | 结果矩阵 C 首地址 |
| DR | 列数寄存器 |
| SR | 行数寄存器 |

测试指令：`D4AB 0A00 0A10 0A20`，R10=0003（3列），R11=0002（2行），矩阵规模 2×3=6 个元素。

### 4.2 寄存器分工

| 寄存器 | 用途 |
|--------|------|
| R0 | ADDR1 当前指针（矩阵 A） |
| R1 | ADDR2 当前指针（矩阵 B） |
| R2 | ADDR3 当前指针（矩阵 C） |
| R3 | 列数备份 |
| R8 | 行计数器 |
| R12 | 当前行列计数器 |
| R9 | 状态位聚合寄存器 |
| Q | 暂存 A 元素和加法结果 |

### 4.3 整体执行流程

采用**行列双循环**，不直接计算 SR × DR：

```text
1. 保存列数 DR → R3
2. 保存行数 SR → R8
3. 清空 R9（C/V 状态聚合）
4. 读取 ADDR1/ADDR2/ADDR3 → R0/R1/R2
5. 每行开始：R3 → R12（当前行列计数器）
6. 每处理一个元素：
     [R2] ← [R0] + [R1]
     R0++, R1++, R2++, R12--
7. 当前行结束后：R8--
8. 若 R8 ≠ 0，进入下一行
9. 全部完成后：STATUS ← R9
```

对应 C 语言伪代码：

```c
Aptr = ADDR1; Bptr = ADDR2; Cptr = ADDR3;
rows = SR; cols = DR; status_agg = 0;

for (i = 0; i < rows; i++) {
    col_count = cols;
    while (col_count != 0) {
        temp = MEM[Aptr] + MEM[Bptr];
        MEM[Cptr] = temp;
        status_agg |= current_status;
        Aptr++; Bptr++; Cptr++; col_count--;
    }
    rows--;
}
STATUS = status_agg;
```

### 4.4 单次元素加法循环体

```text
R0 → AR, R0++        ; AR 指向 A 当前元素，A 指针后移
MEM → Q              ; Q ← A[i]

R1 → AR, R1++        ; AR 指向 B 当前元素，B 指针后移
MEM + Q → Q          ; Q ← A[i] + B[i]

status OR R9 → R9    ; 累计本次加法状态

R2 → AR, R2++        ; AR 指向 C 当前元素，C 指针后移
Q → MEM              ; C[i] ← Q
```

> 本质上是方案 A 单元素原型的升级版——用 R0/R1/R2 作为持续自增的矩阵指针，替代每次重新取地址。

### 4.5 状态位 C/V 聚合

题目要求：任意一个元素加法产生进位 C 或溢出 V，则最终 C=1 或 V=1。

通过 R9 实现**粘滞聚合**：

```text
R9 = 0
每次元素加法后：R9 ← R9 OR 当前状态字
指令结束前：STATUS ← R9
```

只要任意一次加法产生 C 或 V，对应标志位就会保留到最终状态中。

## 五、完整 MADD 测试数据与测试配置

### 5.1 测试参数汇总

在执行矩阵加法测试前，需在 TEC-2 的主存及寄存器中写入以下核心参数：

```text
MADD 测试参数如下：

微码装载：
R1 = 0900H
R2 = 001DH
R3 = 0100H

矩阵规模：
R10 = 0003H，表示列数为 3
R11 = 0002H，表示行数为 2

矩阵地址：
ADDR1 = 0A00H，源矩阵 A 首地址
ADDR2 = 0A10H，源矩阵 B 首地址
ADDR3 = 0A20H，结果矩阵 C 首地址

运行结果：
0A20H~0A25H = 0003H、8000H、0000H、0000H、0000H、000DH
```

### 5.2 矩阵测试数据配置

矩阵规模：R10=0003（3列），R11=0002（2行），共计 6 个元素。

**矩阵 A（首地址 0A00H）：**

| 地址 | 值 | 备注说明 |
|------|-----|---|
| 0A00 | 0001 | 正常正数 |
| 0A01 | 7FFF | 正最大值（验证溢出 V） |
| 0A02 | FFFF | $-1$ 补码（验证进位 C） |
| 0A03 | 0004 | 正常正数 |
| 0A04 | 8000 | 负最大值（验证溢出 V） |
| 0A05 | 0006 | 正常正数 |

**矩阵 B（首地址 0A10H）：**

| 地址 | 值 | 备注说明 |
|------|-----|---|
| 0A10 | 0002 | 正常正数 |
| 0A11 | 0001 | $7FFF + 0001 \rightarrow 8000H$ (产生溢出 $V=1$) |
| 0A12 | 0001 | $FFFF + 0001 \rightarrow 0000H$ (产生进位 $C=1$) |
| 0A13 | FFFC | $-4$ 补码，相加为 $0000H$ |
| 0A14 | 8000 | $8000 + 8000 \rightarrow 0000H$ (产生进位 $C=1$, 溢出 $V=1$) |
| 0A15 | 0007 | 正常正数 |

执行测试指令：`D4AB 0A00 0A10 0A20`。

**预期结果矩阵 C（首地址 0A20H）：**

| 地址 | 计算过程 | 预期结果 |
|------|---------|------|
| 0A20 | 0001 + 0002 | 0003 |
| 0A21 | 7FFF + 0001 | 8000 |
| 0A22 | FFFF + 0001 | 0000 |
| 0A23 | 0004 + FFFC | 0000 |
| 0A24 | 8000 + 8000 | 0000 |
| 0A25 | 0006 + 0007 | 000D |

## 七、两个方案的关系

```text
方案 A：单元素原型
  [ADDR3] ← [ADDR1] + [ADDR2]
  验证矩阵加法的最小循环体

方案 B：完整 MADD
  [ADDR3+i] ← [ADDR1+i] + [ADDR2+i]
  在单元素加法基础上加入地址自增、行列循环和状态聚合
```

**方案 A 是积木，方案 B 是完整建筑。**

方案 A 降低调试复杂度，先证明三地址加法数据通路可行；方案 B 把该逻辑封装进微程序循环，实现真正的矩阵加法指令。

---

## 八、结论

复杂微程序设计适合**分阶段调试**：先验证最小可行循环体，再加入地址自增、循环控制和状态处理。这样能有效降低 TEC-2 微程序设计中的调试难度。
