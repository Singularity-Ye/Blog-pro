# TEC-2实验三：微程序设计（写回ADDR1版）

> 设计新指令 `D4XX`，实现 `[ADDR1] ← [ADDR1] + [ADDR2]`，结果写回第一个内存单元。

---

## 一、实验目标

指令格式：

```text
D4XX
ADDR1
ADDR2
```

功能：

```text
[ADDR1] ← [ADDR1] + [ADDR2]
```

测试指令：

```text
D400 0A00 0A01
[0A00] = 0023, [0A01] = 0023
预期：[0A00] = 0046, [0A01] = 0023
```

---

## 二、与写回 ADDR2 版的区别

| | 写回 ADDR2（旧版） | 写回 ADDR1（本版） |
|---|---|---|
| 功能 | `[ADDR2] ← [ADDR1] + [ADDR2]` | `[ADDR1] ← [ADDR1] + [ADDR2]` |
| 结果 | 0A00=0023, 0A01=0046 | 0A00=0046, 0A01=0023 |

> 加法满足交换律，但写回地址不同，最终内存状态不同。

---

## 三、微操作设计思路

写回 ADDR1 的关键问题：读取 ADDR2 时 AR 会被改成 ADDR2，此时直接 Q → MEM 会写到 ADDR2。必须在写回前让 AR 重新指向 ADDR1。

```text
1. PC → AR, PC + 1 → PC      ; 取 ADDR1 地址字
2. MEM → AR                  ; AR ← ADDR1
3. MEM → Q                   ; Q ← [ADDR1]
4. 保存 ADDR1                ; 暂存写回地址
5. PC → AR, PC + 1 → PC      ; 取 ADDR2 地址字
6. MEM → AR                  ; AR ← ADDR2
7. MEM + Q → Q               ; Q ← [ADDR2] + [ADDR1]
8. 恢复 AR ← ADDR1           ; 重新定位写回地址
9. Q → MEM                   ; [ADDR1] ← Q
10. 返回取下一条机器指令
```

---

## 四、Monitor 操作总流程

```text
E900  → 输入写回 ADDR1 版本的微程序代码
D900  → 检查微程序代码
A800  → 输入微码加载程序
G800  → 执行 LDMC
A820  → 输入测试程序
E826  → 插入新指令 D400 0A00 0A01
U820  → 查看测试程序
G820  → 运行测试程序
DA00  → 查看最终内存结果
```

> 流程与写回 ADDR2 版一致，但 E900 处输入的微码不同。

> ⚠️ 不要把写回 ADDR2 的旧版微码填入。旧版结果是 0A01=0046，新版目标是 0A00=0046。

## 六、Step 2：检查微码

```text
D900
```

检查 0900H 起始区域内容是否与输入一致。

---

## 七、Step 3：输入微码加载程序

```text
A800
```

```asm
MOV R1,0900
MOV R2,0007
MOV R3,0100
LDMC
RET
```

| 寄存器 | 值 | 作用 |
|--------|-----|------|
| R1 | 0900H | 微码起始地址 |
| R2 | 0007 | 微指令条数（需与实际一致） |
| R3 | 0100H | 微控存入口 |

---

## 八、Step 4：执行微码加载

```text
G800
```

返回 `>` 且无 `Dead cycle!` 即成功。

---

## 九、Step 5：输入测试程序

```text
A820
```

```asm
MOV R0,0023
MOV [0A00],R0
MOV [0A01],R0
NOP
NOP
NOP
RET
```

初始化：`[0A00] = 0023`，`[0A01] = 0023`。三个 NOP 预留空间。

## 十、Step 6：插入新指令

```text

D400 0A00 0A01
```

写入效果：

```text
0826: D400

```

执行：`[0A00] ← [0A00] + [0A01]`

## 十三、Step 9：查看实验结果

```text
DA00
```

写回 ADDR1 版本预期：

```text
0A00  0046 0023 0000 0000 ...
```

| 地址 | 值 | 说明 |
|------|-----|------|
| 0A00 | 0046 | 0023 + 0023 = 0046 ✅ |
| 0A01 | 0023 | 原值不变 |

> **判断标准**：0A00=0046 说明写回 ADDR1 版跑通。若 0A01=0046 则仍是旧版写回 ADDR2。

## 十四、常见问题

### Q1：为什么写回 ADDR1 比 ADDR2 麻烦？

读取 ADDR2 时 AR 被改为 ADDR2，必须在写回前恢复 AR ← ADDR1。

### Q2：为什么结果不是两个地址都变？

指令只写回一个目标。`[ADDR1] ← [ADDR1] + [ADDR2]` 只覆盖 ADDR1，ADDR2 保持原值。

### Q3：U820 中 0A00/0A01 显示成 ADC？

正常。U 命令无法识别新指令参数结构。判断依据应是 DA00 结果。

### Q4：出现 Dead cycle 怎么办？

检查 LDMC 参数，尤其是 R2 的微指令条数。错误执行过 LDMC 建议重启。
