# TEC-2 课程设计：E1 / E5 寄存器比较条件转移指令

> 本笔记记录了寄存器比较条件转移指令的设计与演进调试过程。记录了从 E1 绝对跳转失败到 E5 相对偏移跳转成功调试的完整路径。
> 返回主索引：[[TEC-2课程设计：微程序设计|TEC-2 课程设计：微程序指令系统设计]]

## 1. 指令规格与数学模型

本次设计针对课程设计要求的“相等则跳转”指令，经历了两个版本的演进：

### 1.1 初始设计（E101 绝对跳转版 - 失败）
* **指令格式**: 双字指令，微控存入口 `130H`。
  ```text
  E1 DR SR
  ADDR     ; 满足条件时跳转的绝对主存地址 (16位)
  ```
* **逻辑定义**:
  $$
  \text{if } DR = SR \text{ then } PC \leftarrow ADDR \text{ else } PC \leftarrow PC + 2
  $$

### 1.2 修正设计（E501 相对跳转版 - 成功）
* **指令格式**: 双字指令，微控存入口 `140H`。
  ```text
  E5 DR SR
  DISP     ; 满足条件时跳转的相对 IP 偏移量 (16位)
  ```
* **逻辑定义**:
  $$
  \text{if } DR = SR \text{ then } PC \leftarrow IP + DISP \text{ else } PC \leftarrow PC + 2
  $$

## 2. 双轨制设计分析

### 🔬 学术轨：条件转移与相对地址合成
在微指令执行周期中，核心操作分为两步：
1. **状态标志设置**：通过 ALU 执行减法操作 $SR - DR$，该操作不改变寄存器内容，主要用于置标志寄存器的零标志 $Z$ (Zero Flag)。若 $DR = SR$，则结果为 0，标志位 $Z \leftarrow 1$。
2. **条件定序判断**：
   * 在微码定序中，利用 Am2910 的测试条件 $CC$（绑定 $Z$ 标志状态）。
   * 取出指令第二个字，即相对偏移量 $DISP$（主存中表现为相对字数，如 `0005`），此时 PC 自动完成顺序指向（指向 $IP$ 顺序执行位置）。
   * 若 $Z = 1$（测试条件成立），将主存读出的偏移量与当前指令的 $IP$ 进行加法合成，结果写入 PC（$PC \leftarrow IP + DISP$）。
   * 若 $Z = 0$（条件不满足），则跳过写 PC 的微操作，直接返回公共取指微程序，实现顺序执行。

### 💡 直觉轨：“红绿灯检测与偏航规避”
> 快递员走到一个分叉路口，他的任务是比较手上的两个货箱。比较两个货箱是否相同（$SR - DR$）是**触发红绿灯（Z 标志）**的开关：
> 
> * **相同（Z = 1，绿灯亮）**：快递员在 GPS 上启用“偏航路线”，在当前正常行驶坐标的基础上加上一个偏航距离（$PC \leftarrow IP + DISP$），从而改道走新的路线（跳转执行）。
> * **不同（Z = 0，红灯亮）**：绿灯没亮，快递员的偏航指令失效，他只能继续沿着原定路线一直开下去（顺序执行）。

## 3. 微操作流程与符号表示

微程序共包含 3 条微指令，存放在控制存储器 `140H` 开始的区域：

$$
\begin{array}{llll}
\hline
\text{控存地址} & \text{微操作流程} & \text{详细功能说明} \\
\hline
140\text{H} & SR - DR & \text{比较 SR 与 DR 寄存器内容，影响标志 Z} \\
141\text{H} & PC \rightarrow AR, PC + 1 \rightarrow PC, CC\# = Z & \text{取相对偏移量 DISP 读入 AR，且绑定 Z 标志为测试条件} \\
142\text{H} & \text{MEM} + IP \rightarrow PC & \text{条件满足时，完成相对地址相加并送入 PC 触发跳转} \\
\hline
\end{array}
$$

## 4. 最终微码与 Monitor 操作流程

### Step 1: E980 输入微码
在 Monitor 监控程序中输入 `E980`，将微码输入主存 `0980H`：
```text
>E980
0980: 0000 0E01 9190 0088 0029 03E0 A0B5 5402 0029 0310 30D6 5000
```
输入完成后回车退出。使用 `D980` 检查无误。

### Step 2: 加载微程序至控存（`140H`）
输入 `A800` 编写微码装载程序：
```asm
MOV R1, 0980  ; 微码在主存中的源首地址
MOV R2, 0003  ; 共有 3 条微指令
MOV R3, 0140  ; 写入控制存储器的首地址为 140H（E5 绑定入口）
LDMC          ; 加载控存
RET
```

## 6. 调试与演进排错记录：从 E101 绝对跳转失败到 E501 相对跳转

### 6.1 绝对跳转尝试（E101 版故障现象）
起初我们按照设计规格书的要求，编写了基于 `E101 0850` 的绝对跳转微码：
* **微码配置**:
  `0000 0E01 9190 0088` (SR - DR比较)
  `0029 03E0 A0B5 5402` (取目标绝对地址)
  `0029 0300 30F0 5000` (MEM -> PC 绝对写入)
* **载入入口**: `130H`
* **测试用例**: $R0 = R1 = 0011\text{H}$，测试程序位于 `0840H`，期待跳转至 `0850H` 的 `MOV R2, 1111`。

#### 运行异常结果
执行 `G840` 后，使用 `R` 查看寄存器输出：
* $R2 = 2222$
* $IP = 0848$
* 结果证明，即便 $R0 = R1$ 满足相等条件，指令仍然顺序执行了 `0846: MOV R2, 2222`，绝对跳转宣告失败。

![TEC2_CD_E5_E101_Abs_Addr_Failure.png](/notes/attachments/attachments/01_%E8%97%8F%E7%BB%8F%E9%98%81%EF%BC%88%E7%9F%A5%E8%AF%86%E5%BA%93%EF%BC%89/%E6%A0%BC%E7%89%A9%E9%99%A2%EF%BC%88%E8%AF%BE%E7%A8%8B%E4%B8%8E%E5%AD%A6%E7%A7%91%EF%BC%89/%E8%AE%A1%E7%AE%97%E6%9C%BA%E7%BB%84%E6%88%90%E5%8E%9F%E7%90%86/TEC2_CD_E5_E101_Abs_Addr_Failure.png)
*图 6: E101 绝对跳转失败记录（R2仍为 2222，说明未发生跳转动作）*

### 6.2 机理排查与工程修正
分析监控程序特性及微指令逻辑，确定了失败的核心原因：
1. **操作码对 CND 条件的映射机制限制**：TEC-2 中的 `E1` 在硬接线测试部件中未直接绑定 Z 标志。相较之下，操作码前缀 `E5` 在机型内部逻辑里天然用于 Z 标志的条件转移，利用 CND 字段（如 Z 标志位）更为可靠。
2. **地址线合成特性**：TEC-2 原生条件跳转指令推荐使用基于 IP 偏移的相对跳转，其微操作支持 `MEM + IP -> PC` 的加法地址生成（微码控制字段为 `30D6`），而非直接覆盖 PC。
3. **改进方案**：改用 `E5DRSR, DISP` 格式（微程序切入控存 `140H`，微操作换用 `MEM + IP -> PC`），成功排除了绝对地址不响应的硬件层限制，达成了课设功能指标。
