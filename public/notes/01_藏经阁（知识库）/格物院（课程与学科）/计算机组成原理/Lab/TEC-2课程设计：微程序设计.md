## 2. 微指令格式与定序器控制原理

TEC-2 系统采用 **48位微指令字宽的水平微指令** 结构。微程序控制电路以 **Am2910 微程序定序器** 芯片为核心：
* **时序控制**：每个微机器周期执行一条微指令。控存读出数据被锁存到微指令寄存器（Pipeline Register）中以形成重叠的取指-执行流水。
* **分支决策**：Am2910 根据测试条件（$CC$）和微指令中的下一地址控制字段，控制选择微地址的来源（如：下条微指令地址、堆栈返回地址、机器码跳转映射表 MAP 等）。

## 3. Monitor 监控程序命令速查

在调试微程序时，常用的监控程序（Monitor）命令汇总如下：

| 命令 | 格式 | 核心用途 |
| :--- | :--- | :--- |
| **`E` (Examine)** | `Exxxx` | 写入主存数据（如 `E900` 写入微码，`E828` 插入机器指令） |
| **`D` (Dump)** | `Dxxxx` | 检查主存内容（如 `D900` 检查微码，`DA00` 查看测试结果） |
| **`A` (Assemble)** | `Axxxx` | 汇编输入机器语言程序（如 `A800` 输入微码装载程序） |
| **`U` (Unassemble)** | `Uxxxx` | 反汇编查看程序指令（如 `U820` 查看测试程序） |
| **`G` (Go)** | `Gxxxx` | 运行指定首地址的机器程序（如 `G800` 执行装载，`G820` 运行测试） |

## 4. 常见避坑与调试规则

* **`Dead cycle!`（死循环）故障排查**：
  * 装载程序中的通用寄存器 $R2$ 表示加载的微指令条数，必须与实际微码行数严格对应（例如：`D5XX` 对应 9 条，装载写为 `MOV R2, 0009`；`D8XX` 对应 6 条，写为 `MOV R2, 0006`）。
  * 若误执行了错误的微指令条数加载，建议关闭 TEC-2 系统和仿真器并重启，防止微控存空间错乱。
* **参数反汇编助记符偏差**：
  * 使用 `U` 命令查看测试程序时，自定义指令的地址或数据参数（如 `0A00`、`1234`）会被系统内置的普通指令表反汇编成 `ADC` 或 `SBB`。此现象为监控程序的正常反汇编局限，执行时会自动按照新定义的指令微程序流程读取，调试结果应完全以 `D` 命令查看的目标主存实际数值为准。

## 6. 中英专业学术术语对照

| 中文术语 | 英文术语 | 核心定义与功能描述 |
| :--- | :--- | :--- |
| **控制存储器 (控存)** | Control Memory (CM) | 存放 CPU 微程序的高速可读写存储器，独立于系统主存。 |
| **微程序定序器** | Microprogram Sequencer | 控制微指令跳转、调用等顺序或分支转移的控制部件（如 Am2910）。 |
| **微指令** | Microinstruction | 控制数据通路中各微操作控制信号（ALU功能、寄存器读写等）的二进制编码。 |
| **机器指令** | Machine Instruction | 汇编级宏观指令，由 CPU 中的微程序解释执行。 |
| **地址寄存器 (AR)** | Address Register (AR) | 用于保存当前访问主存或外设的绝对内存地址的寄存器。 |
| **程序计数器** | Program Counter (PC) | 指向主存中下一条将要被读取的机器码指令首地址。 |
