# TEC-2实验三：微程序设计

> 设计一条新指令，实现两个绝对地址内存单元内容相加，结果存放到第二个内存单元。

---

## 一、实验目标

设计新指令 `D4XX`，格式：

```text
D4XX
ADDR1
ADDR2
```

功能：

```text
[ADDR2] ← [ADDR1] + [ADDR2]
```

### 测试用例

```text
D400 0A00 0A01

[0A00] = 0023
[0A01] = 0023

预期：[0A01] = 0023 + 0023 = 0046
```

---

## 二、微操作流程

```text
1. PC → AR, PC + 1 → PC      ; 取 ADDR1 地址字
2. MEM → AR                  ; AR ← ADDR1
3. MEM → Q                   ; Q ← [ADDR1]

4. PC → AR, PC + 1 → PC      ; 取 ADDR2 地址字
5. MEM → AR                  ; AR ← ADDR2
6. MEM + Q → Q               ; Q ← [ADDR2] + [ADDR1]

7. Q → MEM                   ; [ADDR2] ← Q
8. 返回取下一条机器指令
```

> Q 用于暂存 [ADDR1]。执行到 ADDR2 后，AR 正好保存 ADDR2，因此最后可以直接 Q → MEM，将结果写回 ADDR2。

---

## 三、Monitor 操作总流程

```text
E900  → 输入微程序代码
D900  → 检查微程序代码
A800  → 输入微码加载程序
G800  → 执行 LDMC，加载微码到微控存
A820  → 输入测试程序
E826  → 插入新指令 D400 0A00 0A01
U820  → 查看测试程序
G820  → 运行测试程序
DA00  → 查看最终内存结果
```

---

## 四、Step 1：输入微程序代码

在 Monitor 中输入：

```text
E900
```

系统显示类似：

```text
0900    0000:
```

在冒号后输入微码：

```text
0000 0E00 A0B5 5402 0000 0E00 10F0 0002 0000 0E00 00F0 0000 0000 0E00 A0B5 5402 0000 0E00 10F0 0002 0000 0E01 00E0 0000 0029 0300 1020 0010
```

输入完成后空回车退出。

---

## 五、Step 2：检查微码

```text
D900
```

应看到：

```text
0900  0000 0E00 A0B5 5402 0000 0E00 10F0 0002
0908  0000 0E00 00F0 0000 0000 0E00 A0B5 5402
0910  0000 0E00 10F0 0002 0000 0E01 00E0 0000
0918  0029 0300 1020 0010 0000 0000 0000 0000
```

`0900` 到 `091B` 与输入一致即成功。

## 六、Step 3：输入微码加载程序

```text
A800
```

逐行输入：

```asm
MOV R1,0900
MOV R2,0007
MOV R3,0100
LDMC
RET
```

显示：

```text
0800: MOV R1,0900
0802: MOV R2,0007
0804: MOV R3,0100
0806: LDMC
0807: RET
```

含义：

| 寄存器 | 值 | 作用 |
|--------|-----|------|
| R1 | 0900H | 微码在主存中的起始地址 |
| R2 | 0007 | 共 7 条微指令 |
| R3 | 0100H | 加载到微控存 100H 入口 |

---

## 七、Step 4：执行微码加载

```text
G800
```

直接返回 `>` 且无 `Dead cycle!` 即成功。

A820

## 九、Step 6：插入新指令 D400

```text
E826
```

输入：

```text
D400 0A00 0A01
```

写入效果：

```text
0826: D400
0827: 0A00
0828: 0A01
```

## 十、Step 7：查看测试程序

```text
U820
```

应看到：

```text
0820: 2C00 0023  MOV  R0,     0023
0822: 3400 0A00  MOV  [0A00], R0
0824: 3400 0A01  MOV  [0A01], R0
0826: D400       DW   D400
0827: 0A00       ADC  R0,     R0
0828: 0A01       ADC  R0,     R1
0829: AC00       RET
```

> 0827 和 0828 显示成 ADC 是正常现象。U 命令按普通机器码反汇编，不知道 0A00、0A01 是 D400 的地址参数。实际执行时 D400 会把它们当 ADDR1、ADDR2。

## 十三、常见问题

### Q1：E900 后出现 `0900 0000:` 是什么？

正常现象。`0900` 是当前地址，`0000` 是原内存内容，冒号后等待输入。不要输入地址和冒号，直接输微码。

### Q2：U820 中 0A00/0A01 显示成 ADC，是不是错了？

不是。U 命令无法识别新指令的参数结构，会按普通指令反汇编。判断依据应是 `DA00` 的内存结果。

### Q3：出现 `Dead cycle!` 怎么办？

通常是 LDMC 参数错误。正确加载程序：

```asm
MOV R1,0900    ; 微码起始地址
MOV R2,0007    ; 微指令条数
MOV R3,0100    ; 微控存入口
LDMC
RET
```

如果错误执行过 LDMC，建议关闭 Monitor 和 TEC-2 后重启，从 E900 重做。

**经验口诀：**

```text
E900 错了 → 重新 E900 改
A800 错了 → 没 G800 前可重新 A800 改
LDMC 错执行了 → 重启
```
