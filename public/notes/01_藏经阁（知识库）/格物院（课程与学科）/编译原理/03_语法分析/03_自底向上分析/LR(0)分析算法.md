# LR(0)分析算法：盲目直接归约的朴素分析机制

> English: **LR(0) Parsing Algorithm**

**LR(0)** 是自底向上语法分析的基石。在决策时，它完全不看向前看符号（Lookahead 数量为 0），仅根据当前分析栈顶的状态查表决定是进行 **移进 (Shift)** 还是 **归约 (Reduce)**。

---

## 1. 直觉比喻：凭直觉盲开的闭眼大客车 (Blindfolded Driver)

> [!NOTE]
> 为了理解 LR(0) 的动作和它的致命缺陷，我们可以把它比作一次**“凭感觉在迷宫里闭眼盲开大客车”**的越野旅程：
> * **分析栈 $\to$ 行车历史记录**：里面装着你之前走过的路线（状态与符号）。
> * **移进 (Shift) ── “继续踩油门往前开”**：司机越过当前的界限，驶入下一个路口（吃进下一个终结符并给它贴上目标状态小贴纸）。
> * **归约 (Reduce) ── “拉起手刹原地封箱”**：当司机凭记忆觉得刚才走过的路线刚好可以拼成一个大路标（匹配到产生式右部）时，直接踩刹车，把这段零件折叠弹出，换成语法大件（非终结符）。
> 
> > [!CAUTION]
> > **LR(0) 的致命缺陷：到站闭眼拉手刹**
> > 只要司机在车站大厅里看到有一条路走完了（项目里有归约项 $B \to \gamma \cdot$），他就会**不管三七二十一，也绝不看眼前的下个路标是什么**，直接一脚把手刹拉死开始打包归约（分析表里这一整行所有的列全部填 `r`）。
> > 如果当前大厅里同时还有别人在准备继续踩油门往前开（移进项 $A \to \alpha \cdot a \beta$），当下一个路标恰好是 $a$ 时，司机会在“踩油门”和“拉手刹”之间彻底抓狂。这就是致命的 **“移进-归约冲突”**。

---

## 2. 输入与输出 (Inputs & Outputs)

*   **输入**：输入缓冲区的 Token 流（以  $\text{＄}$  结尾），以及静态的 `ACTION/GOTO` 分析表。
*   **输出**：若输入串合法，输出归约使用的产生式序列（构建语法分析树的逆过程）；若非法，报告语法错误。
*   **核心结构**：双栈驱动。
    *   **状态栈**：记录历史状态（如 `[0, 3, 5]`）。
    *   **符号栈**：记录匹配符号（如 `[$, id, +]`）。

---

## 3. 驱动程序与物理执行 (Driver & Execution Loop)

LR 家族所有变体均运行**同一个**驱动循环程序，其物理执行逻辑如下：

```text
初始化：状态栈 push 0，符号栈 push $，读入第一个 Token a
while (true) {
    s = 状态栈.top()
    action = ACTION[s, a]
    
    if (action == Shift j) {               // 移进动作 (s_j)
        符号栈.push(a)
        状态栈.push(j)
        a = 读入下一个 Token()
    }
    else if (action == Reduce A -> β) {    // 归约动作 (r_k)
        从符号栈弹出 |β| 个符号
        从状态栈弹出 |β| 个状态
        s_prime = 状态栈.top()
        符号栈.push(A)
        状态栈.push(GOTO[s_prime, A])      // 查 GOTO 表压入新状态
        输出产生式 A -> β
    }
    else if (action == Accept) {           // 接受动作 (acc)
        语法分析成功结束，返回
    }
    else {                                 // 报错 (err)
        报告语法错误并执行恢复
    }
}
```

## 4. 依赖的静态结构 (Dependencies)

LR(0) 分析算法查阅的分析表由以下静态结构映射生成：
1.  **依赖项目**：[[LR(0)项目]] （不带展望符的微观进度）。
2.  **依赖状态图**：[[LR项目集DFA]]。
3.  **填表映射规则**：
    *   若  $[A \to \alpha\cdot a\beta] \in I_i$  且  $GOTO(I_i, a) = I_j$ ，填入  $ACTION[i, a] = s_j$ （移进）。
    *   若  $[A \to \alpha\cdot] \in I_i$ （ $A \neq S'$ ），则**无条件对所有列**填入  $ACTION[i, a] = r_k$ （归约）。
    *   若  $[S' \to S\cdot] \in I_i$ ，填入  $ACTION[i, \text{＄}] = acc$ 。

## 5. 算法对比 (Comparison)

| 特性 | LR(0) | [[SLR(1)分析算法]] |
| :--- | :--- | :--- |
| **状态数** | 较少 | 较少（与 LR0 完全相同） |
| **归约填表依据** | 包含归约项即在**整行所有列**填归约 | 仅在属于  $\text{FOLLOW}(A)$  的列填归约 |
| **冲突消解** | 无法消解任何移进-归约和归约-归约冲突 | 能够通过  $\text{FOLLOW}$  集合交叉过滤，消解大部分冲突 |
