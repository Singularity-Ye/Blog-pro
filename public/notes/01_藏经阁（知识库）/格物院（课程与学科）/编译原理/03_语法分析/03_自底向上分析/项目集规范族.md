# 项目集规范族：分析器所有可能状态的完整合集

> English: **Canonical Collection of LR Items**

**项目集规范族**（在构建 DFA 时常称为 $\mathcal{C}$）说白了就是我们**“开地铁时手里拿着的全城地铁线路总导航图”**。它是从最开始的起点车站出发，通过反复求闭包和算跳转，把全城所有可能修起来的“地铁换乘站（状态）”以及它们之间“轨道连线（跳转边）”全部绘制出来后，得到的**终极路线图**。

---

## 🚗 直觉比喻：地铁线路网中的“换乘大厅与导航地图” (Metro Hub & Navigation Map)

> [!NOTE]
> **项目集规范族（大白话通俗解释）**
> 
> 如果把自底向上语法分析看作开地铁，那么项目集规范族就是整座城市的**“地铁线路总导航图”**：
> 
> *   **换乘大厅（DFA 状态 / 项目集 $I_i$）**：
>     状态机里的每个状态都是一个庞大的“换乘枢纽站”。这个车站大厅里挂着多块路线指示牌（各种项目），比如“1号线正开往A站”、“2号线已驶过B站正准备前往C站”。这些指示牌共同描述了当前可能面临的多种推导状态。
> *   **换乘通道（GOTO 边 / 跳转弧）**：
>     当你读到字符 $X$ 时（相当于在站台坐上了 $X$ 线路地铁），列车就会开动，带你沿着特定的换乘通道**瞬移到下一座确定的换乘大厅**（跳转到下一个状态）。
> *   **全城地铁图（项目集规范族）**：
>     指把所有可能建出来的换乘站（所有可能的状态）以及连接它们的全部地铁轨道（GOTO 边）绘制在一起，形成的一张**全城地铁总地图**。只要手里拿着这幅地图，分析器就能确保在任何复杂输入下闭着眼睛开地铁，也绝不会开错车、迷失站台。

## 形式化构建算法

项目集规范族的构建是一个典型的 **不动点迭代算法** ：

$$\text{构建项目集规范族 } \mathcal{C} \text{ 的算法：}$$

```text
输入：增广文法 G' (引入起始产生式 S' → S)
输出：项目集规范族 C = { I_0, I_1, ..., I_n }

1. 初始化初始状态：
   I_0 = CLOSURE({ [S' → ·S] })  (对于 LR1，初始为 [S' → ·S, $])
   
2. 初始化规范族集合：
   C = { I_0 }
   
3. 反复迭代求取不动点：
   repeat
     对 C 中的每一个状态 I：
       对文法中的每一个符号 X（包括终结符和非终结符）：
         J = GOTO(I, X)
         if J ≠ ∅ 且 J ∉ C：
           将新状态 J 加入到规范族中：C = C ∪ { J }
   until C 不再发生任何变化（达到收敛）
   
4. 返回最终规范族 C
```

## 🔍 不同 LR 算法下的规范族差异（核心对比）

不同的自底向上算法在构造平行宇宙地图时，所用的“材料精度”和“最终地图大小”有着天壤之别：

| 自底向上算法 | 项目（状态单元）的物理结构 | $\text{CLOSURE}$ 运算特征 | 规范族的状态总数 (地图尺寸) |
| :--- | :--- | :--- | :--- |
| **LR(0)** | 粗糙的无前看项目：<br>$A \to \alpha\cdot\beta$ | 经典闭包，直接向下游展开产生式。 | **最小基数 $N$** |
| **SLR(1)** | 完全使用 **LR(0) 的规范族** | 完全同 LR(0)。（*前看符号只在填 ACTION 表时用 Follow 过滤，不影响规范族构造*） | **等于 LR(0) 状态数 $N$** |
| **LR(1)** | 带局部精确前看的项目：<br>$[A \to \alpha\cdot\beta, \; a]$ | 高精度闭包，利用 $\text{FIRST}(\beta a)$ 向下游传递和分裂前看符号。 | **状态数爆炸** ，因前看分裂，膨胀至 $5N \sim 15N$ |
| **LALR(1)** | 同心的局部前看项目：<br>$[A \to \alpha\cdot\beta, \; a_1 \cup a_2 \dots]$ | 先构造 LR(1) 规范族，再将 **核心项完全一致（同心）** 的状态合并，前看符号集取并集。 | **压缩回 LR(0) 的数量级 $N$** |

## 💡 核心概念：活前缀 (Active Prefix) 与规范族的关系

在考研和期末大题中，常会出现关于“活前缀”的论述。
*   **活前缀 (Active Prefix)** ：是指规范句型的一个前缀，它 **绝不越过当前句型的最右端句柄** 。只要输入的前缀是“活的”，就说明到目前为止分析器没有读入任何错字，语法解析依然有戏。
*   **状态的本质** ：规范族中的每一个状态 $I_k$，在物理上都代表了 **所有能够到达该状态的活前缀的集合** 。DFA 就像是一个漏斗，把无数种走向同一进度的前缀路线（输入序列）全部收纳进了同一个状态中。

## 🎯 应试痛点与解题模板 (拿分关键)

> [!IMPORTANT]
> **考试避坑指南**
> 
> 1. **规范族收敛判定（不动点）** ：
>    什么时候停止状态机的计算？当对当前规范族中的所有状态 $I_i$，在所有可能的文法符号 $X$ 上求 $\text{GOTO}(I_i, X)$ 所得到的所有非空新项目集，都已经存在于规范族中时，说明地图绘制完成，算法收敛。
> 2. **千万别忘了求闭包** ：
>    在计算 $\text{GOTO}(I, X)$ 转移时，很多同学在把圆点移过 $X$ 后就直接把结果写成新状态了。 **必须对移过圆点后的核心项集求 $\text{CLOSURE}$** ，否则状态内会漏掉大量闭包项，导致后续分析动作彻底出错。
> 3. **同心合并与冲突判定** ：
>    - **移进-归约冲突** ：合并同心状态 **绝对不会** 引入新的移进-归约冲突（如果原本就没有的话）。
>    - **归约-归约冲突** ：合并同心状态 **可能会** 引入新的归约-归约冲突（因为前看符号集取了并集，使得两个产生式在同一个符号下都可以触发归约）。
