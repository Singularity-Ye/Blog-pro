# Chomsky文法层级：从最自由到最规矩的语言谱系

> English: **Chomsky Hierarchy**

**Chomsky 文法层级** 是由语言学家诺姆·乔姆斯基（Noam Chomsky）于 1956 年提出的形式语言分类体系。它根据 **产生式规则的严格程度**，将形式文法划分为 4 个层级（从 0 型到 3 型），呈包含关系。

> [!NOTE] 双轨直觉：规则松紧带 ── 从宇宙神明法典到死板流水线指令
> 乔姆斯基层级本质上是**规则严格程度的四级松紧带**。规则越松，表达能力越恐怖，但机器解析的代价也越灾难：
> 1. **0 型（无限制）像“宇宙神明法典”**：写什么规则都可以（$\alpha \to \beta$），甚至可以直接把一大段咒语抹除（$\beta = \varepsilon$）。表达力通天（等价图灵机），但根本无法保证编译器能计算出结果。
> 2. **1 型（上下文有关）像“见人说人话，见鬼说鬼话”**：只有在特定的环境 $\alpha \dots \beta$ 中，非终结符 $A$ 才能变身。并且不能越变越短。
> 3. **2 型（上下文无关）像“乐高积木拼装”**：左边只有一个非终结符 $A$。只要看见这个积木，不管它插在城堡顶上还是车轮底下（上下文无关），都按相同的配方拆解。这对于描述具有嵌套对称结构的程序语法（如 `if-else`、配对括号）刚刚好。
> 4. **3 型（正则/正规）像“单向行驶的死板流水线”**：非终结符只能乖乖呆在最左端或最右端，无法做居中嵌套。它只能单向延伸，虽然做不出完美的“对称嵌套括号”，但识别速度极快，是词法分析（Scanner）的御用引擎。

## 🗺️ 乔姆斯基四类文法总览

| 文法类型 | 常见名称 | 产生式规则形式 | 识别自动机 | 目标语言 | 编译原理应用 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **Type 0** | **无限制文法**<br>(Unrestricted) | $\alpha \to \beta$<br>无任何字符限制 | **图灵机**<br>(Turing Machine) | 递归可枚举语言<br>(Recursively Enumerable) | 理论计算极限，编译器不采用 |
| **Type 1** | **上下文有关文法**<br>(Context-Sensitive) | $\alpha A \beta \to \alpha \gamma \beta$<br>其中  $\gamma \neq \varepsilon$ ，且左部长度 $\le$ 右部 | **线性有界自动机**<br>(Linear Bounded Automaton) | 上下文有关语言<br>(Context-Sensitive Language) | 自然语言处理，部分静态语义分析 |
| **Type 2** | **上下文无关文法**<br>(Context-Free, CFG) | $A \to \alpha$<br>左部**必须是单个**非终结符 | **下推自动机**<br>(Pushdown Automaton, PDA) | 上下文无关语言<br>(Context-Free Language) | **语法分析器 (Parser)**<br>如用 Bison/Yacc 描述语法 |
| **Type 3** | **正规文法 / 正则文法**<br>(Regular Grammar) | 右线性： $A \to w B$  或  $A \to w$ <br>左线性： $A \to B w$  或  $A \to w$  | **有限自动机**<br>(Finite Automaton, FA) | 正规语言 / 正则语言<br>(Regular Language) | **词法分析器 (Scanner)**<br>如用 Lex/Flex 识别 Token |

## 📐 形式化定义与限制剖析

四种文法都定义在四元组  $G = (V_N, V_T, P, S)$  之上，区别完全在于产生式集合  $P$  的规则限制：

### 1. 0 型文法 (Type 0 - Unrestricted Grammar)
*   **规则限制**： $\alpha \to \beta$ ，其中  $\alpha \in (V_N \cup V_T)^+$ ，且至少包含一个非终结符； $\beta \in (V_N \cup V_T)^*$ 。
*   **解读**：最随意的文法，左边的一堆符号可以随意被替换为右边的一堆符号，甚至可以直接被抹除（即  $\beta = \varepsilon$ ）。

### 2. 1 型文法 (Type 1 - Context-Sensitive Grammar, CSG)
*   **规则限制**：产生式为  $\alpha A \beta \to \alpha \gamma \beta$ ，且要求  $\gamma \neq \varepsilon$ （除非  $S \to \varepsilon$  为特例）。或者一般形式为  $\alpha \to \beta$ ，满足  $|\alpha| \le |\beta|$ 。
*   **解读**：**上下文有关**。只有在  $A$  处于左侧是  $\alpha$ ，右侧是  $\beta$  的特定“上下文”中时，才能被改写为  $\gamma$ 。并且，替换出来的结果不能变短（防止无限缩减产生死循环）。

### 3. 2 型文法 (Type 2 - Context-Free Grammar, CFG)
*   **规则限制**：产生式为  $A \to \alpha$ ，其中  $A \in V_N$ ， $\alpha \in (V_N \cup V_T)^*$ 。
*   **解读**：**上下文无关**。左侧**有且仅有**一个非终结符  $A$ ，它的展开与替换**完全不需要考虑它所处的上下文**，在任何地方它都可以直接变为  $\alpha$ 。
*   **为什么用于语法分析**：因为程序设计语言的语法结构（如配对的括号、嵌套的 if-else 语句）需要用栈（下推自动机）来进行自底向上或自顶向下的嵌套归约，这恰好对应 2 型文法的计算能力。

### 4. 3 型文法 (Type 3 - Regular Grammar)
*   **规则限制**：产生式必须全部为**右线性**（ $A \to w B \mid w$ ）或全部为**左线性**（ $A \to B w \mid w$ ），其中  $w \in V_T^*$ ， $A, B \in V_N$ 。注意**左右线性不能混用**。
*   **解读**：产生的非终结符只能老老实实呆在最左边或最右边，无法产生对称嵌套（如递归括号配对）。
*   **为什么用于词法分析**：正规文法与正则表达式（RE）等价，其识别自动机是没有内存和调用栈的**有限状态自动机（FA）**。这使得我们可以用极高的速度（ $O(n)$ 时间复杂度）进行 Token 匹配，特别适合 Scanner。

## 🎨 包含关系图

从表达能力上，Type 0 最强，Type 3 最弱，呈严格包含关系：

```text
+-----------------------------------------------------+
| Type 0: 无限制文法 (图灵机)                         |
|   +-----------------------------------------------+ |
|   | Type 1: 上下文有关文法 (线性有界自动机)       | |
|   |   +-----------------------------------------+ | |
|   |   | Type 2: 上下文无关文法 (下推自动机)     | | |
|   |   |   +-----------------------------------+ | | |
|   |   |   | Type 3: 正规文法 (有限自动机)     | | | |
|   |   |   +-----------------------------------+ | | |
|   |   +-----------------------------------------+ | |
|   +-----------------------------------------------+ |
+-----------------------------------------------------+
```

> [!WARNING]
> **考试经典判断题**：
> *   *“所有正规文法都是上下文无关文法吗？”* —— **是的**，因为 Type 3 包含在 Type 2 之中。
> *   *“上下文无关文法可以表达任何程序设计语言的特征吗？”* —— **不能**。例如“变量必须先声明后使用”这一规则在上下文无关文法中是无法描述的（因为这与上下文强相关），编译器通常在**静态语义分析**（Chapter 6）阶段通过属性计算来处理这种“上下文相关”特征。
