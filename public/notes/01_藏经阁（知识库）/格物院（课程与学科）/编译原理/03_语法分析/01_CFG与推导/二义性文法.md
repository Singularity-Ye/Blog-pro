# 二义性文法：同一句话能长出两棵语法树

> English: **Ambiguous Grammar**

**二义性文法** 是指能够对同一个句子产生 **多种不同语法分析树** 的上下文无关文法。在编译器设计中，二义性会导致语义含糊不清（如产生不同的运算顺序），因此必须进行消除或在解析器中强加限制规则。

> [!NOTE] 双轨直觉：歧义双关语 ── 听者脑补的两幅面孔
> - **文字双关语的歧义**：就像中文里的“我想起来了”（到底是“我想起某件事了”，还是“我打算从床上起床了”）。因为缺少前置优先级/结合性约束，听者脑海里会蹦出两幅完全不同的意思画面。
> - **文法的二义性**：同一个输入符号串（如 `3 + 5 * 2`），在没有规定结合性时，编译器可以脑补出“先加后乘”或“先乘后加”两棵不同的语法分析树。由于分析树的结构直接决定了后续生成代码的求值顺序，这种理解歧义会产生灾难性的运行时计算错误。

---

## 形式化定义

如果对于文法  $G$  描述的语言中的某个句子  $w$ ，在  $G$  中存在以下任意一种情况，则称文法  $G$  是二义性的：
1.  $w$  存在 **两棵或两棵以上不同的语法分析树**。
2.  $w$  存在 **两个或两个以上不同的最左推导**。
3.  $w$  存在 **两个或两个以上不同的最右推导**。

> [!CAUTION]
> **考试概念细节**：判断一个文法是否具有二义性，**只需找到一个句子**能构造出两棵分析树即可。
> 另外，文法的二义性是**不可判定的**，即不存在一个通用算法能对任意文法输入自动判断其是否二义。

## 经典二义性案例

### 1. 经典算术表达式二义性
对于文法：
$$E \to E + E \mid E \times E \mid \text{id}$$

对于句子 `id + id * id`，可以产生以下两棵截然不同的语法分析树：

```text
树 A (先加后乘，违背常理):                树 B (先乘后加，符合常理):
         E                                        E
       / | \                                    / | \
      E  *  E                                  E  +  E
    / | \    |                                 |    / | \
   E  +  E  id                                id   E  *  E
   |     |                                         |     |
  id    id                                        id    id
```

*   **后果**：由于分析树的结构直接决定了编译器后续生成代码时的求值顺序，树 A 会导致先计算 `+` 再计算 `*`，产生逻辑错误。

### 2. 悬空 else 二义性 (Dangling-Else Problem)
对于文法：
$$\text{stmt} \to \text{if } \text{expr} \text{ then } \text{stmt} \mid \text{if } \text{expr} \text{ then } \text{stmt} \text{ else } \text{stmt} \mid \text{other}$$

对于嵌套语句 `if E1 then if E2 then S1 else S2`：
*   **树 1**：将 `else S2` 与**外层**的 `if E1` 配对。
*   **树 2**：将 `else S2` 与**内层**的 `if E2` 配对（人类普遍直觉）。
*   **消解**：多数程序语言规定 `else` 总是与最近的且未配对的 `then` 结合，并在 Bison 等 Parser 工具中硬性规定优先级来消解此冲突（参考 [[移进-归约冲突]] 中的 Bison 消解）。

## 二义性文法的消除（文法重写）

消除二义性的主要手段是根据算术运算符的 **优先级（Precedence）** 和 **结合性（Associativity）** 规律，通过引入新的非终结符重写文法。

### 1. 优先级嵌套规则
*   **规律**：越是需要 **优先计算（高优先级）** 的运算符，在文法中必须被安排在 **越深（距离开始符号越远，越靠近叶子）** 的层次结构中。
*   **层次构造**：
    *   表达式层  $E$ ：处理低优先级的加减法。
    *   项层  $T$ ：处理高优先级的乘除法。
    *   因子层  $F$ ：处理括号、标识符等基本粒子。

### 2. 结合性递归规则
*   **左结合**（如 `+`）：应用 **左递归**。如  $E \to E + T \mid T$ 。
*   **右结合**（如幂运算 `^`）：应用 **右递归**。如  $E \to T \text{ \^{} } E \mid T$ 。

### 改造后无二义性表达式文法：
$$\begin{aligned}
E &\to E + T \mid T \\
T &\to T \times F \mid F \\
F &\to (E) \mid \text{id}
\end{aligned}$$

用此文法推导 `id + id * id` 时，只能构造出唯一的树 B（先乘后加），二义性完美消解。具体改造步骤与黄金公式模板可参考解题套路卡片 [[01_重写二义性文法套路]]。
