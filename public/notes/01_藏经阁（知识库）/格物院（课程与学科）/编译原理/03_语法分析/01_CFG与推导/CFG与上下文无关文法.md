# CFG与上下文无关文法：描述程序语言结构的蓝图

> English: **Context-Free Grammar (CFG)**

**上下文无关文法 (CFG)** 是描述程序设计语言语法结构的数学模型，也是编译器 **语法分析器 (Parser)** 的理论基石。

> [!NOTE] 双轨直觉：俄罗斯套娃与上下文无关规则
> - **非终结符** 是各种各样的“嵌套大礼盒”（如 `[expr]`, `[stmt]`），**终结符** 是最底层的“实体小玩具”（如 `id`, `+`）。
> - **“上下文无关（Context-Free）”的含义**：只要你看到一个标记为 `[Table]` 的礼盒，无论它是摆在厨房还是操场（完全不需要理会它所处的上下文环境），你都可以根据唯一的配方，把它直接拆开替换为 `[4根桌腿 1个桌面]` 的下一层礼盒或玩具。

---

## 形式化数学定义

一个上下文无关文法  $G$  是一个四元组：
$$G = (V_N, V_T, P, S)$$

其中各部分的严格定义为：
1.  $V_N$  是 **非终结符集合**（Nonterminals）。在推导中会被进一步展开替换，通常用大写字母（如  $E, T, F, S$ ）表示。
2.  $V_T$  是 **终结符集合**（Terminals）。即词法分析输出的 Token，是推导的终点，通常用小写字母或关键字（如  $\text{id}, +, \text{if}$ ）表示。且满足：
    $$V_N \cap V_T = \emptyset$$
3.  $P$  是 **产生式集合**（Productions）。它是定义替换规则的有限集合，每个产生式的形式为：
    $$A \to \alpha$$
    其中  $A \in V_N$ （左侧必须是**单个**非终结符），而  $\alpha \in (V_N \cup V_T)^*$ （右侧是由非终结符和终结符构成的任意符号串，可以为空  $\varepsilon$ ）。
4.  $S \in V_N$  是唯一的 **开始符号**（Start Symbol）。推导过程的起点。

---

## 推导方式 (Derivations)

从开始符号  $S$  出发，通过不断应用产生式规则，将非终结符替换为右部的符号串，最终得到全由终结符组成的句子的过程，称为 **推导**。根据每次替换的位置不同，推导分为：

### 1. 最左推导 (Leftmost Derivation)
*   **规则**：在推导的每一步中，**仅替换符号串中最左边的非终结符**。
*   **特点**：自顶向下语法分析器（如 [[LL(1)文法]] ）在匹配句子时，物理上模拟的就是最左推导的顺序。

### 2. 最右推导 (Rightmost Derivation / 规范推导)
*   **规则**：在推导的每一步中，**仅替换符号串中最右边的非终结符**。
*   **规范推导 (Canonical Derivation)**：在编译原理中，最右推导又被称为规范推导。
*   **特点**：自底向上语法分析器（如 LR 家族）的“归约”动作，其逆过程恰好是 **最右推导的逆过程**（称为最左归约 / 规范归约）。

---

## 语法分析树 (Parse Tree / 推导树)

推导过程可以图形化地表示为一棵树，称为 **语法分析树**。

### 语法分析树特征：
*   **根节点**：文法的开始符号  $S$ 。
*   **内部节点**：必须是非终结符  $A \in V_N$ 。其子节点按从左到右顺序恰好对应应用在  $A$  上的产生式右部  $\alpha$ 。
*   **叶子节点**：可以为终结符（Token）或空字符  $\varepsilon$ 。从左到右拼接所有叶子节点，即得到推导出的输入串（句型）。

```text
文法：E → E + T | T，T → id
句子：id + id

最左推导过程：
E ⇒ E + T ⇒ T + T ⇒ id + T ⇒ id + id

对应语法分析树：
       E
     / | \
    E  +  T
    |     |
    T     id
    |
   id
```

## 避坑与总结

1.  **Chomsky 层级定位**：上下文无关文法属于 [[Chomsky文法层级]] 中的 **2 型文法**。它在表达能力上远强于正则文法（3型），能够描述括号嵌套、if-else 嵌套等非规则语言结构。
2.  **左侧限制**：牢记 CFG 左部必须是**单个**非终结符！若左边出现两个以上符号（如  $\alpha A \to \gamma$ ），则属于上下文有关文法（1型）。
3.  **多叉树表示**：语法分析树与抽象语法树（AST）是不同的物理概念。详细对比与连线，可参考 Ch 1 中的 [[1.1_编译器结构与翻译流程#CST (分析树) vs. AST (语法树)]]。
