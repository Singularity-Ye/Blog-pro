# 属性文法设计套路

> [!NOTE] **🌟 大白话通俗解释：设计公司的“自下而上汇报”与“自上而下派发”系统**
> 属性文法设计，本质上就是给一棵语法树（就像一个公司的组织架构图）建立一套**信息流转机制**。
> 
> - **综合属性（自下而上汇报）**：就像基层员工向经理汇总数据。比如统计一个二叉树里有多少个节点、整棵树的最大值是多少。这些数据在子树没算完之前，根节点是无法知道的。
> - **继承属性（自上而下派发 / 兄弟横向交接）**：就像总经理往下发红头文件，或者左边的同事把接力棒交接给右边的同事。比如“当前作用域的嵌套深度”、“变量声明的类型（`int` 或 `float`）”。子节点必须依赖上级或左侧传下来的信息。
> 
> 设计一个及格的属性文法，就像设计物流派送系统，不能让任何一个节点出现“断流”或“空等”。

---

## 📐 考场硬核设计四步法

在考场上，面对“设计属性文法/用 Bison 编写语义动作”的大题，按照以下四步规范作答：

### 步骤一：属性拆解与角色定性
明确你需要哪些属性，并划分它们是“综合”还是“继承”。
*   **计算性/验证性结果**（如 `order`, `val`, `type`）：一般设计为**综合属性**。
*   **子树极值/上下界**（如 `min`/`smallest`, `max`/`largest`）：必须设计为**综合属性**，以便父节点做区间相容性判定。
*   **环境上下文**（如 `env`, `depth`）：一般设计为**继承属性**。

### 步骤二：设定极值哨兵（空/叶子节点的边界值）
当文法支持空节点（如 `nil` 或 `ε`）时，极值的初始化非常关键：
*   **求最小值（min/smallest）**：空树的最小值应设为 **正无穷 ($+\infty$)**（代码中用 `INT_MAX`）。
*   **求最大值（max/largest）**：空树的最大值应设为 **负无穷 ($-\infty$)**（代码中用 `INT_MIN`）。
*   *直觉原因*：这样任何真实的数字与空树进行 `min`/`max` 运算时，真实数字都能顺利胜出。

### 步骤三：编写完整的极值与状态传递方程
> **🚨 考场致命扣分点：遗漏更新方程。**
> 如果你引入了 `largest` 属性，你不仅要在根节点处用它，还**必须在每个产生式里写出它的合成规则**，把信息一级一级送上去。如果链条断了，整道题基本归零。

### 步骤四：套用 Bison 语法翻译
如果题目指定使用 Bison 语法，按照以下规则转换：
*   **`$$`**：指代产生式左部的非终结符。
*   **`$i`**：指代产生式右部第 $i$ 个符号（1-indexed）。
*   **注意**：严格数清每个符号的位置，**括号等终结符也要占位**。

## 🎯 经典真题步骤追踪 (以 [[Ex6.3_属性文法_二叉搜索树有序性判定]] 为例)

### 📌 题目原型
> 给定二叉树文法 `btree → ( number btree btree ) | nil`，编写属性文法判定其是否是有序二叉树（BST）。

### ✍️ 考场解题规范作答

```yacc
btree: '(' number btree btree ')'
    {
        /* 
           右部符号位置严格对照：
           $1: '(' (不带属性)
           $2: number (对应其数值 $2.num)
           $3: btree1 (左子树)
           $4: btree2 (右子树)
           $5: ')' (不带属性)
        */
        
        // 1. 判断当前子树是否满足有序条件（综合属性合成）
        $$.order = $3.order && $4.order 
                  && ($2.num >= $3.largest) 
                  && ($2.num <= $4.smallest);
        
        // 2. 传递极值链条（千万别漏！否则父节点求值断裂）
        $$.largest = max($2.num, $4.largest);
        $$.smallest = min($2.num, $3.smallest);
        
        $$.num = $2.num;
    }
    | nil
    {
        // 3. 空树的哨兵初值定义
        $$.order = true;
        $$.largest = INT_MIN; // 极大值设为负无穷
        $$.smallest = INT_MAX; // 极小值设为正无穷
        $$.num = -1;
    }
;
```

## ⚠️ 避坑指南

> [!CAUTION] **1. Bison 属性索引越界（Bug之王）**
> 产生式 `btree : '(' number btree btree ')'` 中，根节点数字 `number` 是第 **2** 个符号，引用它的属性必须写为 **`$2.num`**。
> 官方课件曾写为 `$1.num`，直接把 `'('` 误认为数字，导致严重越界，考场切记要在草稿纸上把右部符号按 1, 2, 3, 4, 5 标好。
> 
> **2. 极值逻辑不要写反**
> 判断 BST 时，根节点必须 **$\ge$ 左子树最大值**（`$3.largest`）且 **$\le$ 右子树最小值**（`$4.smallest`）。注意是与左子树的最大值、右子树的最小值进行比较，不要张冠李戴写成 `$3.smallest` 和 `$4.largest`。
