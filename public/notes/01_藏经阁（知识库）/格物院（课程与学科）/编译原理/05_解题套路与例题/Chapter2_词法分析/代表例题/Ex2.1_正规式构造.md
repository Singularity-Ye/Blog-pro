# Ex2.1 正规式构造

## Original Question

**2.1** Write regular expressions for the following character sets, or give reasons why no regular expression can be written:
*   **a.** All strings of lowercase letters that begin and end in *a*.
*   **b.** All strings of lowercase letters that either begin or end in *a* (or both).
*   **c.** All strings of digits that contain no leading zeros.
*   **d.** All strings of digits that represent even numbers.
*   **e.** All strings of digits such that all the 2's occur before all the 9's.
*   **f.** All strings of *a*'s and *b*'s that contain no three consecutive *b*'s.
*   **g.** All strings of *a*'s and *b*'s that contain an odd number of *a*'s or an odd number of *b*'s (or both).
*   **h.** All strings of *a*'s and *b*'s that contain an even number of *a*'s and an even number of *b*'s.
*   **i.** All strings of *a*'s and *b*'s that contain exactly as many *a*'s as *b*'s.

*(注：本次作业重点考察 **b, d, f, h** 四题)*

---

## 中文题意

**2.1** 为以下字符集编写正规式（正则表达式），或者说明为什么无法编写正规式：
*   **a.** 所有以 $a$ 开头且以 $a$ 结尾的由小写字母组成的字符串。
*   **b.** 所有以 $a$ 开头或以 $a$ 结尾（或两者兼有）的由小写字母组成的字符串。
*   **c.** 所有不含前导零的数字字符串。
*   **d.** 所有表示偶数的数字字符串。
*   **e.** 所有满足“所有 $2$ 都在所有 $9$ 之前出现”的数字字符串。
*   **f.** 所有不包含三个连续 $b$ 的由 $a$ 和 $b$ 组成的字符串。
*   **g.** 所有包含奇数个 $a$ 或奇数个 $b$ （或两者兼有）的由 $a$ 和 $b$ 组成的字符串。
*   **h.** 所有包含偶数个 $a$ 且包含偶数个 $b$ 的由 $a$ 和 $b$ 组成的字符串。
*   **i.** 所有包含相同数量的 $a$ 和 $b$ 的由 $a$ 和 $b$ 组成的字符串。

## ⚠️ 真实考场还原与作答深度对比

针对本次作业中重点考察的 **b, d, f, h** 四道典型题，我们将 **学生作答手稿** 与 **官方标准答案** 进行逐一比对和深度学术剖析：

### 1. 题 b：开头或结尾为 a
*   **学生手稿**：`(a[a-z]*)|([a-z]*a)`
*   **官方答案**：`a[a-z]*|[a-z]*a|a[a-z]*a`
*   **学术剖析**：**完全等价，学生答案甚至更简洁（无冗余项）**。
    *   官方答案中的最后一项 `a[a-z]*a`（表示既以 $a$ 开头又以 $a$ 结尾）纯粹是为了对应题干中“or both”的直译。
    *   但在集合论和正规代数中，$a[a-z]*a$ 本身就是 $a[a-z]*$ 的子集（也是 $[a-z]*a$ 的子集）。因此，`a[a-z]*|[a-z]*a` 已经通过并集运算完全覆盖了“两者兼有”的情况，官方答案中的第三项属于 **数学上的冗余项**。学生手稿解答非常精准。

### 2. 题 d：偶数字符串
*   **学生手稿**：`digit*[02468]`
*   **官方答案**：`[0-9]*(0|2|4|6|8)`
*   **学术剖析**：**完全正确**。
    *   学生使用了定义好的字符类 `digit`（在教材上下文中定义 `digit = [0-9]`）。
    *   只要末尾数字属于偶数集合 `[02468]`，即可代表十进制偶数。

### 3. 题 f：不含三个连续 b
*   **学生手稿**：`(a|ba|bba)*(b|bb|ε)`
*   **官方答案**：`((ε|b|bb)a)*(ε|b|bb)`
*   **学术剖析**：**完全正确，写法风格不同**。
    *   学生手稿采用的是“以 $a$ 结尾的块”作为循环主体，即 `a`, `ba`, `bba`，这确保了循环体内部任何地方最多只有两个连续的 $b$。最后在尾部补上可选的 `b` 或 `bb`（或空 `ε`）。
    *   官方答案将可选的前置 $b$ 合并到了循环项内部：`((ε|b|bb)a)*`。如果将括号展开，该项等价于 `(a|ba|bba)*`。
    *   两种写法在正规式代数上完全等值，且学生答案直接写出 `a|ba|bba` 的平铺块更加具有直观的解题逻辑，不易出错。

### 4. 题 h：偶数个 a 且偶数个 b（经典易错题 🌟）
*   **学生手稿**：`(aa|bb|abab|abba|baba|baab)*` （**错误 ❌**）
*   **官方答案**：`((aa|bb)|(ab|ba)(aa|bb)*(ab|ba))*` （**正确 ✅**）
*   **学术剖析**：
    *   **手稿病因**：学生手稿试图通过列举长度为 2 和 4 的“偶数 $a$ 偶数 $b$”原子块（如 `aa`, `bb`, `abab` 等）并使用克莱尼闭包 `*` 来组合出所有合法串。然而，这遗漏了 **任意长度的 parity 嵌套转移**。
    *   **反例证明**：
        *   考虑字符串 **`abaabbba`**。它包含 4 个 $a$（偶数）和 4 个 $b$（偶数），属于目标语言。
        *   使用学生手稿的正则尝试匹配该串：开头是 `ab...`，但后接 `aa...`。在手稿块 `{aa, bb, abab, abba, baba, baab}` 中，没有任何一个块能够匹配前缀 `abaa`。因此，学生手稿无法匹配该合法串。
        *   使用官方正则匹配：它可以看作由 `(ab|ba)`（匹配 `ab`）开头的序列，中间夹着 `(aa|bb)*`（匹配其中的 `aa` 和 `bb` 组合，如 `aabb`），最后以 `(ab|ba)`（匹配末尾的 `ba`）封口。路径为：`ab` + `aabb` + `ba` = `abaabbba`，完美匹配。
    *   **状态机推导解释**：我们必须通过构建四状态 DFA 并进行状态消解来求解此题（参见下文“详细解析”）。

## Standard Solution 标准答案

### 1. 2.1 完整正规式表

| 题号 | 语言描述 | 规范正规式 (Regular Expressions) | 说明 |
| :--- | :--- | :--- | :--- |
| **a** | $a$ 开头且 $a$ 结尾 | `a[a-z]*a\|a` | 需单提长度为 1 的单个 `a` |
| **b** | $a$ 开开或 $a$ 结尾 | `a[a-z]*\|[a-z]*a` | 无需补充冗余交叉项 |
| **c** | 无前导零的数字 | `[1-9][0-9]*\|0` | 单个 `0` 是合法的，但不能有 `01` |
| **d** | 偶数 | `[0-9]*(0\|2\|4\|6\|8)` | 末尾为偶数字符 |
| **e** | $2$ 在所有 $9$ 之前 | `[0-8]*[0-1\|3-9]*` | 只要不出现 $9$ 后面跟 $2$ 的情况即可 |
| **f** | 无三个连续 $b$ | `(a\|ba\|bba)*(b\|bb\|ε)` | 以 $a$ 分割，$b$ 的连续个数被限制在 2 个以内 |
| **g** | 奇数个 $a$ 或奇数个 $b$ | `((b*ab*ab*)*b*ab*)\|((a*ba*ba*)*a*ba*)` | 奇数 $a$ 的正规式与奇数 $b$ 的正规式求并集 |
| **h** | 偶数个 $a$ 且偶数个 $b$ | `((aa\|bb)\|(ab\|ba)(aa\|bb)*(ab\|ba))*` | 经典的四状态偶数校验状态机消解（详见下文） |
| **i** | $a$ 和 $b$ 数量相等 | **无法写出正规式** | 正规语言无记忆（计数）能力（详见下文） |

### 2. 深入探讨 2.1(h) 的自动机推导

为了彻底理清 **“偶数个 $a$ 且偶数个 $b$”** 的构造逻辑，我们需要借助状态机模型：

#### (1) 四状态 DFA 建模
定义以下四个状态，代表已读入的 $a$ 和 $b$ 数量的奇偶性：
*   $S_{00}$：偶数个 $a$，偶数个 $b$（**初态 & 终态**）
*   $S_{10}$：奇数个 $a$，偶数个 $b$
*   $S_{01}$：偶数个 $a$，奇数个 $b$
*   $S_{11}$：奇数个 $a$，奇数个 $b$

DFA 的状态转移图如下：

```mermaid
stateDiagram-v2
    S00 --> S10 : a
    S10 --> S00 : a
    S00 --> S01 : b
    S01 --> S00 : b
    
    S10 --> S11 : b
    S11 --> S10 : b
    S01 --> S11 : a
    S11 --> S01 : a
    
    state S00 {
        [*] --> S00
    }
    style S00 fill:#d4edda,stroke:#28a745,stroke-width:2px
```

#### (2) 状态消解法（State Elimination）推导正规式
我们要从初态 $S_{00}$ 出发，在图上找出所有能够回到 $S_{00}$ 的路径集合：
1.  **直接环路**：直接由 $S_{00}$ 出发并返回的短环有：
    *   通过 $S_{10}$ 返回：`a a`
    *   通过 $S_{01}$ 返回：`b b`
    *   合并这两者：`(aa|bb)`
2.  **跨越环路（通过 $S_{11}$）**：
    *   从 $S_{00}$ 出发到达中间状态 $S_{11}$ 的路径有：先读入 $a$ 再读入 $b$（`ab`），或者先读入 $b$ 再读入 $a$（`ba`），即 `(ab|ba)`。
    *   在状态 $S_{11}$ 内部，可以进行自循环。在 $S_{11}$ 停留并返回自身的路径为：
        *   通过 $S_{01}$ 返回自身：`a a`
        *   通过 $S_{10}$ 返回自身：`b b`
        *   因此在 $S_{11}$ 的自循环正规式为：`(aa|bb)*`。
    *   从 $S_{11}$ 返回终态 $S_{00}$ 的路径同样为：`ab` 或 `ba`，即 `(ab|ba)`。
    *   因此，跨越 $S_{11}$ 并返回 $S_{00}$ 的完整路径为：
        $$(ab \mid ba)(aa \mid bb)^*(ab \mid ba)$$
3.  **最终闭包**：
    将上述直接环路与跨越环路求并集，并施加克莱尼星号 `*`，即得到最终正规式：
    $$\left( (aa \mid bb) \mid (ab \mid ba)(aa \mid bb)^*(ab \mid ba) \right)^*$$

### 3. 深入探讨 2.1(i) 的非正规语言证明

为什么 **“包含相同数量的 $a$ 和 $b$”** 无法写出正规式？

*   **直观解释**：正规式（或等价的 DFA）代表的是 **有限内存** 的计算模型。为了确保 $a$ 和 $b$ 的数量完全相等，自动机必须能够记住已经读入的 $a$ 的任意累积个数（这个数字可以是无限大），从而在读入 $b$ 时进行对等的减法。由于 DFA 的状态数是有限的，它无法存储无限的计数信息。
*   **数学证明（泵引理 Pumping Lemma）**：
    假设该语言 $L = \{ w \in \{a,b\}^* \mid N_a(w) = N_b(w) \}$ 是正规的，设其泵长度为 $p$。
    选择字符串 $s = a^p b^p$。显然 $s \in L$ 且长度 $|s| = 2p \ge p$。
    根据泵引理，$s$ 可以被拆分为 $s = xyz$，满足：
    1.  $|xy| \le p$
    2.  $|y| > 0$
    3.  对所有 $i \ge 0$，$xy^i z \in L$。
    
    由于 $|xy| \le p$，子串 $y$ 必定完全由字符 $a$ 组成（即 $y = a^k$，其中 $k \ge 1$）。
    此时，我们令 $i = 2$ 进行“泵入”，得到新串 $s' = xy^2 z = a^{p+k} b^p$。
    显然，在 $s'$ 中，$a$ 的数量为 $p+k$，而 $b$ 的数量为 $p$。因为 $k \ge 1$，所以 $p+k \neq p$。
    因此 $s' \notin L$，这与泵引理的假设矛盾。
    **结论**：该语言不是正规语言，无法为其书写正规式。该语言属于上下文无关语言（CFL），需要使用下推自动机（PDA）或上下文无关文法（CFG）来描述。

## 避坑指南 与 易错点

> [!WARNING]
> **小心“偶数校验”中的 Parity 漏项**：
> 构造带有多个字符奇偶校验（如“偶数个 $a$ 且偶数个 $b$”）的正规式时，**绝对不能简单地采用短周期串进行简单拼接闭包**（如写成 `(aa|bb|abab|abba)*`）。这种写法会直接漏掉像 `abaabbba` 这种由于内部嵌套了多对偶数项而导致长度拉长的合法情况。
> 
> **解题标准套路**：遇到此类奇偶校验题，考场上最稳妥的办法是：**先画出四状态 DFA，然后利用状态消解法一步步推导写出正规式**，切忌凭直觉瞎凑。
