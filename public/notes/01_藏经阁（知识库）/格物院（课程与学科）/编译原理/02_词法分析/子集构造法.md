# 子集构造法：从分头试探收敛到单线直达

> English: **Subset Construction**

**子集构造法** 是一种用于将包含模糊空转移（ $\varepsilon$ 边）和分裂分身路径的 [[NFA]] 转换为等价的确定性 [[DFA]] 的经典算法。

> [!NOTE] 双轨直觉：袋装收网与分身合体
> - **子集构造法（Subset Construction）** 的核心思想就是 **“收网合体与装袋行动”**。
> - **既然你在 NFA 里会自动瞬移、分身无数，那我干脆拿一张大网，把所有分身“可能站立的位置”统统网住，并塞进一个“确定的状态袋子”里。**
> - **袋子间跳转**：我们不再追踪单个分身，而是只追踪这一个个状态袋子。只要知道了怎么从“袋子 A”跳到“袋子 B”，非确定性的分身网络就变成了确定性的 DFA 状态网！

---

## 算法核心算子

> [!NOTE]- 📐 理论数学算子（精确集合定义，可折叠不看）
> 为了在算法中能够精确地将多个 NFA 状态合体打包，定义了两个核心算子：
> 
> ### 1. $\varepsilon\text{-closure}(s)$ （空闭包运算）
> - **数学定义**：从单个 NFA 状态 $s$ 出发，**仅通过 $\varepsilon$ 跳转边**所能到达的所有 NFA 状态的集合（包含自身）。
> - **集合扩展**：若 $T$ 是一个状态集合，则：
>   $$
>   \varepsilon\text{-closure}(T) = \bigcup_{s \in T} \varepsilon\text{-closure}(s)
>   $$
> - **直觉理解**：**“免费传送闭包”**。看看你或你的分身在某个状态下，顺着所有免费的 $\varepsilon$ 魔法滑梯溜过去，最远能同时占领哪些房间的组合。
> 
> ### 2. $\text{move}(T, a)$ （状态转移）
> - **数学定义**：从状态集合 $T$ 中的任意状态出发，在消耗了**一个且仅一个字符** $a$ 之后，所能直接到达的所有有向边终点状态 of 集合。
> - **直觉理解**：**“刷卡跨房”**。网袋 $T$ 里的所有人，必须刷卡消耗掉一个字符 $a$，整齐地越过有向边跨到下一个房间。
> 
> ### 3. 核心组合算子：$\varepsilon\text{-closure}(\text{move}(T, a))$
> - **直觉理解**：**“刷卡跨房 + 免费滑滑梯”**。先刷卡 $a$ 跨到对面的房间（第一步），然后允许大家从新到的位置出发，各自顺着免费滑梯（$\varepsilon$ 边）溜到最远的地方。最后将所有人站立的新状态网起来装入新袋子。

## 算法步骤：如何制作 DFA 的“袋子跳转网络”

### 1. 算法工作流大白话

1. **制作第一个袋子**：从 NFA 的初态 $s_0$ 开始，让它滑遍所有免费的 $\varepsilon$ 滑梯，得到初始状态子集 $s_0\_D = \varepsilon\text{-closure}(s_0)$。这就是 DFA 的初态袋子，放在“未处理堆”中。
2. **探索新袋子（迭代收敛）**：
   - 拿出一个“未处理”的袋子 $T$，标记为“已处理”。
   - 对于字母表中的每个可能字符 $a$，将袋子 $T$ 里的所有人打包进行一次“刷卡跨房 + 免费滑滑梯”动作：$U = \varepsilon\text{-closure}(\text{move}(T, a))$。
   - 如果 $U$ 是个全新的位置组合（我们还没缝制过这个袋子），就将 $U$ 缝制为一个新袋子并丢进“未处理堆”中。
   - 记录新边：从袋子 $T$ 读入字符 $a$ 跳转到袋子 $U$。
3. **寻找出口袋子（确定终态）**：
   - 检查我们缝制的所有袋子。如果某个袋子里装有的 NFA 状态中**包含至少一个**原 NFA 的终态，就把这个袋子标记为 DFA 的终态（双圈表示）。

---

### 2. 算法伪代码表示

```text
输入: NFA N = (S, 𝝨, 𝜹, s_0, F_N)
输出: DFA D = (S_D, 𝝨, 𝜹_D, s_0_D, F_D)

1. 计算初态袋子:
   s_0_D = 𝜺-closure(s_0)
   将 s_0_D 加入 Dstates 集合，标记为“未处理”

2. 迭代直至无新袋子产生:
   while (Dstates 中存在“未处理”的袋子 T) {
       标记 T 为“已处理”
       for (字母表 𝝨 中的每个字符 a) {
           U = 𝜺-closure(move(T, a))
           if (U 非空) {
               if (U 不在 Dstates 中) {
                   将 U 作为“未处理”的袋子加入 Dstates
               }
               𝜹_D[T, a] = U  // DFA 边：袋子 T 经由字符 a 跳转到袋子 U
           }
       }
   }

3. 确定哪些袋子是出口:
   对于 Dstates 中的每个状态子集 T:
     若 T 包含了原 NFA 中的任意一个终态 (T ∩ F_N ≠ ∅)，则该袋子 T ∈ F_D
```

## 列表法求解示例（考试必考）

在考场上手算 NFA 转 DFA 时，我们通常采用 **列表追踪表** 来清晰地记录每一次“装袋”与“状态迁移”过程。

设 NFA 的初始状态为 0，输入字符集为 `{a, b}`，终态为 3：

| DFA 状态 | NFA 状态子集 | 读入 `a` 转移 ($I_a = \varepsilon\text{-closure}(\text{move}(T, a))$) | 读入 `b` 转移 ($I_b = \varepsilon\text{-closure}(\text{move}(T, b))$) | 是否为终态 (包含 3) |
| :---: | :---: | :---: | :---: | :---: |
| **A (初态)** | $\varepsilon\text{-closure}(0) = \{0, 1, 2\}$ | $\{1, 3\}$ (新袋子，标记为 **B**) | $\{2, 4\}$ (新袋子，标记为 **C**) | 否 |
| **B** | $\{1, 3\}$ | $\{3\}$ (新袋子，标记为 **D**) | $\{4\}$ (新袋子，标记为 **E**) | **是**（包含终态 3） |
| **C** | $\{2, 4\}$ | $\emptyset$ | $\{4\}$ (即已有的袋子 **E**) | 否 |
| **D** | $\{3\}$ | $\emptyset$ | $\emptyset$ | **是**（包含终态 3） |
| **E** | $\{4\}$ | $\emptyset$ | $\emptyset$ | 否 |

> 📌 **做题套路提示**：
> 在做题时，求出新的子集后立刻用字母标记。如果求出的是空集 $\emptyset$，代表该路径撞墙。手算详细步骤和做题模板，可以参考 [[01_正规式转NFA与DFA套路]]。
