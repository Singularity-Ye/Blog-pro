# Flex与Bison实验避坑指南（核心考点速记）

在《编译原理》期末笔试中，老师非常喜欢将理论算法（如 NFA 确定化、LALR 属性传递、移进-归约冲突）与实验工具（Flex / Bison / Yacc）的实际语法结合考察。本卡片为您提炼了 Lab 1 与 Lab 2 中最核心的应试干货。

---

## 🧪 Lab 1：多模式词法分析（Flex / Lex 运行机制）

词法分析器（Scanner）的本质是：**输入源程序字符流，通过正规匹配切割出 Token 流。**

### 1. NFA $\varepsilon$-并与子集构造（多模式合并）
Flex 并不是并行跑好几个自动机，而是通过以下步骤将多个 Token 规则（如 `if`、`[a-z]+`、`[0-9]+`）合而为一：
*   **$\varepsilon$-并（Union）**：为每个 Token 构造一个独立的 NFA，然后引入一个**唯一的新起点 $S$**，通过 $\varepsilon$ 边连接到各个子 NFA 的初态。此时合并为一台包含空转移的巨大 NFA。
*   **确定化**：对这台 $\varepsilon$-NFA 进行**子集构造**（求解 $\varepsilon\text{-closure}$ 和 $\text{move}$），收拢为一个单一的 DFA。详细的构造套路请参考：[[01_正规式转NFA与DFA套路]]。

### 2. 考场冲突消解与最长匹配
*   **终态冲突判定**：如果化简后的 DFA 某个子集中同时包含多个 Token 的终态（例如字符串 `"if"` 既匹配关键字 `if`，也匹配标识符 `[a-z]+`），怎么判定？
    *   **Flex 的消解规则**：在规则文件中**定义越靠前，优先级越高**。因此关键字必须写在标识符的前面。
*   **最长匹配原则（Maximal Munch）**：从左到右读入时，即使当前进入了某个 Token 的终态，也会尝试继续往下读；直到遇到死状态（无法转移）时，才**回退**到最近一次经过的接受状态截断并输出。

## 🧪 Lab 2：科学计算器与语法分析（Bison / Yacc 实践）

Bison（Yacc 的现代版）通过 `LALR(1)` 算法自底向上分析 Token 流，并在规约时通过**语义动作**构造**抽象语法树（AST）**并求值。

### 1. Bison 文件的五大逻辑层次

一个标准的 `.y` 文件由两个 `%%` 分割为三段，但在结构上必须包含以下五个要素：

*   **① C 声明区（`%{ %}`，第一段）**：
    引入头文件，声明全局变量 and 函数原型（如错误处理 `yyerror`、`yylex`）。这部分代码会被原样复制到生成的 C 语言源文件中。
*   **② 语义值联合体（`%union`，第一段）**：
    定义分析栈中所有符号可能携带的**属性值类型**（对应[[属性文法]]中的属性类型）：
    ```yacc
    %union {
        double number;    /* NUMBER 词素的值 */
        char *text;       /* 标识符变量名字 */
        Node *node;       /* AST 节点的指针 */
    }
    ```
*   **③ 符号类型绑定（第一段）**：
    将 Token 和非终结符绑定到 `%union` 对应的数据类型，用于在语义动作中正确寻址：
    ```yacc
    %token <number> NUMBER  /* 声明 NUMBER 携带 double 类型的值 */
    %token <text> ID        /* 声明 ID 携带 char* 类型的值 */
    %type <node> expr       /* 声明非终结符 expr 的语义值是 Node* 指针 */
    ```
*   **④ 优先级与结合性声明（第一段）**：
    ```yacc
    %left '+' '-'           /* 左结合，优先级低 */
    %left '*' '/'           /* 左结合，优先级高 */
    %right '^'              /* 右结合，最高优先级 */
    ```
    *   **Bison 填表规则**：**越往后声明，优先级越高**。这直接用于消解[[移进-归约冲突]]。
*   **⑤ 文法规则与语义动作（第二段）**：
    ```yacc
    expr: expr '+' expr    { $$ = ast_binary("+", $1, $3); }
        | NUMBER           { $$ = ast_number($1); }
        ;
    ```
    *   **`$$`**：产生式左部符号（`expr`）的属性值（准备压栈）。
    *   **`$1`, `$3`**：产生式右部第 1 个和第 3 个符号的属性值。这套机制在物理底层被翻译为对 Bison 值栈数组指针 `yyvsp` 的相对偏移寻址（如 `yyvsp[1-3]` ）。

### 2. 后台辅助存储设计
在 Lab 2 语义计算时，必须在后台设计辅助存储：
*   **符号表 (Symbol Table)**：使用**单向链表**或哈希表，实现 `set("x", 5)` 和 `get("x")`。主要用于变量赋值语句和变量引用时读写。
*   **Memory 寄存器**：为了实现科学计算器的 `MC` (清除)、`MR` (读取)、`M+`/`M-` (累加/累减) 功能，使用一个独立于符号表的全局变量（如 `double calculator_memory`）来存放累加值，其语义与普通变量赋值不同。
