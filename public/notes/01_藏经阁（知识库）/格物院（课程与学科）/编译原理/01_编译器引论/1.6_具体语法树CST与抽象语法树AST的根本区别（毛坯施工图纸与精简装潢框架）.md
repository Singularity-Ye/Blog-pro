# 具体语法树CST与抽象语法树AST的根本区别：剥离语法细节的本质树结构

> English: **Concrete Syntax Tree (CST) vs. Abstract Syntax Tree (AST)**

在语法分析器的输出阶段，源程序的嵌套层次结构可以用树形结构表示。根据保留语法符号细节的多寡，分为**具体语法树 (CST / 分析树)** 与 **抽象语法树 (AST / 语法树)** 。

---

## 1. 🌟 大白话通俗解释 (核心直觉)

*   **生活化比喻 —— 房屋毛坯框架与简装图纸**：
    要把一栋房屋的结构记录下来：
    1.  **具体语法树 (CST)**：像一张**房屋施工毛坯图纸**。上面密密麻麻画满了临时支撑的钢筋、脚手架、水泥搅拌站的位置、还有定位用的水平线和辅助线。这些东西对于“建好这栋房”的中间过程必不可少，但对于“住进这栋房”来说完全多余。
    2.  **抽象语法树 (AST)**：像一张**精简装潢框架图**。脚手架和支撑钢筋全拆了，只画着：“客厅里摆一张沙发，卧室里放一张床”。它只保留了最终有用的空间结构，把施工过程中的辅助垃圾全部清扫干净。
*   **一句话总结**：
    **CST** 记录了文法推导的“施工过程”，保留了括号、分号等全部冗余符号；**AST** 只记录了推导的“最终骨架”，只保留算子（操作符）和操作数。

## 2. 📝 学术规范定义 (考试硬核)

### 2.1 具体语法树 (Concrete Syntax Tree / CST)

**CST**（又称**分析树 / Parse Tree**）是上下文无关文法推导过程的图形化表示。它严格、忠实地记录了语法分析器在推导句子时使用的**每一条产生式规则**。

*   **树节点特征**：
    *   **内部节点**：全是非终结符（如 `Expression`, `Term`, `Factor`）。
    *   **叶子节点**：全是终结符（包括变量名、数字，以及括号 `( )`、分号 `;`、逗号 `,`、运算符等所有具体的 Token）。
    *   **深度极大**：存在大量单产生式传递链（如 `E → T → F → id`），产生庞大冗余的层级。

### 2.2 抽象语法树 (Abstract Syntax Tree / AST)

**AST**（又称**语法树 / Syntax Tree**）是 CST 经过浓缩与提炼后的简易表示。它去除了文法中那些为了消除歧义而特意设计的辅助性非终结符与多余终结符。

*   **树节点特征**：
    *   **内部节点**：全是**算子（Operators / 运算符，如 `+`, `-`, `=`, `[]`）**或控制流关键字（如 `if`, `while`）。
    *   **叶子节点**：全是**操作数（Operands / 变量名、数值常量）**。
    *   **极度紧凑**：剔除了单链传递，树的深度和节点数量大幅压缩。

## 3. 🎯 考场真题演练与画图模板 (拿分关键)

### 经典考题：为表达式 `x = a + 3` 绘制 CST 和 AST

#### 步骤 1：分析底层文法规则
设算术表达式文法为：
$$
E \to E + T \mid T
$$
$$
T \to \text{id} \mid \text{num}
$$
$$
S \to \text{id} = E
$$

#### 步骤 2：绘制 CST (分析树)
CST 必须展现完整的推导轨迹，将每一个符号作为单独的节点画出：

```
                S
             /  |  \
           id   =   E
           |      / | \
           x     E  +  T
                 |     |
                 T     num
                 |     |
                id     3
                 |
                 a
```

#### 步骤 3：绘制 AST (抽象语法树)
在 AST 中，等号 `=` 和加号 `+` 直接作为内部算子节点，操作数直接挂在下方：

```
               =
             /   \
            x     +
                 / \
                a   3
```

### 💡 考场避坑指南
1.  **叶子节点检查**：CST 的叶子节点从左到右拼接，必须**完全等于原输入字符串**（包含等号、加号等所有字符）；而 AST 的叶子节点只有变量和数值。
2.  **算子位置**：在 AST 中，运算符（如 `+`）**绝对不能**作为叶子节点，它必须作为分叉的父节点（内部节点）。
