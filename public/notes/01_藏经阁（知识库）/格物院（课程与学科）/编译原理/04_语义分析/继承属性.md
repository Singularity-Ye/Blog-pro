# 继承属性：自顶向下流转的数据流

> English: **Inherited Attribute**

**继承属性**是指根据语法分析树中**父节点**或当前节点的**兄弟节点**（尤其是左侧兄弟）的属性值，计算并赋予给当前节点的文法属性。其核心特征是信息流在语法树中**自顶向下（Top-Down）**或**自左向右（Left-to-Right）**传播。

---

## 1. 🌟 大白话通俗解释 (核心直觉)

> [!NOTE]
> **长辈分发传家宝与长幼有序的比喻**：
> 如果说[[综合属性]]是“白手起家、自食其力”，全靠下层的子节点众筹汇报；那么**继承属性**则是“依靠衣食父母与兄长长辈的恩赐”：
> *   **父传子（自顶向下）**：子节点自己无法决定自己的继承属性（比如“变量的数据类型”或“嵌套作用域的层数”），它必须乖乖等上方（父节点）的红头文件分发下来，继承父母的家产。
> *   **兄传弟（自左向右）**：弟弟（右侧符号）不能凭空出生，他需要从左侧的哥哥（左侧兄弟节点）那承接交接下来的工作或数据。
> *   **绝对禁止向右看**：在规则里，子节点**绝对不能**向右侧还没有生出来的弟弟妹妹（右侧兄弟节点）索要数据。这种“只向左看、不向右看”的长幼尊卑规则，保证了我们在从左到右深度优先遍历（DFS）语法树时，当前节点所依赖的所有属性早已被计算完毕，绝不会卡死。

*   **一句话总结**：子节点（儿子）的属性值，全靠父节点（父母）或左兄弟（哥哥）分发赐予。

## 2. 📝 学术规范定义 (考试硬核)

### 形式化定义
对于文法产生式：
$$p: X_0 \to X_1 X_2 \dots X_n$$

右部某个符号 $X_i$（其中 $1 \le i \le n$）的属性 $b$ 是由如下语义方程定义的：
$$X_i.b = f(X_0.a, X_1.c, \dots, X_{i-1}.d)$$
*(其中 $X_0$ 是父节点，$X_1 \dots X_{i-1}$ 是 $X_i$ 左侧的兄弟节点，而 $f$ 是计算函数)*，则称属性 $b$ 是符号 $X_i$ 的一个**继承属性**。

### 信息流向
$$父节点属性/左侧兄弟属性 \longrightarrow 当前节点属性 \quad (自顶向下 或 自左向右)$$

### 典型例子：声明类型的传播
在变量声明文法中类型信息的横向和向下传递：
```text
decl → type var-list
  var-list.dtype = type.dtype          ← 父 type → 子 var-list (继承)

var-list₁ → id , var-list₂
  id.dtype = var-list₁.dtype           ← 父 → 子 (继承)
  var-list₂.dtype = var-list₁.dtype    ← 左侧兄弟 → 右侧兄弟 (继承)
```

对于声明语句 `float x, y`，其属性继承流动树如下：
```
       type.dtype = real (叶子节点提供类型源)
           │
           ▼ (1. 父子继承)
       var-list.dtype = real
        /      \
       /        ▼ (2. 兄弟横向继承)
   x.dtype    var-list₂.dtype = real
    = real        │
                  ▼ (3. 父子继承)
              y.dtype = real
```

## 3. 🎯 应试痛点与解题模板 (拿分关键)

### 判别秘籍
*   **判别式**：检查语义方程的等式左侧。若形如 `X_i.b = ...` 且 `X_i` 位于产生式的**右部**，则 `b` 必定是继承属性。

### S-属性与继承属性的冲突
*   **痛点**：继承属性在自底向上的 LR 语法分析器中极其难缠。因为 LR 归约是自底向上的，在分析子树 $X_i$ 时，父节点 $X_0$ 甚至还没有被识别出来，更别提从父节点往下传属性了。
*   **解题对策**：如果文法中包含了继承属性，那么它绝对不属于 [[S-属性文法]]。为了能够单遍分析，必须满足 [[L-属性文法]] 的约束，并利用虚产生式标记进行值栈寻址转换。

### ⚔️ 综合属性 vs 继承属性对比表

| 维度 | [[综合属性]] (Synthesized) | 继承属性 (Inherited) |
| :---: | :--- | :--- |
| **求值流动方向** | 自底向上（子节点 $\to$ 父节点） | 自顶向下/自左向右（父/兄 $\to$ 子） |
| **信息物理来源** | 子树节点的属性 | 祖先节点或左侧兄弟节点的属性 |
| **典型应用场景** | 算术求值、表达式类型推导 | 变量声明类型分发、嵌套作用域深度 |
| **适合的分析器** | 自底向上分析器 (LR) | 自顶向下分析器 (LL) 或带属性栈的 LR |
