# 静态语义：在生成代码前做最后的逻辑质检

> English: **Static Semantics**

**静态语义**是指程序在**编译期间**（即不运行程序的情况下）即可由编译器进行检查的语义规则。它是[[自底向上语法分析|语法分析]]和运行时行为（动态语义）之间的过渡桥梁。

---

## 1. 🌟 大白话通俗解释 (核心直觉)

> [!NOTE]
> **出厂前静态安检与行驶中动态车祸的比喻**：
> *   **静态语义（编译期安检）**：就像一辆新车出厂前，在静止的车库里做的“静态大检查”。安检员在不发动引擎的情况下，检查汽车的基本安全合规：车牌照挂了没有（变量是否先声明后使用）、加的油是否匹配发动机类型（赋值与运算的类型是否相容）。如果发现没挂牌照或者柴油车加了汽油，直接扣留不准出厂（编译报错，拒绝生成目标代码）。
> *   **动态语义（运行期车祸）**：则是汽车开上高速公路（程序运行）后才可能发生的“车祸”。比如车子开着开着突然没油了（内存溢出崩溃）、在湿滑路面滑行打转（除数为零）、或者偏离公路撞到了防护栏（数组越界、空指针解引用）。静态安检无法在车库里完全预防这些运行时异常，但至少能保证连牌照都没挂、发动机类型错配的车根本无法上路运行。

*   **一句话总结**：编译期能查出来的低级逻辑错误叫静态语义；必须跑起来才能爆发的车祸叫动态语义。

## 2. 📝 学术规范定义 (考试硬核)

### 程序语义的层次结构
程序的语义检查分为两大层次：

| 层次 | 检查时机 | 核心检查工具 | 典型示例 |
|---|---|---|---|
| **静态语义** | **编译时（Compile-time）** | [[属性文法]] + 符号表 + 类型系统 | `int x = "hello";` 类型不匹配，编译直接报错。 |
| **动态语义** | **运行时（Run-time）** | 运行时系统（虚拟机/CPU异常） | `a / b` 中 `b` 的值为 `0`，运行到此处发生除零崩溃。 |

### 静态语义分析的具体检查项目

| 检查类别 | 形式化判据 / 检查逻辑 | 依赖的编译信息 |
|---|---|---|
| **声明检查** | $\forall id \in \text{program}, \exists \text{declaration}(id) \wedge \text{decl} < \text{use}$ | 符号表（通过[[继承属性]]传递作用域环境） |
| **类型相容性** | $\text{binop}(expr_1, op, expr_2) \to expr_1.type$ 与 $expr_2.type$ 可合式 | 类型系统规则（通过[[综合属性]]自底向上推导） |
| **赋值相容性** | $expr.type \le var.type$ | 类型系统 + 声明的静态类型 |
| **函数调用** | 实参数量 = 形参数量 $\wedge\ \forall i: arg_i.type \le param_i.type$ | 函数签名表 |

### 静态语义的物理边界
静态语义分析无法感知运行时的动态输入，因此存在明确的检查边界：

| ❌ 无法静态检查（需运行时拦截） |  能够静态检查（编译期拦截） |
|---|---|
| **除零错误**（除数可能来自用户输入） | **类型错配**（如 `int` 与 `struct` 直接相加） |
| **数组越界**（下标在运行时动态计算） | **变量未声明**（使用未定义标识符） |
| **空指针解引用**（指针的值在运行时确定） | **函数参数数量或类型错误** |
| **动态内存分配失败**（OOM 崩溃） | **不可达代码**（部分可静态检查优化） |

## 3. 🎯 应试痛点与解题模板 (拿分关键)

### 常见题型一：辨析编译期错误 (Compile-time) 与运行时错误 (Run-time)
*   **考场例题**：指出以下哪项错误可以在编译时被检测出来：
    A. 数组越界  B. 变量未声明  C. 除数为 0  D. 动态内存耗尽
*   **答案与分析**：选 B。变量未声明属于声明检查，是典型的静态语义检查项；而 A、C、D 的值都取决于运行时的计算或输入，属于动态语义检查项。

### 常见题型二：静态语义信息在属性文法中的流向设计
*   **解题模板**：
    1.  **作用域与声明信息**：通常使用**继承属性**（如 `env`, `symbolTable`）自顶向下或自左向右在语法树中分发，供叶子节点在使用标识符时查找。
    2.  **类型检查与推导**：通常使用**综合属性**（如 `type`）自底向上一步步合成，最终在赋值节点或运算符节点处进行相容性判别。
