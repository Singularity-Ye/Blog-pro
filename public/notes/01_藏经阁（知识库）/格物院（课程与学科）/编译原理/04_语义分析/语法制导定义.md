# 语法制导定义：只画设计图纸的声明式语义规范

> English: **Syntax-Directed Definition** (简称 **SDD** )

**语法制导定义（SDD）** 是属性文法的一种声明式（Declarative）规范。它只写出文法符号属性计算的数学等式（等号 `=`），而不指定具体的计算时机、求值顺序或代码执行步骤。它是静态语义分析和中间代码生成的“装修设计图纸”。

---

## 1. 🌟 大白话通俗解释 (核心直觉)

> [!TIP]
> **装修设计效果图与施工指南的比喻**：
> 想象你买了一套毛坯房：
> *   **上下文无关文法（CFG）** 规定了房间的结构（哪里是墙，哪里是门）。
> *   **语法制导定义（SDD）** 就像一张 **精装房效果图/设计指标单**。它只写明要求：“客厅墙面面积 = 4面墙的面积总和”，“客厅墙面漆颜色 = 天蓝色”。设计图上**绝对不会**写怎么搅拌油漆、先刷哪面墙。这是一种**声明式**的设计，只看结果，不管过程。
> *   **语法制导翻译（SDT）** 则是一份 **硬核施工指南**。它会具体到命令：“第1步：买2桶漆；第2步：把油漆往墙上刷；第3步：等待晾干。”这是一种**命令式**的动作流。
> 
> SDD 的最大好处是干净、优雅。它不管具体的编译器是用自顶向下还是自底向上，只负责把属性之间的“数学等量关系”规定死。

*   **一句话总结**：SDD 只用“数学等式”规定属性怎么算，不夹带任何“副作用代码”和“执行时序”。

---

## 2. 📝 学术规范定义 (考试硬核)

### 形式化结构
语法制导定义（SDD）是在上下文无关文法（CFG）的每条产生式 $A \to \alpha$ 旁边，挂载一组**语义规则（Semantic Rules）**：
$$
\text{Rule: } b = f(c_1, c_2, \dots, c_k)
$$

其中，$b$ 是产生式中某个符号的属性，且：
1.  若 $b$ 是左部非终结符 $A$ 的属性，则 $b$ 必须是**[[综合属性]]**；
2.  若 $b$ 是右部某个文法符号 $X_i$ 的属性，则 $b$ 必须是**[[继承属性]]**。

### 🚨 绝对禁忌：语义规则是“等式”，不是“赋值语句”
在 SDD 中，语义规则是**数学等式**（Semantic Equations）。
*   **正确写法**：必须使用等号 **`=`**（如 `E.val = E1.val + T.val`）。
*   **错误写法**：严禁使用赋值号 **`:=`**、**`<-`** 或在等式中夹带程序指令（如 `print()`、`insert()`）。
*   *原因*：数学等式没有副作用且无序；赋值语句是有时序的。一旦夹带了副作用代码，SDD 就会退化为命令式的 **[[属性文法#⚔️ 核心对比：SDD 与 SDT|SDT]]**。

### SDD 属性分类一览
根据属性流动的方向，SDD 包含两类核心属性：

| 属性类型 | 信息流动方向 | 典型语义规则形式 | 物理直觉 |
| :--- | :--- | :--- | :--- |
| **[[综合属性]] (Synthesized)** | **自底向上** (从子到父) | $A.a = f(X_1.b, \dots, X_n.c)$ | 树叶的数值向上汇总，最终结出果实。 |
| **[[继承属性]] (Inherited)** | **自顶向下/横向** (父到子，或兄到弟) | $X_i.a = f(A.b, X_j.c)$ | 上层环境或左侧兄弟的上下文信息向右/向下渗透。 |

## 3. 🎯 应试痛点与解题模板 (拿分关键)

### 常见题型一：判断属性是综合还是继承
*   **秒杀法则**：盯着语义规则中等号 `=` **左侧**的那个属性挂在谁头上：
    *   若左侧挂在**产生式左部非终结符**（爸爸）头上，如：
        $$
        A \to B C \quad \{ A.val = B.val + C.val \}
        $$
        由于 $A$ 是左部符号，其属性 $A.val$ 必然是**综合属性**（儿子们合成给爸爸）。
    *   若左侧挂在**产生式右部文法符号**（儿子）头上，如：
        $$
        A \to B C \quad \{ B.inh = A.val \}
        $$
        由于 $B$ 是右部符号，其属性 $B.inh$ 必然是**继承属性**（爸爸传给儿子）。

### 常见题型二：写出将中缀表达式转换为后缀表达式的 SDD
*   **真题重现**：为表达式文法写出 SDD，将 `3 + 4 * 5` 输出为后缀式 `3 4 5 * +`。
*   **标准解答模板**：
    由于要进行字符串拼接，我们为文法符号引入一个综合属性 `code`（代表其后缀指令序列）。
    
    | 产生式 | 语义规则 (必须用等号 `=`) |
    | :--- | :--- |
    | $E \to E_1 + T$ | $E.code = E_1.code \mathbin{\Vert} T.code \mathbin{\Vert} \text{'+'}$ |
    | $E \to E_1 - T$ | $E.code = E_1.code \mathbin{\Vert} T.code \mathbin{\Vert} \text{'-'}$ |
    | $E \to T$ | $E.code = T.code$ |
    | $T \to T_1 * F$ | $T.code = T_1.code \mathbin{\Vert} F.code \mathbin{\Vert} \text{'*'}$ |
    | $T \to F$ | $T.code = F.code$ |
    | $F \to ( E )$ | $F.code = E.code$ |
    | $F \to \text{digit}$ | $F.code = \text{digit.lexval}$ |

    > *注：$\Vert$ 表示字符串拼接算子。*
