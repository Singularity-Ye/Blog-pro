# 建站故障排查清单

> 真实踩过的坑，不是教程猜的坑。

## 问题一：Pages 自定义域名 Active 但访问 404

**2026-05-21 亲历。**

### 症状

- Custom domains 显示 `Active` + `SSL enabled`
- 浏览器访问 → `HTTP ERROR 404`
- Pages 默认域名也 404

### 排查顺序

```
1. 自定义域名 404
2. → 访问 Pages 默认域名 (xxx.pages.dev)
3. → 默认域名也 404 → 问题不在 DNS，在 Pages 部署
4. → 查看 Build log
5. → 发现输出目录错误（CRA 是 build，不是 dist）
6. → 修正 Build output directory
7. → Retry deployment
```

### 根因

把 Create React App 的输出目录误写成 Vite 的 `dist`。

| 项目 | 正确输出目录 |
|------|-------------|
| CRA | `build` |
| Vite | `dist` |

### 修复

1. Settings → Build → Build output directory → 改为 `build`
2. Deployments → `...` → **Retry deployment**（必须手动重部署）

详见 → [[建站流程指南-静态网页/平台部署/Cloudflare Pages 404 排错记录]]

## 问题二：GitHub 更新后 Pages 不自动部署

**2026-05-21 亲历。完整复盘见 → [[建站流程指南-静态网页/平台部署/Cloudflare Pages 部署错误复盘]]**

### 症状

- GitHub push 了新 commit
- 点旧部署 Retry deployment → 网站没变
- Retry 的仍是旧 commit

### 根因

```
GitHub App 授权断开
  → Cloudflare 收不到 push 事件
  → 没有新 deployment
  → Retry 只重跑旧 commit
```

### 核心误区

**Retry deployment ≠ 拉取最新代码。** Retry 只重跑当前 deployment 绑定的旧 commit。

### 修复

1. Settings → Build → Git repository → Manage → 修复 GitHub App 权限
2. `git push` 触发新 deployment

## 问题三：Vercel 国内访问慢或无法访问

**可能原因**：Vercel 默认域名国内解析不稳定。

**解决思路**：
1. 换用自定义域名
2. 使用 Cloudflare Pages 替代
3. 使用国内云服务或对象存储

---

## 问题三：Cloudflare 出现 502

**可能原因**：源站不可达、SSL 模式错误、DNS 解析错误、握手失败。

**排查顺序**：
1. 检查源站能否直接访问
2. 检查 DNS 记录
3. 检查 SSL 模式（Flexible → Full）
4. 清除 Cloudflare 缓存
5. 暂停 Cloudflare 代理测试
6. 重新启用

## 问题五：只有某个浏览器能访问

1. 换浏览器 / 无痕模式
2. 清除浏览器缓存
3. `ipconfig /flushdns`
4. 清除 Cloudflare 缓存
5. 等待 DNS 传播
