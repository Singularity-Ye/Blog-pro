# 继承属性 (Inherited Attribute)

**继承属性**是由父节点或兄弟节点（左兄弟）的属性计算得到的属性。信息流从语法树的祖先或左侧向右侧/下方传播。

## 形式化定义

对于产生式 `p: X₀ → X₁ X₂ ... Xₙ`，符号 `Xᵢ`（i ≥ 1）的继承属性 b 由语义规则定义：

```
Xᵢ.b = f( X₀.a, X₁.c, ..., Xᵢ₋₁.m )
```

- 函数的参数只能来自：父节点 `X₀` 的属性，或**左侧兄弟** `X₁...Xᵢ₋₁` 的属性
- 不能依赖右侧兄弟 `Xᵢ₊₁...Xₙ` 的属性

## 信息流方向

```
父节点/左兄弟属性 → 当前节点属性
  (自顶向下 或 自左向右)
```

## 典型例子

### 声明类型的传播

```
decl → type var-list
  var-list.dtype = type.dtype          ← 父 type → 子 var-list

var-list₁ → id , var-list₂
  id.dtype = var-list₁.dtype           ← 父 → 子
  var-list₂.dtype = var-list₁.dtype    ← 父 → 兄弟
```

`float x, y`：

```
type.dtype = real
    ↓ 继承
var-list.dtype = real
    ↓ 继承           ↓ 继承
x.dtype = real    y.dtype = real
```

### 进制传播

```
based-num → num basechar
  num.base = basechar.base             ← 兄弟(右) basechar → 兄弟(左) num
```

这里 `num` 的 `base` 从右侧符号 `basechar` 获得——但需要注意这与 L-属性文法的"只能从左和父获取"约束有出入。这个例子实际上需要更复杂的文法重构才能成为严格的 L-属性文法。

## 与综合属性的区别

| | [[综合属性]] | 继承属性 |
|---|:--:|:--:|
| 流向 | 子 → 父 | 父/兄 → 子 |
| 信息源 | 子节点 | 祖先或左兄弟 |
| 典型场景 | 表达式值、类型 | 声明类型传播、作用域 |
| 适合的分析器 | LR 自底向上 | LL 自顶向下（或带属性栈的 LR） |
