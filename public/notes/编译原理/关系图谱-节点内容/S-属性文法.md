# S-属性文法 (S-Attributed Definition)

**S-属性文法**是仅有[[综合属性]]、不含[[继承属性]]的属性文法。它是属性文法中最简单的一类，且天然兼容自底向上的 LR 语法分析。

## 形式化定义

一个属性文法是 S-属性的，当且仅当：

```
所有属性都是综合属性（synthesized）
没有继承属性（inherited）出现在任何文法符号上
```

等价条件：所有语义规则的形式为 `X₀.a = f(子节点属性)`，不存在 `Xᵢ.b = f(父/兄弟属性)` 的规则。

## 求值策略

因为所有属性都是综合的，依赖边全部指向祖先方向。S-属性文法的属性可以在**自底向上的一次遍历**中完成：

```
LR 分析器：
  每次归约（reduce）时，执行产生式的语义规则
  因为子节点的综合属性已经计算完成
  可以直接算父节点的综合属性
```

这就是 Yacc 的工作方式——`$$ = $1 + $3` 在归约时执行，所有 `$n` 已经就绪。

## 典型例子

### 无符号整数

```
number₁ → number₂ digit      number₁.val = number₂.val × 10 + digit.val
number → digit                number.val = digit.val
```

全部是综合属性，自底向上计算。

### 算术表达式

```
exp₁ → exp₂ + term           exp₁.val = exp₂.val + term.val
exp → term                    exp.val = term.val
```

同样全综合。`(34 - 3) * 42` 在 LR 归约时同步算出 `1302`。

## 与 L-属性文法的关系

```
S-属性文法  ⊂  L-属性文法
```

任何 S-属性文法也是 L-属性文法（继承属性集合为空的退化情况）。但 L-属性文法允许继承属性，因此表达能力更强。
