# 属性文法 (Attribute Grammar)

**属性文法**是为上下文无关文法（CFG）的每个符号和产生式附加**语义规则**的数学框架。它是语法树到语义信息的桥梁。

## 形式化定义

一个**属性文法 AG** 是一个三元组：

```
AG = (G, A, R)
```

| 组件 | 定义 |
|------|------|
| **G** | 底层上下文无关文法（CFG），定义程序的句法结构 |
| **A** | 属性集合：对每个文法符号 X，定义 `A(X) = A_syn(X) ∪ A_inh(X)`，分别是该符号的[[综合属性]]集和[[继承属性]]集。初始符号 S 的 `A_inh(S)` 包含预定义的初始值 |
| **R** | 语义规则集合：对每条产生式 `p: X₀ → X₁ ... Xₙ`，为 A(X₀) 中的每个综合属性和 A(Xᵢ)（i ≥ 1）中的每个继承属性提供一条计算方程 |

## 语义规则的形式

对于产生式 `p`，语义规则有两种形式：

```
综合属性方程：X₀.a = f( X₁.b, X₂.c, ... , Xₙ.m )
继承属性方程：Xᵢ.b = f( X₀.a, X₁.c, ... , Xᵢ₋₁.m )    (i ≥ 1)
```

每条方程的右侧可以是属性值、常量，或它们的任意计算函数。

## 与 CFG 的区别

| | 普通 CFG | 属性文法 |
|---|---------|---------|
| 描述内容 | 句法结构（产生式的形状） | 句法结构 + 语义计算 |
| 产生式右部 | 仅符号序列 | 符号序列 + 属性计算规则 |
| 分析输出 | 语法树（仅结构） | 带注解的语法树（结构 + 属性值） |

## 分类

根据属性类型和依赖方向，属性文法分为：

| 类别 | 允许的属性 | 依赖约束 | 求值策略 | 详细笔记 |
|------|:--:|------|------|------|
| **S-属性文法** | 仅综合属性 | 无继承属性 | LR 自底向上同步求值 | [[S-属性文法]] |
| **L-属性文法** | 综合 + 继承 | 继承仅依赖父/左兄弟 | 深度优先左→右遍历 | [[L-属性文法]] |
| **一般属性文法** | 任意 | 可能形成依赖环 | 需依赖图拓扑排序 | [[依赖图与属性计算]] |

## 属性计算的通用框架

```
1. 语法分析器产出语法树
2. 对语法树每个节点展开属性实例
3. 对每条产生式的语义规则实例化
4. 构造依赖图 → 拓扑排序 → 按序求值
5. 得到完全注解的语法树
```
