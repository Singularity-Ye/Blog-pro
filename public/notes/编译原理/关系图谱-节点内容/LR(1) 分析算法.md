# LR(1) 分析算法

**LR(1)** 是最精确的 LR 分析算法。每个项目自带一个**前看符号（lookahead）**，表示"只有当下一个输入符号是它时才允许归约"。这使其能够处理几乎所有实际编程语言的文法，代价是状态数可能很大。

## 形式化定义

### LR(1) 项目

```
[A → α·β, a]
```

- `A → α·β` 是 LR(0) 核心：圆点左边已识别，右边待识别
- `a` 是前看符号（lookahead），含义：**当将来用这个项目完成归约（圆点到最右）时，下一个输入符号必须是 a**

> `a` 不是指导圆点后面如何解析的——圆点右边仍然按 CFG 文法匹配。`a` 只在**归约发生的那一刻**才出场。

### LR(1) 的 CLOSURE 运算

```
CLOSURE(I)：
  repeat
    for each [A → α·Bβ, a] in I：      —— 圆点后是非终结符 B
      for each B → γ in 文法：
        for each b in FIRST(βa)：        —— 关键：前看符号在这里传播！
          把 [B → ·γ, b] 加入 I
  until I 不再变化
```

**`FIRST(βa)` 的含义**：B 解析完成后，parser 面对的第一个符号来自哪里？

- 如果 β 非空：来自 `FIRST(β)`（B 后面的符号序列的开头）
- 如果 β 可以为空（β ⇒* ε）：来自 `FIRST(β)` ∪ {a}（可能直接跳到原来的 lookahead）

所以 B 的项目的 lookahead = `FIRST(βa)`，不是简单照搬原来的 a。

### LR(1) 的 GOTO 运算

```
GOTO(I, X) = CLOSURE( { [A → αX·β, a] | [A → α·Xβ, a] ∈ I } )
```

GOTO 只移动圆点，**不改变 lookahead**。lookahead 仅通过 CLOSURE 传播。

## ACTION 和 GOTO 表的构造

对状态 Iᵢ 中的每个项目：

| 条件 | 表条目 |
|------|--------|
| `[A → α·aβ, b] ∈ Iᵢ`，且 `GOTO(Iᵢ, a) = Iⱼ` | `ACTION[i, a] = shift j` |
| `[A → α·, a] ∈ Iᵢ` | `ACTION[i, a] = reduce A→α`（只对 lookahead a 归约！） |
| `[S' → S·, $] ∈ Iᵢ` | `ACTION[i, $] = accept` |
| `GOTO(Iᵢ, A) = Iⱼ`（A 是非终结符） | `GOTO[i, A] = j` |

> **与 LR(0)/SLR(1) 的关键区别**：每个归约项目有自己的 lookahead。同一产生式 `A → α` 在不同状态中可以带不同的前看符号，从而只在**上下文恰当**的输入下触发归约。

## 为什么 LR(1) 状态数膨胀

同一个产生式 `B → γ`，在 LR(0) 里只有一个初始项目 `B → ·γ`。但在 LR(1) 里，由于 `FIRST(βa)` 随 `β` 变化，`B → ·γ` 会携带**不同的 lookahead** 进入不同的 CLOSURE 实例。核心相同的项目集因 lookahead 不同而分裂为多个状态。典型膨胀比为 5-15 倍。

## 例题

对于文法片段：

```
S → A a | B b
A → c
B → c
```

LR(0) 将 `A → c` 和 `B → c` 视为同一核心。但 LR(1) 的前看符号不同：

| 项目 | 前看来源 |
|------|------|
| `[A → ·c, a]` | 来自 `S → ·A a` 的 `FIRST(a a-here)` = `{a}` |
| `[B → ·c, b]` | 来自 `S → ·B b` 的 `FIRST(b b-here)` = `{b}` |

LR(1) 能区分 `c` 后面应跟 `a`（归约到 A）还是 `b`（归约到 B）。LR(0) 和 SLR(1) 无法区分。

## 与 SLR(1) 的精确对比

| | SLR(1) | LR(1) |
|---|--------|-------|
| 前看来源 | 全局 `FOLLOW(A)` | 项目自带，从 `FIRST(βa)` 传播 |
| 精确度 | `FOLLOW(A)` 可能比实际上下文宽 | lookahead 精确到状态内的项目 |
| 状态数 | = LR(0) | 显著膨胀 |
| 归约条件 | `a ∈ FOLLOW(A)` | `a == 项目自己的 lookahead` |
