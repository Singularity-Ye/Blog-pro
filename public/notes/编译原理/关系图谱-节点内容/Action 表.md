# Action 表

**Action 表**是自底向上语法分析器（如 LR 分析器）的核心控制大脑之一，通常与 [[Goto 表]] 配合使用。它决定了分析器在特定状态下面对特定字符时，应该采取什么动作。

## 核心构成

Action 表是一个二维数组：
- **行（Row）**：代表 [[DFA]] 的当前**状态**（如 $I_0, I_1, \dots$）。
- **列（Column）**：代表下一个输入的**终结符**（Terminal，包括结束符 `$`）。

### 示例结构

```
状态 |  id   |   +   |   *   |   $   |
-----|-------|-------|-------|-------|
 I0  |  s5   |       |       |       |
 I1  |       |  s6   |       |  acc  |
 I2  |       |  r2   |  s7   |  r2   |
```

> `s5` = 移进并跳转到状态 5，`r2` = 按第 2 条规则归约，`acc` = 接受

## 四种基本动作

1. **移进 (Shift, 缩写 `s`)**：例如 `s5`，表示将当前输入字符压入栈，并跳转到状态 5。关联概念：[[移进]]。
2. **归约 (Reduce, 缩写 `r`)**：例如 `r2`，表示根据第 2 条文法规则，将栈顶的几个字符合并替换为一个非终结符。关联概念：[[归约]]。
3. **接受 (Accept, 缩写 `acc`)**：表示整个代码解析成功。通常需要用到 [[增广文法]]。
4. **报错 (Error)**：表中的空白处，表示语法错误。

## 构建过程

Action 表的构建基于 [[DFA]] 的边（[[跳转函数]]）：
- 如果存在跳转 $GO(I, a) = I_j$ 且 $a$ 是终结符 → `ACTION[I, a] = s j`
- 如果状态 $I$ 包含归约项目 $A \to \alpha \cdot$ → `ACTION[I, a] = r k`（$k$ 是规则编号）
- 如果状态 $I$ 包含 $S' \to S \cdot$ → `ACTION[I, $] = acc`

## 在不同算法中的差异

| 算法 | 归约策略 | 冲突风险 | 表大小 |
|------|---------|---------|--------|
| **LR(0)** | 闭眼归约。状态内出现完整项目就在整行填入 Reduce | 高——大量 [[移进-归约冲突]] | 状态数中等 |
| **[[SLR(1)]]** | 谨慎归约。只有当下一个字符属于该非终结符的 [[Follow 集合]] 时，才填入 Reduce | 中等——解决部分冲突 | 与 LR(0) 相同 |
| **LR(1)** | 精确归约。每个项目携带向前看符号，大幅减少冲突 | 低——几乎无冲突 | 状态数可能膨胀数倍 |

## 与 Goto 表的分工

| 维度 | Action 表 | Goto 表 |
|------|----------|---------|
| 输入符号类型 | 终结符（Token） | 非终结符（语法成分） |
| 触发时机 | 读入新字符时 | 归约操作完成后 |
| 输出动作 | s / r / acc / err | 下一个状态编号 |
