# 自底向上语法分析 (Bottom-Up Parsing)

**自底向上语法分析**是从输入符号串（Token 流）逐步归约到文法起始符号的分析方法，也是当前大部分编译器（如 YACC、Bison、GCC）采用的解析技术。它比自顶向下分析（如 LL、递归下降）能处理更广泛的文法，包括左递归。

## 核心思想：归约

与自顶向下（从起始符号出发推导 Token 串）相反，自底向上分析**从 Token 串出发"反向推导"**：
- 先读入具体的 Token（如 `id`、`+`、`*`）
- 每当匹配到某条文法规则的右部，就将其**归约**（Reduce）为该规则的左部非终结符
- 最终归约到唯一的起始符号 $S$，则分析成功

```
  自顶向下（预测推导）             自底向上（归约）
  S → ...                          id + id * id
  ↓ 展开                          ↓ 归约
  ... → id + id * id              E + E * E
                                  ↓ 归约
                                  E + E
                                  ↓ 归约
                                  E（成功！）
```

## LL 与 LR：两种范式的根本区别

| 维度 | 自顶向下 (LL) | 自底向上 (LR) |
|------|-------------|-------------|
| 推导方向 | 从起始符号→Token | 从 Token→起始符号 |
| 操作方式 | **展开**（Expand）非终结符 | **归约**（Reduce）为非终结符 |
| 数据结构 | 递归调用栈 | 显式分析栈 + 状态栈 |
| 查表方式 | LL 分析表（预测表） | Action+Goto 表 |
| 左递归 | ❌ 不支持 | ✅ 天然支持 |
| 适用文法范围 | 较窄（LL(k) 文法） | 较宽（LR(k) 文法） |
| 错误检测 | 较早 | 稍晚但更精准 |

## 一个完整的自底向上分析流程

以 `id + id` 为例，使用 LR 分析器：

```
步骤 |   栈（状态+符号）     | 输入剩余  | 动作
─────┼─────────────────────┼──────────┼──────
  1  |  0                  | id + id $ | 移进
  2  |  0 id 5             |   + id $ | 按 T→id 归约
  3  |  0 T 3              |   + id $ | 按 E→T 归约
  4  |  0 E 1              |   + id $ | 移进
  5  |  0 E 1 + 6          |     id $ | 移进
  6  |  0 E 1 + 6 id 5     |       $ | 按 T→id 归约
  7  |  0 E 1 + 6 T 8      |       $ | 按 E→E+T 归约
  8  |  0 E 1              |       $ | 接受（acc）
```

该流程完整依赖本图谱中所有节点的协同——[[LR(0) 项目]] 构建状态、[[跳转函数]] 连接状态、[[Action 表]] 指导动作、[[Goto 表]] 完成归约跳转、[[增广文法]] 标记终点。
