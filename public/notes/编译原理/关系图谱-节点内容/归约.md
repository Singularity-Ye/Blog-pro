# 归约 (Reduce)

**归约**是语法分析器最重要的"收网"动作——它是 [[移进]] 的逆操作，也是自底向上分析的灵魂所在。每次归约都意味着分析器识别出了一个完整的语法结构。

## 动作流程

1. **匹配成功**：当栈顶的几个符号刚好拼凑成了某条文法产生式的右部（称为一个**句柄** Handle）。
2. **弹栈**：把这些零碎的符号及其对应的状态从栈里弹出去（出栈），露出栈中的**历史状态**。
3. **替换**：将它们替换为产生式左部的大模块（非终结符）。
4. **跳转**：拿着这个新合成的非终结符，根据弹栈后露出的历史状态去查 [[Goto 表]]，决定下一步去哪个新状态，并将新状态压栈。

### 析栈操作可视化

```
文法规则：E → E + T
当前栈（栈顶在右侧）：
┌──────┬──────┬──────┬──────┬──────┬──────┐
│  I0   │  E   │  I1   │  +   │  I3   │  T   │ I2
└──────┴──────┴──────┴──────┴──────┴──────┘

步骤 1：检测到栈顶 E + T 匹配 E → E + T 的右部
步骤 2：弹出 T、I2、+、I3、E、I1 → 露出历史状态 I0
         ┌──────┐
         │  I0   │  ← 历史状态
         └──────┘
步骤 3：拿着新合成的 E，查 Goto[I0, E] = I1
步骤 4：将 E 和 I1 压栈
         ┌──────┬──────┐
         │  I0   │  E   │ I1
         └──────┴──────┘
```

## 归约的触发条件

归约在不同算法中有不同的触发策略：

| 算法 | 触发条件 | 特点 |
|------|---------|------|
| **LR(0)** | 状态中有任何完整项目 $A \to \alpha \cdot$ | 不看输入，直接归约——容易冲突 |
| **[[SLR(1)]]** | 完整项目 $A \to \alpha \cdot$ **且**当前输入字符 $\in Follow(A)$ | 加入 [[Follow 集合]] 过滤，更安全 |
| **LR(1)** | 完整项目 $A \to \alpha \cdot,\, a$ **且**当前输入字符等于项目的向前看符号 $a$ | 最精确，冲突最少 |

## 归约与"句柄"的关系

**句柄（Handle）** 是归约操作的核心概念：
- 句柄是栈顶的一段符号序列，恰好匹配某条产生式的右部。
- 归约操作的本质就是**识别句柄并将其替换为非终结符**。
- 自底向上分析也被称为"句柄识别"过程。

## 归约的完成标志

当归约触发 $S' \to S$（[[增广文法]] 的接受规则）时，意味着整个输入串已被成功归约为起始符号——这就是 **Accept（接受）** 状态，标志着语法分析圆满结束。
