# 子集构造法 (Subset Construction)

**子集构造法**（也称"幂集构造"）是将 [[NFA]] 转化为等价 [[DFA]] 的标准算法。在 LR 语法分析中，它用来将 LR(0) 项目 NFA 转换为 LR(0) DFA，由此生成 [[Action 表]] 和 [[Goto 表]]。

## 核心思想

NFA 的"非确定性"体现在：同一状态 + 同一输入符号可能对应**多个**后继状态。子集构造法的思路是——把 NFA 状态的**集合**打包为 DFA 的**一个**状态。

$$DFA\text{ 的一个状态} = NFA\text{ 的一组状态的集合}$$

## 算法描述

```
输入：NFA（状态集 Q_N、字母表 Σ、转移函数 δ_N、起始状态 q₀、终态集 F_N）
输出：DFA（状态集 Q_D、转移函数 δ_D、起始状态 q₀'、终态集 F_D）

1. q₀' = ε-closure({q₀})      ← 起始状态：NFA 起始状态的 ε 闭包
2. Q_D = {q₀'}                 ← DFA 状态集初始只含起始状态
3. 将 q₀' 标记为"未处理"
4. while 存在未处理的 DFA 状态 S ∈ Q_D:
5.   将 S 标记为"已处理"
6.   for each 输入符号 a ∈ Σ:
7.     T = ε-closure(∪_{q∈S} δ_N(q, a))   ← 核心转移
8.     if T ∉ Q_D:
9.       Q_D = Q_D ∪ {T}                  ← 发现新 DFA 状态
10.      δ_D(S, a) = T
11. F_D = { S ∈ Q_D | S ∩ F_N ≠ ∅ }      ← DFA 终态

其中 ε-closure(P) = 从 P 中任意状态通过 0 条或多条 ε 边可达的状态集合。
```

## 在 LR 语法分析中的具体应用

在 LR 分析器的构建中，子集构造法的各概念映射如下：

| NFA 中的概念 | LR 分析中的对应物 |
|-------------|-----------------|
| NFA 状态 | 单个 [[LR(0) 项目]] |
| $\varepsilon$ 边 | 从 $A \to \alpha \cdot B \beta$ 到 $B \to \cdot \gamma$ 的推导展开 |
| $\varepsilon$-closure({项目集}) | [[闭包运算]] $closure(I)$ |
| 消耗符号 $X$ 的转移 | [[跳转函数]] $GO(I, X)$ |
| DFA 状态 | LR(0) 项目集（完整状态） |
| DFA 的转移边 | 跳转函数的结果 |

### 关键洞察

在 LR 分析的特殊场景下，子集构造法被特化为两个紧密耦合的函数：

$$closure(I) \quad\longleftrightarrow\quad GO(I, X)$$

- $closure(I)$ 对应 NFA 的 $\varepsilon$-closure
- $GO(I, X)$ 对应 NFA 中消耗符号 $X$ 的转移 + 对新目标做 closure

### 算法：为 LR 文法构建 DFA

```
1. 起始 DFA 状态：
   I₀ = closure({S' → ·S})

2. 对每个新状态 Iᵢ、每个符号 X（终结符或非终结符）：
   Iⱼ = closure({A → αX·β | A → α·Xβ ∈ Iᵢ})
   if Iⱼ ≠ ∅ and Iⱼ is new:
     添加 Iⱼ 到 DFA 状态集
     添加转移 δ(Iᵢ, X) = Iⱼ
```

这正是子集构造法在 LR 领域的标准应用形式。

## 示例

以 LR(0) NFA → DFA 为例：

### NFA（片段）

```
E' → ·E ──E──→ E' → E·
  │               │
  │ ε              │ ε
  ↓               ↓
E → ·E+T ──E──→ E → E·+T ──+──→ E → E+·T
  │                               │
  │ ε                              │ ε
  ↓                               ↓
T → ·id ──id──→ T → id·          T → ·id
```

### 对应的 DFA（子集构造法结果）

```
I₀ = closure({E'→·E}) = {
  E'→·E, E→·E+T, E→·T, T→·id }

I₁ = GO(I₀, E) = closure({E'→E·, E→E·+T}) = {
  E'→E·, E→E·+T }

I₂ = GO(I₀, T) = closure({E→T·}) = {
  E→T· }

I₃ = GO(I₀, id) = closure({T→id·}) = {
  T→id· }

I₄ = GO(I₁, +) = closure({E→E+·T}) = {
  E→E+·T, T→·id }
...
```

每个 DFA 状态 $I_n$ 都是 NFA 状态的一个**子集**——这就是"子集构造"名称的由来。

## 性质与复杂度

- **最坏情况**：如果 NFA 有 $n$ 个状态，DFA 最多可能有 $2^n$ 个状态（指数爆炸）。
- **LR 现实情况**：在 LR(0) 构造中，DFA 状态数通常与文法规模呈**线性或近线性**关系，不会触发指数爆炸。
- **终止保证**：算法一定终止，因为 NFA 状态集是有限的，其幂集也有限。
- **语义等价**：构造出的 DFA 与原始 NFA 识别的语言完全相同。
