# DFA (确定性有限状态自动机)

在[[自底向上语法分析]]中，**DFA** 是一张指引程序如何解析代码的"地图"。它是整个 LR 分析体系的核心数据结构。

## 特点

- **节点（状态）**：DFA 的每一个节点都是一个**项目集**（包含了多个 [[LR(0) 项目]]）。
- **边（转换）**：节点之间的连线由 [[跳转函数]] $GO(I, X)$ 决定。
- **确定性**体现在：面对任何一个当前状态和输入字符，DFA 都能给出**唯一确定**的下一步走向，不会产生模棱两可的选择。

## DFA 的构建算法

```
1. 从增广文法的初始项目出发
   I₀ = closure({S' → ·S})

2. 对于每个尚未处理的状态 Iᵢ：
   对于每个符号 X（终结符或非终结符）：
     Iⱼ = GO(Iᵢ, X)
     如果 Iⱼ ≠ ∅ 且 Iⱼ 是新的状态 → 添加为新的状态节点
     添加从 Iᵢ 到 Iⱼ 的边，标记为 X

3. 重复直到所有状态都被处理
```

### 示例 DFA 构建

文法：
```
E → E + T | T
T → id
```

完整的 LR(0) DFA：
```
┌─────────────────────────────────┐
│         I₀                       │
│  E' → ·E    E → ·T              │
│  E  → ·E+T  T → ·id             │
└────────┬────────────────────────┘
     E   │      T          id
     ▼   │      ▼           ▼
┌──────────┐ ┌────────┐ ┌────────┐
│   I₁     │ │  I₂    │ │  I₃    │
│ E'→E·    │ │ E→T·   │ │ T→id·  │
│ E →E·+T  │ └────────┘ └────────┘
└─────┬────┘
   +  │
      ▼
  ┌──────────┐
  │   I₄     │
  │ E→E+·T   │
  │ T→·id    │
  └─────┬────┘
    T   │    id
    ▼   │     ▼
┌──────────┐ ┌────────┐
│   I₅     │ │  I₆    │
│ E→E+T·   │ │ T→id·  │
└──────────┘ └────────┘
```

## DFA 与 LR 分析表的关系

DFA 是生成 [[Action 表]] 和 [[Goto 表]] 的直接数据来源：

- DFA 的每个状态 → 分析表的一行
- DFA 的每条标记为**终结符**的边 → Action 表中的 `s` 条目
- DFA 的每条标记为**非终结符**的边 → Goto 表中的条目
- DFA 中包含**归约项目**的状态 → Action 表中的 `r` 条目
- DFA 中包含 $S' \to S \cdot$ 的状态 → Action 表中 `acc` 条目

DFA 是通过 **[[子集构造法]]** 从 [[NFA]] 转换而来的。这个过程将 NFA 中因 $\varepsilon$ 边产生的非确定性全部消解，合并为一个确定性的状态机。
