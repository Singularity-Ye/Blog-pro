# 项目集规范族 (Canonical LR Collection)

**项目集规范族**是从初始项目出发，通过反复应用 [[闭包运算|CLOSURE]] 和 GOTO 运算构造出的所有 LR 状态的集合。它等价于识别所有活前缀的 [[DFA]]。

## 形式化定义

给定一个上下文无关文法 G，其**LR(0) 项目集规范族**（不区分 LR(0)/SLR(1)/LR(1)，仅核心概念）按以下算法构造：

```
输入：增广文法 G' = G ∪ { S' → S }
输出：LR(0) 项目集规范族 C = { I₀, I₁, ..., Iₙ }

1. I₀ = CLOSURE({ S' → ·S })
2. C = { I₀ }
3. repeat
     for each I ∈ C 和每个文法符号 X（终结符或非终结符）：
       J = GOTO(I, X)
       if J ≠ ∅ 且 J ∉ C：
         C = C ∪ { J }
   until C 不再增大
4. return C
```

其中 GOTO(I, X) = CLOSURE( { A → αX·β | A → α·Xβ ∈ I } )。

## 每个状态的含义

规范族中的每个状态 I 对应一个**活前缀的有效项目集**：

- 状态 I 中的项目 `A → α·β` 表示"当前我们已经读取了 α，接下来期望读到 β"
- 状态 = DFA 的一个节点，代表 parser 在分析过程中可能停留的一个"位置"
- 从初始状态 I₀ 出发，沿符号 X₁X₂...Xₖ 到达的状态 Iₖ，恰好代表已经读入了前缀 X₁X₂...Xₖ

## LR(0) vs SLR(1) vs LR(1) 的规范族差异

| 方法 | 规范族的状态来源 | 状态的内部结构 | 状态数量 |
|------|--|--|:--:|
| **LR(0)** | LR(0) 项目 + CLOSURE(无 lookahead) | `A → α·β` | 最少 |
| **SLR(1)** | 完全同 LR(0) | `A → α·β`（同 LR(0)） | 同 LR(0) |
| **LR(1)** | LR(1) 项目 + CLOSURE(FIRST(βa)) | `[A → α·β, a]` | 最多 |
| **LALR(1)** | LR(1) 核心相同者合并 | `[A → α·β, a]` 合并后 | 同 LR(0) |

## 活前缀

**活前缀**是规范句型的一个前缀，它不超过句柄的右端。规范族中的每个状态恰好对应一个活前缀的集合——这些活前缀在 DFA 中到达同一个状态。
