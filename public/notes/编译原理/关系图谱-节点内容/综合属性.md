# 综合属性 (Synthesized Attribute)

**综合属性**是由子节点属性计算得到的属性。信息流从语法树的叶子向根方向传播。

## 形式化定义

对于产生式 `p: X₀ → X₁ X₂ ... Xₙ`，`X₀` 的综合属性 a 由语义规则定义：

```
X₀.a = f( X₁.b, X₂.c, ..., Xₙ.m )
```

- `X₀` 是产生式左部（父节点）
- `X₁...Xₙ` 是右部符号（子节点）
- `f` 是纯值计算函数

## 信息流方向

```
子节点属性 → 父节点属性
  (自底向上)
```

## 典型例子

### 数值计算

```
number₁ → number₂ digit
  number₁.val = number₂.val × 10 + digit.val
```

`digit.val` 和 `number₂.val`（子）→ `number₁.val`（父）。

### 表达式求值

```
exp₁ → exp₂ + term
  exp₁.val = exp₂.val + term.val
```

`exp₂.val` 和 `term.val`（子）→ `exp₁.val`（父）。

### 类型推导

```
exp₁ → exp₂ + term
  exp₁.type = if (exp₂.type == integer ∧ term.type == integer) then integer else error
```

## 在 LR 分析中的意义

综合属性天然适合 [[自底向上语法分析]]：LR 分析器在归约（reduce）时，子节点的属性值已经计算完毕，正好可以用来计算父节点的综合属性。Yacc 中的 `$$ = $1 + $3` 就是综合属性的工程实现。

## 与继承属性的区别

| | 综合属性 | [[继承属性]] |
|---|:--:|:--:|
| 流向 | 子 → 父 | 父/兄 → 子 |
| 信息源 | 子节点 | 祖先或左兄弟 |
| 适合的语法分析 | LR 自底向上（归约时计算） | LL 自顶向下或属性求值器 |
