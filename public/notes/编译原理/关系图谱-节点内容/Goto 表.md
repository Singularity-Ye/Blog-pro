# Goto 表 (状态转移表)

**Goto 表**是与 [[Action 表]] 并列的另一个二维数组。Action 表负责"吃进字符"，而 Goto 表专门负责"[[归约]] 后的善后工作"。

## 工作机制

当分析器完成一次归约（比如把栈里的 $x, y, z$ 合并成大模块 $A$），由于旧的零散零件被弹栈了，分析器退回到了一个历史状态。此时，分析器拿着新合成的非终结符 $A$，去查 Goto 表：
$$Goto[历史状态, 非终结符\;A] = \textbf{新状态}$$

### 完整的归约-跳转流程

```
步骤 1：归约前，栈顶有完整右部
┌──────┬──────┬──────┬──────┐
│  I3   │  x   │  I7   │  y   │    → 用规则 A → x y 归约
└──────┴──────┴──────┴──────┘

步骤 2：弹出右部符号及其状态，露出历史状态 I3
┌──────┐
│  I3   │    → 拿着新合成的非终结符 A，查 Goto[I3, A]
└──────┘

步骤 3：Goto[I3, A] = I9，将 A 和 I9 压栈
┌──────┬──────┐
│  I3   │  A   │ I9
└──────┴──────┘
```

## Goto 表与 Action 表的分工

| 维度 | Action 表 | Goto 表 |
|------|----------|---------|
| 列的类型 | **终结符**（Token） | **非终结符**（语法成分） |
| 何时查表 | 每次**读入新字符**时 | 每次**归约完成后** |
| 输出 | `s j`（移进）、`r k`（归约）、`acc`、空白（报错） | 下一个状态编号 $j$ |
| 值来源 | [[跳转函数]] $GO(I, a)$ | 跳转函数 $GO(I, A)$ |

## Goto 表的构建

Goto 表的构建完全基于 [[跳转函数]]：

$$GOTO[I, A] = 
\begin{cases}
j & \text{如果 } GO(I, A) = I_j \text{（$A$ 是非终结符）} \\
\text{空白} & \text{如果 } GO(I, A) = \emptyset
\end{cases}$$

### 构建示例

已知跳转函数：
- $GO(I_0, E) = I_1$ → $GOTO[I_0, E] = 1$
- $GO(I_0, T) = I_2$ → $GOTO[I_0, T] = 2$
- $GO(I_1, +) = I_3$ → 这不是 Goto 表的内容（`+` 是终结符，去 Action 表）

## 为什么需要回溯历史状态？

归约后查看历史状态而非直接跳转的原因是：**同一个非终结符在不同上下文中含义不同**。例如 $E$ 可以作为一个表达式单独出现，也可以作为 $T \to (E)$ 中括号内的表达式。从不同的历史状态出发，$Goto[I, E]$ 的结果不同。
