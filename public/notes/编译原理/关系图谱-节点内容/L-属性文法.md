# L-属性文法 (L-Attributed Definition)

**L-属性文法**是允许[[综合属性]]和[[继承属性]]共存的属性文法。继承属性的依赖来源被严格限制为：父节点或**左侧兄弟**节点。

## 形式化定义

一个属性文法是 L-属性的，当且仅当对于每条产生式 `p: X₀ → X₁ X₂ ... Xₙ`：

每个右部符号 `Xᵢ`（1 ≤ i ≤ n）的每个继承属性 `Xᵢ.b`，其语义规则

```
Xᵢ.b = f(属性集合)
```

中的参数只能来自：

1. `X₀`（父节点）的继承属性
2. `X₁, X₂, ..., Xᵢ₋₁`（**左侧兄弟**）的任何属性（综合或继承）

> 禁止依赖右侧兄弟 `Xᵢ₊₁...Xₙ` 的任何属性。

## 名称由来

"L" 表示 Left-to-right（从左到右）——属性依赖只能沿语法树从左向右传播，这与一次深度优先、左→右的遍历方向一致。

## 求值策略

L-属性文法可以通过**深度优先、左到右**的遍历在一次扫描中完成所有属性计算：

```
dfs(node):
  for child in node.children (从左到右):
    计算 child 的继承属性（来自 node 和已访问的 left siblings）
    dfs(child)
  计算 node 的综合属性（来自所有 children，它们已全部访问完毕）
```

## 典型例子

### 声明类型传播（继承属性从左向右）

```
decl → type var-list
  var-list.dtype = type.dtype                ← 父 → 子

var-list₁ → id , var-list₂
  id.dtype = var-list₁.dtype                 ← 父 → 子
  var-list₂.dtype = var-list₁.dtype          ← 从左向右沿兄弟传播
```

这里 `var-list₂.dtype` 依赖于左边兄弟 `var-list₁.dtype`——合法的 L-属性依赖。

### 反例：不属于 L-属性文法

```
A → B C
  B.x = C.y          ← B 的继承属性依赖右侧兄弟 C！
```

这**不**是 L-属性文法，因为 `B.x` 依赖于它右边的兄弟 `C.y`。深度优先从左到右遍历时，遍历到 B 时 C 尚未访问，`C.y` 还没算出来。

## 与 S-属性文法的关系

```
S-属性文法  ⊂  L-属性文法
```

S-属性文法是 L-属性文法的退化情况（继承属性集合为空）。L-属性文法通过引入**受限制的继承属性**，表达能力显著增强。
