# 静态语义 (Static Semantics)

**静态语义**是编译器在编译时（不运行程序）即可检查的语义规则。它是[[自底向上语法分析|语法分析]]和运行时行为之间的过渡层。

## 形式化定义

程序的语义可以分为两层：

| 层次 | 检查时机 | 工具 | 示例判断 |
|------|----------|------|----------|
| **静态语义** | 编译时 | [[属性文法]] + 类型系统 | `int x = "hello";` 类型不匹配，编译报错 |
| **动态语义** | 运行时 | 运行时系统 | `a / b` 中 `b == 0`，运行到此处才崩溃 |

## 静态语义分析的具体检查项

| 检查类别 | 形式化判据 | 依赖的信息 |
|----------|-----------|-----------|
| **声明检查** | `∀ id ∈ program, ∃ declaration(id) ∧ decl 先于 use` | 符号表（[[继承属性]]传递作用域） |
| **类型相容性** | `binop(expr₁, op, expr₂)` → `expr₁.type` 与 `expr₂.type` 在 `op` 下可合式 | 类型系统规则（[[综合属性]]推导类型） |
| **赋值相容性** | `expr.type ≤ var.type` | 类型系统 + 声明信息 |
| **函数调用** | 实参数量=形参 ∧ `∀i: argᵢ.type ≤ paramᵢ.type` | 函数签名表 |

## 为什么"静态"

"静态"指在程序运行之前（编译阶段）就能完成检查。正是因为编译器在代码执行前构建了语法树并计算了属性（类型、作用域），这些语义违规才能提前暴露。

## 静态语义的边界

并非所有语义错误都能被静态检查捕获：

| 可静态检查 | 需运行时检查 |
|-----------|------------|
| 类型不匹配 | 除零错误 |
| 变量未声明 | 数组越界（一般情况无静态上界信息） |
| 函数参数数量错 | 空指针解引用 |
| 不可达代码（部分） | 动态内存分配失败 |
