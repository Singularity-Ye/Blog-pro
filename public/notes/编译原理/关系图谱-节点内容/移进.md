# 移进 (Shift)

**移进**是语法分析器两大核心动作之一（另一个是 [[归约]]）。它代表了分析器对输入符号的实际消耗——将当前的输入 Token 读入并压入分析栈。

## 动作流程

1. **吃字符**：从输入流中读取当前的终结符（Token）。
2. **查表**：根据当前状态和这个字符，查找 [[Action 表]]，得到 `s j`（shift to state j）。
3. **压栈**：将该字符和 Action 表指示的新状态 $j$ 一起压入分析栈中。

### 栈的变化过程

```
移进前（栈顶在右侧）：
┌──────┬──────┬──────┐
│  ...  │  st  │  id   │   ← 当前输入: + (下一个要移进的Token)
└──────┴──────┴──────┘
           ↑
         栈顶状态 st, ACTION[st, +] = s6

移进后：
┌──────┬──────┬──────┬──────┐
│  ...  │  st  │  id   │  s6  │
└──────┴──────┴──────┴──────┘
                    ↑
                 新栈顶状态 6
```

## 移进在分析中的角色

- **消耗输入**：每次移进，输入指针前进一个 Token。
- **积累状态**：分析栈中压入的不只是符号，还包括状态编号——这是 LR 分析器"记忆历史"的方式。
- **可逆性**：移进的过程可以通过 [[归约]] 回滚。归约相当于"把移进的东西吐出来并合并"。

## 移进 vs 归约

| 维度 | 移进 (Shift) | [[归约]] (Reduce) |
|------|-------------|-------------------|
| 方向 | 吃入 | 吐出 |
| 栈变化 | 增加一个符号 | 弹出若干符号 |
| 输入消耗 | 消耗一个 Token | 不消耗 Token |
| 查表 | [[Action 表]] | Action 表 + [[Goto 表]] |
| 意义 | 推进解析进度 | 完成一个语法模块 |

## 关于"待约项目"与"移进项目"的区分

在 [[LR(0) 项目]] 的分类中：
- **移进项目**：$A \to \alpha \cdot a \beta$（圆点后是终结符 $a$），移进动作直接读入 $a$。
- **待约项目**：$A \to \alpha \cdot B \beta$（圆点后是非终结符 $B$），不能直接移进，要先完成 $B$ 的匹配。

> 移进动作只发生在圆点后是**终结符**时。如果圆点后是非终结符，分析器要等待（或通过 [[闭包运算]] 展开），而不是直接移进。

## 移进在 LR(0) 与 SLR(1) 中的异同

- **LR(0)**：只要状态中有合法移进项目且没有冲突就直接移进。
- **SLR(1)**：在存在 [[移进-归约冲突]] 时，通过 [[Follow 集合]] 决定是否移进。
