# 编译原理阶段总结

> 从语法分析到语义分析：LR(0) → SLR(1) → LR(1) → LALR(1) → Yacc → 属性文法 → 依赖图。这是编译器前端最密集的理论章节。

---

## 一、编译器前端管线

```
源代码字符流
  ↓  词法分析 (Lexer) —— 字符 → token
token 序列
  ↓  语法分析 (Parser) —— LR/SLR/LALR → 语法树
语法树
  ↓  ★ 语义分析 —— 属性文法 + 依赖图 → 带注解的语法树
中间代码生成
```

---

## 二、本阶段覆盖的知识体系

```
① LR 系列理论与构造算法
  LR(0) → SLR(1) → LR(1) → LALR(1)
  项目形式、CLOSURE、GOTO、项目集规范族、ACTION/GOTO 表

② 自底向上语法分析
  shift / reduce / accept / error 循环、分析栈的结构

③ Yacc 实践
  .y 文件三部分、%left/%right 优先级、$$/$n 属性传递、error 恢复

④ 语义分析
  类型检查、声明检查、静态语义 vs 运行时错误

⑤ 属性文法
  综合属性（子→父）、继承属性（父/兄→子）、S-属性、L-属性

⑥ 依赖图与拓扑排序
  属性求值顺序、有环判定、无环拓扑计算算法
```

## 五、一句话总括

> **LR 系列**解决"如何识别程序的语法结构"；**Yacc** 把 LR 理论变成可用的 parser 工具；**语义分析**在语法树上继续计算属性、检查语义规则；**属性文法**描述这些语义信息怎么算（S-属性子→父，L-属性允许父→子/兄→弟）；**依赖图**通过拓扑排序决定这些信息按什么顺序执行。
