# 编译原理阶段总结

> 从语法分析到语义分析：LR 系列方法 → Yacc 实践 → 属性文法 → 依赖图。

---

## 一、编译器前端管线

```
源代码
  ↓
词法分析 (Lexical Analysis)  → 字符流 → token 序列
  ↓
语法分析 (Syntax Analysis)   → token 序列 → 语法树
  ↓
语义分析 (Semantic Analysis)  → 语法树 + 属性 → 类型检查 / 语义检查
  ↓
中间代码生成 / 优化 / 目标代码
```

---

## 二、本阶段知识总线

```
1. LR 系列理论
   LR(0) → SLR(1) → LR(1) → LALR(1)

2. 自底向上语法分析
   shift / reduce / ACTION / GOTO / 项目集 / DFA

3. Yacc 实践
   写 .y 文件 → 自动生成 LALR(1) parser

4. 语义分析
   语法正确之后，检查类型、声明、语义信息

5. 属性文法
   给文法符号加属性，用语义规则计算属性

6. 依赖图
   右侧属性先算，左侧属性后算
```

---

## 三、各节点速记

| 概念 | 一句话 |
|------|--------|
| **LR(0)** | 不看前看符号，容易冲突 |
| **SLR(1)** | 归约时看 FOLLOW 集 |
| **LR(1)** | 每个项目自带 lookahead，最精确 |
| **LALR(1)** | 合并核心相同的 LR(1) 状态，工程首选 |
| **Yacc** | 用 LALR(1) 的 parser generator |
| **语义分析** | 检查类型、声明、表达式合法性 |
| **属性** | 文法符号上的语义信息（val, type, dtype） |
| **综合属性** | 子 → 父（如表达式值） |
| **继承属性** | 父/兄 → 子（如声明类型传播） |
| **依赖图** | 决定属性计算顺序，箭头从被依赖指向依赖者 |

---

## 四、一句话总括

> 前面 LR 系列解决「如何识别程序的语法结构」；Yacc 把 LR 理论变成可用的 parser 工具；语义分析在语法树上继续计算属性、检查语义规则；属性文法描述这些语义信息**怎么算**，依赖图决定这些信息**按什么顺序算**。
