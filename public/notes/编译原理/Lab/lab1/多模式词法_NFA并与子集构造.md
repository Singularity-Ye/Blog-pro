# 多模式词法分析：NFA ε-并 + 子集构造（方案二）

本文说明如何将多个 token 的自动机合并为**一台**确定性自动机，并与「积构造」（方案一）对照。适用于课程报告、与 `flex`/`lex` 类工具原理对照阅读。

---

## 1. 预处理：合并之前要做什么？

在把多台自动机合成一台之前，通常先对每个 **token 模式** 各自构造识别器（NFA 或 DFA）。

| 步骤 | 是否必须 | 说明 |
|------|----------|------|
| 每个 token 有一台识别器 | **必须** | 无单模式自动机则无法合并。 |
| 对字母表 Σ **补全转移**（死状态） | **强烈建议** | 避免某条路径在读下一字符时「突然无转移」，导致无法比较**更长词素**或与另一模式竞争。 |
| 单模式 **确定化** | 可选 | 可先对每个 NFA 做子集构造，也可多个 NFA 先并再**一次性**确定化。 |
| **最小化**（Hopcroft 等） | **不必须** | 减少状态数，**不改变识别语言**；可在合并前对每个 DFA 做，也可在得到总 DFA 后再做。 |

**结论**：第一步更准确的说法是「每个模式有定义清楚、最好对 Σ 完备的自动机」；**最小 DFA 是优化选项**，不是方案二在数学上的前提。

---

## 2. 方案二在做什么

### 2.1 直观

- 为每个 token 准备一台自动机 \(M_i\)（常为从正则经 Thompson 构造得到的 NFA，或已是 DFA）。
- 用 **ε 转移** 把一个新起始状态 \(S\) 连到各 \(M_i\) 的起点：**读入尚未开始**时，先「选择」进入哪一种模式。
- 各 \(M_i\) 的**终态**打上 **token 标签**（同一终态只属于一种 token）。
- 对得到的 **ε-NFA** 做 **子集构造**，得到**单一 DFA**；若某个子集状态同时「包含多种 token 的终态」，用 **优先级**（及常与 **最长匹配** 配合）决定输出。

### 2.2 图示（概念）

```text
              ε                 ε
       S ───────► q₀₁    ...    S ───────► q₀ₖ
                  │  M₁  ...                  │  Mₖ
                  ▼                           ▼
            (带标签的终态)              (带标签的终态)
```

原图若已是 DFA，加上 ε 后一般不再是 DFA，故整体视为 **NFA**，再确定化。

## 3. 子集构造（含 ε-闭包）

设输入字母表为 **Σ**（**不包含** ε 作为读入符号）。

### 3.1 ε-闭包

对状态集 \(T\)，定义  
\(\varepsilon\text{-closure}(T)\) 为：从 \(T\) 中任一点出发，**只沿 ε 边**能到达的所有状态（包含 \(T\) 自身）。

### 3.2 DFA 状态与转移

1. **初态**  
   \[
   D_0 = \varepsilon\text{-closure}(\{S\})
   \]

2. **转移**  
   对每个已出现的子集 \(D \subseteq Q\)、每个字符 \(a \in \Sigma\)：  
   令  
   \[
   U = \{ q' \mid \exists q \in D,\ q \xrightarrow{a} q' \},\quad
   D' = \varepsilon\text{-closure}(U)
   \]
   若 \(U\) 为空，通常不连边（或连到显式**死状态**，取决于是否对 Σ 完备）。

3. **接受与标签**  
   若 \(D\) 中含有 **带标签的 NFA 终态**，则 \(D\) 为 DFA 的接受态；标签可来自其中任意一个被接受的 token。若 **多种 token 的终态同时出现在 \(D\)**，实现上按 **优先级表** 选一个（例如关键字优先于标识符）。

### 3.3 最长匹配（与构造的关系）

- **构造阶段**：终态可带多个「可能 token」，用优先级消解部分歧义。  
- **扫描阶段**：从左到右读入，维护「最近一次仍处于某接受态的位置」；无法继续转移时，回退到该位置切出一个词素——即 **maximal munch**。

## 4. 与方法一（积 DFA）的关系

| 维度 | 方案一：积构造 | 方案二：ε-并 + 子集构造 |
|------|----------------|-------------------------|
| 思路 | 多台 DFA **同步**读同一字符，状态为元组 \((s_1,\ldots,s_k)\) | 先 ε **选模式**，再在一条路径上读；确定化后子集可能**同时包含**多台机中的状态 |
| 典型实现 | 手写多 DFA 并行、或显式建积 | `flex`：多正则 → 大 NFA → 一次确定化 |
| 本质 | 都是对「多模式并行匹配」做 **确定化**，只是起点不同 | 同上 |

## 5. 例子

### 5.1 例 A：`A`（单字符 `a`）与 `AB`（`ab`）

**Token A**：只接受 `"a"`。  
**Token AB**：只接受 `"ab"`。

简化 NFA/DFA（未画死状态）：

```text
M_A:   s ──a──► f_A   （接受 A）

M_AB:  p ──a──► q ──b──► r   （接受 AB）
```

**合并**：新起点 \(S\)，\(S \xrightarrow{\varepsilon} s\)，\(S \xrightarrow{\varepsilon} p\)；\(f_A\) 标 A，\(r\) 标 AB。

读第一个 `a` 时，从 \(s\) 可到 \(f_A\)，从 \(p\) 可到 \(q\)。子集构造会得到包含 **\(f_A\) 与 \(q\)** 的状态，表示「既可结束为 A，也可能是 AB 的前缀」。若再读 `b`：AB 路径 \(q \xrightarrow{b} r\)；若 A 侧未补全，`f_A` 上可能无 `b`，则仅 AB 继续——配合 **最长匹配** 与优先级 **AB 先于 A** 时，前缀 `"ab"` 标为 AB。

**要点**：若未对 Σ 补全，可能出现无转移；工程上常 **补全到死状态**，使转移表完整、最长匹配实现简单。

### 5.2 例 B：两 token 识别同一串（冲突）

**Token X** 与 **Token Y** 均只接受 `"a"`，终态分别标 X、Y。

ε-并后读入一个 `a`，某个 DFA 子集可能 **同时包含** 两个 NFA 终态 → **必须** 用 **优先级** 选一个，无法仅靠自动机结构区分。

### 5.3 例 C：工具链（flex 类）

1. 每个正则 → Thompson NFA →（可选）确定化为 DFA，终态记 token id。  
2. 或：多个模式 **ε-并** 成一大 NFA。  
3. **子集构造** → 总 DFA；表中每状态记录「可接受的 token 集合」+ 优先级。  
4. 运行时结合 **最长匹配** 切分词素。

这与「先写 \(R_1 \mid R_2 \mid \cdots\) 再确定化」是同一类流程的不同叙述。

## 6. 实现提示（ checklist）

- [ ] 明确 **Σ**（词法分析字符类）。  
- [ ] ε-NFA：**唯一**新起点 \(S\)，ε 到各 \(M_i\) 起点；终态 **打标签**。  
- [ ] 子集构造：**先 ε-闭包**，再按 \(a \in Σ\) 扩展并再 ε-闭包。  
- [ ] 接受态：子集中若含多种标签 → **优先级**。  
- [ ] 扫描：**最长匹配** + 非法字符/无转移时切分。  
- [ ] 可选：合并后对总 DFA **最小化**以省空间。
