# 语义分析与属性文法

> 在语法正确的基础上，判断程序是否"有意义"。用属性文法描述语义信息的计算规则。

---

## 一、语义分析是什么

编译器前端三阶段：

```
词法分析 → 语法分析 → 语义分析
```

| 阶段 | 检查的问题 |
|------|-----------|
| 词法分析 | 这些字符能拼成 token 吗？ |
| 语法分析 | 这个 token 序列符合文法吗？ |
| **语义分析** | 这个程序**有意思**吗？ |

语法分析说"句子结构对"，语义分析说"句子意思合理"。

### 典型检查项

- `x` 有没有声明过？
- `a + b` 两边的类型能不能相加？
- 表达式结果类型能不能赋给左边变量？
- 函数调用参数个数/类型对不对？

---

## 二、静态语义 vs 运行时错误

| 类型 | 说明 | 示例 |
|------|------|------|
| **静态语义** | 编译时就能检查出来 | `x = "hello"` 但 `x` 是 `int` |
| **运行时错误** | 只有运行时才知道 | `a / b` 中 `b == 0` |

语义分析（静态语义分析）只能检查编译期可判断的规则。

---

## 三、属性 (Attribute)

属性是附着在文法符号上的语义信息。

```
X.a = ...
```

| 属性 | 含义 | 示例 |
|------|------|------|
| `val` | 数值 | `number.val = 7` |
| `type` | 类型 | `exp.type = integer` |
| `dtype` | 数据类型（声明） | `id.dtype = real` |
| `name` | 名字 | `id.name = "x"` |
| `code` | 生成的代码 | `stmt.code` |

可以理解成语法树节点上的字段：`node->val`、`node->type`。

---

## 四、属性文法 (Attribute Grammar)

**属性文法 = 文法规则 + 属性计算规则**

普通 CFG 只描述结构：

```
number → number digit | digit
```

属性文法进一步说清楚语义信息怎么算：

```
number₁ → number₂ digit
  number₁.val = number₂.val × 10 + digit.val
```

### 例子：无符号整数

```
number₁ → number₂ digit
  number₁.val = number₂.val × 10 + digit.val

number → digit
  number.val = digit.val
```

`345` 的计算过程：

```
3                    → 3
3 × 10 + 4 = 34      → 34
34 × 10 + 5 = 345    → 345
```

### 例子：算术表达式

```
exp₁ → exp₂ + term
  exp₁.val = exp₂.val + term.val

exp₁ → exp₂ - term
  exp₁.val = exp₂.val - term.val

term₁ → term₂ × factor
  term₁.val = term₂.val × factor.val

factor → ( exp )
  factor.val = exp.val

factor → number
  factor.val = number.val
```

`(34 - 3) × 42` → `31 × 42` → `1302`

### 例子：变量声明的类型传播

```
decl → type var-list
  var-list.dtype = type.dtype

type → int
  type.dtype = integer

type → float
  type.dtype = real

var-list₁ → id , var-list₂
  id.dtype = var-list₁.dtype
  var-list₂.dtype = var-list₁.dtype
```

`float x, y` 的类型传播：

```
type.dtype = real → var-list.dtype = real → x.dtype = real, y.dtype = real
```

## 五、两种属性方向

### 综合属性 (Synthesized)

**从子节点 → 父节点**。子先算好，父再汇总。

```
number.val = digit.val          ← digit 算好 → number 接收
exp.val = term.val              ← term 算好 → exp 接收
```

常见于：表达式值、表达式类型、生成的代码。

### 继承属性 (Inherited)

**从父节点或兄弟节点 → 子节点**。上层或左边的信息传给下层。

```
var-list.dtype = type.dtype     ← type 算好 → var-list 接收
id.dtype = var-list.dtype       ← var-list 接收后 → 传给 id
```

常见于：变量声明类型传播、作用域信息、上下文传递。

### 对比

| 属性类型 | 流向 | 典型场景 |
|------|------|------|
| 综合属性 | 子 → 父 | `exp.val`、`number.val` |
| 继承属性 | 父/兄 → 子 | `id.dtype`、`var-list.dtype` |

## 六、八进制错误检查：语法对不代表语义对

带进制后缀的数字：`345d`（十进制）、`345o`（八进制）

```yacc
based-num → num basechar
basechar → o    { basechar.base = 8; }
basechar → d    { basechar.base = 10; }
```

文法允许 `189o`，因为 digit 可以是 `0-9`。但语义上八进制不应该有 `8` 和 `9`。

语义规则需额外检查：

```
digit → 8
  digit.val = if base == 8 then error else 8

digit → 9
  digit.val = if base == 8 then error else 9
```

> **语法正确 ≠ 语义正确**。189o 语法可能合法，但语义上是错误。
