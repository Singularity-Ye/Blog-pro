# RAG 上下文注入 (RAG Context Injection)

> **"不是把整本小说塞进 Prompt，而是只喂给模型它此刻需要知道的那一页。"**

**RAG 上下文注入**是 WeaveInk 的"记忆中枢"——负责在每次 AI 调用前，从知识库中精准检索最相关的设定片段，并编织为结构化的 System Prompt。

---

## 为什么不能直接把设定全塞进去？

| 方法 | Token 消耗 | 准确性 | 推理速度 |
|------|-----------|--------|---------|
| 全量注入（把 300 章设定全塞进 Prompt） | 极高（可能超出窗口） | 低（信息噪音淹没关键设定） | 极慢 |
| 无注入（让模型凭训练记忆） | 低 | 极低（幻觉概率极高） | 快但无效 |
| **RAG 按需注入（WeaveInk 方案）** | **可控** | **高** | **快** |

## 注入管线（6 步流水）

```
┌──────────────────────────────────────────────────┐
│ Step 1: 上下文切片                                │
│ 提取用户当前输入的"上下文窗口"（前后各 500 字）    │
└──────────────────────┬───────────────────────────┘
                       ▼
┌──────────────────────────────────────────────────┐
│ Step 2: 命名实体识别 (NER)                        │
│ 识别：人物名、地点名、事件名、道具名、概念名       │
│ 输出：实体列表 [{type: "character", name: "林夜"}]│
└──────────────────────┬───────────────────────────┘
                       ▼
┌──────────────────────────────────────────────────┐
│ Step 3: 向量检索                                  │
│ 对每个实体 → 在向量库中 Top-K 检索相关设定        │
│ K 值动态调整：人物 K=5，地点 K=3，事件 K=3        │
└──────────────────────┬───────────────────────────┘
                       ▼
┌──────────────────────────────────────────────────┐
│ Step 4: 知识图谱邻域展开                          │
│ 对检索到的每个节点，做 1-hop 邻域展开              │
│ 将关联的 [[人物]]、[[地点]]、[[事件]] 一并纳入    │
└──────────────────────┬───────────────────────────┘
                       ▼
┌──────────────────────────────────────────────────┐
│ Step 5: 去重 + 排序                               │
│ 按 (相关性 × 新鲜度) 加权排序                     │
│ 相关性：向量相似度；新鲜度：最近被修改/引用的时间  │
│ 裁剪到上下文预算上限                              │
└──────────────────────┬───────────────────────────┘
                       ▼
┌──────────────────────────────────────────────────┐
│ Step 6: 编织 System Prompt                       │
│ 将排序后的设定节点 JSON 化 → 注入 Prompt Template │
│ → 调用 Gemini API                                │
└──────────────────────────────────────────────────┘
```

## 子系统

| 子系统 | 职责 | 关键技术 |
|--------|------|---------|
| [[向量检索路由]] | 基于语义相似度查找相关设定节点 | embedding 模型 + 向量索引 |
| [[System Prompt 编织协议]] | 将检索结果组装为 AI 可理解的约束模板 | JSON → Prompt 模板引擎 |
| [[上下文预算管理]] | 动态分配 20%-60%-20% 的 Token 预算 | Token 计数器 + 优先级裁剪 |
