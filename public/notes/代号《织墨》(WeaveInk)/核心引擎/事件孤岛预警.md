# 事件孤岛预警 (Orphan Event Detection)

检测作品中未被主线因果链覆盖的独立事件簇，防止情节碎片化。

## 检测方法

1. 构建全作品的事件-章节二部图
2. 计算图的**连通分量**（Connected Components）
3. 对于规模较小的连通分量（如 < 3 个事件节点且无主角参与）→ 标记为"事件孤岛"

## 孤岛分级

| 级别 | 条件 | 建议 |
|------|------|------|
| 🟢 正常 | 独立事件被 ≥ 3 章引用 | 可能是重要的独立支线 |
| 🟡 关注 | 被 1-2 章引用，无因果链 | 考虑是否需要回收 |
| 🔴 孤岛 | 无章节引用，孤立存在 | 删除或并入主线的候选 |

## 技术实现

```python
import networkx as nx
G = nx.Graph()
# 添加事件节点和章节节点，边 = 引用关系
components = list(nx.connected_components(G))
for comp in components:
    if len(comp) <= 2 and all(n.startswith("事件·") for n in comp):
        report_orphan(comp)
```
