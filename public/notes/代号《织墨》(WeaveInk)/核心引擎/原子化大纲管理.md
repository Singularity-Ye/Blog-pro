# 原子化大纲管理 (Atomic Outline Management)

> **"不要把鸡蛋放在一个篮子里，也不要把所有设定塞进一个章节文件里。"**

**原子化大纲管理**是 WeaveInk 解决"大纲失控"问题的答案——将传统章节文件中内嵌的人物、地点、事件设定**拆分**为独立的 `.md` 原子节点，章节仅保留引用指针。

---

## 核心原则：关注点分离

```
传统大纲                           WeaveInk 原子大纲
┌──────────────────────┐            ┌──────────────────────┐
│ 第3章.md             │            │ 第3章.md             │
│                      │            │                      │
│ 人物：林夜（重伤）     │            │ 出场角色：            │
│ 地点：雾隐镇废墟       │     →      │   [[角色·林夜]]       │
│ 事件：遭遇帝国军       │            │   [[角色·陈雪]]       │
│ ...5000 字正文...     │            │ 场景：[[地点·雾隐镇]]  │
│                      │            │ 事件：[[事件·遭遇战]]   │
│ （设定与正文紧密耦合）  │            │ ...5000 字正文...     │
└──────────────────────┘            │                      │
                                    │ （设定引用，正文独立）   │
           ↑                        └──────────────────────┘
           │ 解耦                               │
           ▼                                    │
┌──────────────────────┐            ┌───────────┼───────────┐
│                      │            ▼           ▼           ▼
│  设定在正文中散落     │     ┌──────────┐ ┌──────────┐ ┌──────────┐
│  "林夜重伤"         │     │角色·林夜  │ │地点·雾隐镇│ │事件·遭遇战│
│  可能出现在第3章     │     │状态:重伤  │ │状态:已毁  │ │参战方:A,B │
│  但第5章才说明      │     └──────────┘ └──────────┘ └──────────┘
└──────────────────────┘     ← 修改一处，全局生效
```

## 节点分类体系

| 节点类型 | 命名前缀 | YAML 必填字段 | 示例 |
|---------|---------|-------------|------|
| **角色** | `角色·` | `status`, `aliases`, `first_appearance` | `角色·林夜.md` |
| **地点** | `地点·` | `region`, `status`, `first_appearance` | `地点·雾隐镇.md` |
| **事件** | `事件·` | `timestamp`, `participants`, `causes`, `effects` | `事件·帝都沦陷.md` |
| **道具** | `道具·` | `type`, `owner`, `significance` | `道具·霜月剑.md` |
| **组织** | `组织·` | `type`, `members`, `alignment` | `组织·帝国军.md` |
| **概念** | `概念·` | `domain`, `rules` | `概念·灵力体系.md` |

## 子系统

| 子系统 | 功能 | 算法 |
|--------|------|------|
| [[节点引用计数]] | 统计每个设定节点被多少章节引用 | 入度计算 + 异常检测 |
| [[变更传播策略]] | 修改设定节点后，标记所有受影响的章节 | 依赖图 BFS + 脏标记 |
| [[依赖图分析]] | 分析设定之间的依赖关系，防止循环引用 | 拓扑排序 + 环检测 |

## 章节文件标准模板

```markdown
---
chapter: 3
title: "废墟中的重逢"
timestamp: "永安二年·秋·黄昏"
characters:
  - "[[角色·林夜]]"
  - "[[角色·陈雪]]"
locations:
  - "[[地点·雾隐镇]]"
events:
  - "[[事件·遭遇帝国军]]"
word_count: 4500
status: draft
---

# 第3章：废墟中的重逢

（正文内容...）
```
