# 时间线冲突检测 (Timeline Conflict Detection)

解析所有章节和事件的 YAML `timestamp` 字段，构建全作品时间轴，自动检测三种典型异常。

## 检测类型

| 异常 | 触发条件 | 示例 |
|------|---------|------|
| **时间倒流** | 第 N+1 章的 timestamp 早于第 N 章 | 第 28 章"永安二年"引用"永安三年"事件 |
| **角色复活** | 死亡事件在时间 T，后续章节角色仍在活动 | 第 18 章林夜"确认死亡"，第 25 章林夜出场 |
| **事件因果反转** | 事件 A 的 `effects` 引用了时间戳更早的事件 B | 效果的时间戳早于原因 |

## 时间戳解析器

支持多种格式：
- `"永安三年·三月·黎明"` → 解析为 `year:3, month:3, period:dawn`
- `"第23章"` → 从章节索引推断
- 绝对时间轴 → Canonical Timeline Index (CTI)

## 与 [[因果链追溯]] 的关系

时间线冲突检测是**时序维度**的分析，因果链追溯是**逻辑维度**的分析。两者交叉验证。
