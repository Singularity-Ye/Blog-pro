# 人物状态机 (Character State Machine)

每个角色都是一个**状态向量**，其属性随时间演进并有明确的转移规则。

## 状态向量定义

```yaml
character_state:
  health: [healthy, wounded, critical, dead]
  location: 地点·节点引用
  equipment: [道具·节点引用]
  relationships: [{target: 角色, type: 关系, strength: 0-1}]
  knowledge: [事件·节点引用]  # 角色已知的信息
  emotional_state: enum
```

## 状态转移校验规则

- `dead → alive`：**硬性阻断**（除非有复活事件节点链接）
- `critical → healthy`：需链接到"治疗事件"
- 装备变更必须与手部状态一致（独臂 → 无法双持）

## 与 [[设定守护者]] 的关系

状态机是守护者 L3 约束层的核心数据结构。每次生成前，守护者查询当前角色的状态向量，将其转化为约束条件注入 [[System Prompt 编织协议]]。
