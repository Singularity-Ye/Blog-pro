# 变更传播策略 (Change Propagation Strategy)

当创作者修改设定节点（如将"林夜重伤"改为"林夜康复"），系统自动标记所有引用该节点的章节并生成变更影响报告。

## 传播算法

```
1. 创作者修改 [[角色·林夜]].md 的 status 字段
2. 索引器检测到文件变更 → Hash diff
3. 查询 [[节点引用计数]] 中的 in_links：
   受影响章节 = ["第3章", "第5章", "第8章", ...]
4. 对每个受影响章节：
   - 提取涉及该角色的段落
   - 用新设定校验段落中的描写是否冲突
   - 生成"建议修改列表"
```

## 三种传播策略

| 策略 | 行为 | 适用场景 |
|------|------|---------|
| **静默标记** | 只标记章节为 dirty，不中断 | 小改动（修改外表描述） |
| **建议提示** | 弹出冲突列表，可忽略 | 中等改动（状态变化） |
| **强制审查** | 阻塞生成，直到确认 | 重大改动（角色死亡/复活） |
