# 设定守护者 (Lore Keeper)

> **"在织墨的世界里，设定不是文档，是牢不可破的法律。"**

**设定守护者**是 WeaveInk 的**核心防御机制**——它不是修改你的文字，而是在你即将打破自己立下的规则时，轻轻拉住你的衣袖。

---

## 设计哲学

### 从"记忆工具"到"约束引擎"

| 层级 | 能力 | 技术手段 |
|------|------|---------|
| **L1 记忆层** | "这一段提到了林夜" | NER + 向量检索 |
| **L2 关联层** | "林夜怕火，而这场戏有火光" | 知识图谱邻域查询 |
| **L3 约束层** | "林夜现在不能用双手武器——他刚刚失去了左臂" | 状态机 + System Prompt 约束注入 |
| **L4 告警层** | "⚠️ 冲突：第 23 章设定林夜左臂已断，当前描写双手握剑" | 跨节点一致性校验 |

## 工作流

```
1. 创作者输入新段落
        │
2. NER 提取 → {人物, 地点, 事件, 道具}
        │
3. 向量检索 Top-K 相关设定节点
        │
4. 知识图谱邻域展开 (n-hop)
        │
5. 状态机校验 → 是否有冲突？
        │
    ┌───┴───┐
    ▼       ▼
  无冲突   有冲突
    │       │
    │       ▼
    │   ┌──────────────────┐
    │   │ 生成冲突报告      │
    │   │ 类型 / 严重度 /   │
    │   │ 建议修复方案      │
    │   └──────────────────┘
    ▼
6. 编织 System Prompt
        │
7. Gemini API 调用（携带约束）
```

---

## 冲突分级

| 级别 | 名称 | 示例 | 行为 |
|------|------|------|------|
| 🟡 L1 | 轻微不一致 | "林夜的眼睛是黑色" vs "深棕色" | 静默记录，不中断 |
| 🟠 L2 | 状态偏差 | 角色在重伤状态下进行高强度战斗 | 弹窗警告，可忽略 |
| 🔴 L3 | 硬性矛盾 | 已死角色再次登场且没有复活解释 | **阻断生成**，强制修复 |
| ⚫ L4 | 世界观崩塌 | 违反核心物理/魔法规则 | 拒绝生成，要求主编确认 |
