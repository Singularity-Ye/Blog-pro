# 上下文预算管理 (Context Budget Manager)

在 Gemini 2M Token 上下文窗口内，按 20%-60%-20% 策略动态分配 Token 配额。

## 预算计算器

```python
class ContextBudget:
    TOTAL = 2_000_000      # Gemini 2.5 Pro 窗口
    STATIC_PCT = 0.20      # System Prompt 模板
    DYNAMIC_PCT = 0.60     # RAG 检索设定
    ELASTIC_PCT = 0.20     # 对话历史 + 用户输入

    def allocate(self, rag_nodes: list[SettingNode]) -> dict:
        static = int(self.TOTAL * self.STATIC_PCT)
        dynamic = int(self.TOTAL * self.DYNAMIC_PCT)
        elastic = self.TOTAL - static - dynamic

        # 按 (相关性 × 新鲜度) 排序，贪婪填充
        sorted_nodes = score_and_sort(rag_nodes)
        allocated_budget = 0
        selected = []
        for node in sorted_nodes:
            tokens = estimate_tokens(node)
            if allocated_budget + tokens > dynamic:
                break
            selected.append(node)
            allocated_budget += tokens

        return {"static": static, "dynamic": allocated_budget,
                "elastic": elastic, "selected": selected}
```

## 裁剪策略

当检索到的设定节点总 Token 超出 60% 预算时：
1. 保留所有"硬约束"角色（当前场景出场角色）→ 完整内容
2. 背景角色 → 仅保留 `status` 摘要
3. 历史事件 → 仅保留 `effects` 摘要
