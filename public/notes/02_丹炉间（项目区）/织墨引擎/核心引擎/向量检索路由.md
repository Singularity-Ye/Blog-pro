# 向量检索路由 (Vector Retrieval Router)

根据用户当前写作上下文中提取的实体，在向量数据库中检索最相关的设定节点。

## 检索策略

```
输入：实体列表 + 上下文 embedding
输出：Top-K 设定节点（按相关度排序）

1. 对每个实体单独检索 Top-3 最匹配节点
2. 对全文上下文做语义检索 Top-5
3. 合并 → 去重 → 按 (cosine_similarity × freshness_score) 排序
4. 分配到上下文预算的各槽位
```

## 嵌入模型选型

| 模型 | 维度 | 优点 | 缺点 |
|------|------|------|------|
| Gemini text-embedding-004 | 768 | 与生成模型同源 | 需网络 |
| BGE-M3 | 1024 | 多语言、本地部署 | 需 GPU |
| text2vec-large-chinese | 1024 | 中文优化 | 仅中文 |

MVP 阶段推荐 Gemini embedding（零运维），长期可切换 BGE-M3 本地推理。
