# WeaveInk：创作流协议

> **"好的工具不规定你怎么写，但坏的规范会让工具无法工作。这里是底线。"**

**创作流协议**是 WeaveInk 生态中创作者与工具之间的"接口规范"——只要遵循这些约定，WeaveInk 就能正确解析、索引、检索你的设定。

---

## 文件命名规范

| 类型  | 命名格式        | 示例              |     |
| --- | ----------- | --------------- | --- |
| 章节  | `第N章：标题.md` | `第3章：废墟中的重逢.md` |     |
| 角色  | `角色·名称.md`  | `角色·林夜.md`      |     |
| 地点  | `地点·名称.md`  | `地点·雾隐镇.md`     |     |
| 事件  | `事件·名称.md`  | `事件·帝都沦陷.md`    |     |
| 道具  | `道具·名称.md`  | `道具·霜月剑.md`     |     |
| 组织  | `组织·名称.md`  | `组织·帝国军.md`     |     |
| 概念  | `概念·名称.md`  | `概念·灵力体系.md`    |     |

## [[YAML 元数据标准]]

### 章节文件

```yaml
---
chapter: 3
title: "废墟中的重逢"
timestamp: "永安二年·秋·黄昏"
characters:
  - "[[角色·林夜]]"
  - "[[角色·陈雪]]"
locations:
  - "[[地点·雾隐镇]]"
events:
  - "[[事件·遭遇帝国军]]"
word_count: 4500
status: draft
prev_chapter: "[[第2章：边境迷雾]]"
next_chapter: "[[第4章：援军到来]]"
---
```

### 人物文件


## [[Wiki-link 约定]]

### 必做事项

- ✅ 章节中引用角色/地点/事件时，**始终使用** `[[角色·XXX]]` 格式
- ✅ 建立角色关系时，在双方文件中**互链**
- ✅ 道具节点中标注 `owner` 字段，指向当前持有角色

### 禁止事项

- ❌ 不使用 `[角色名](app://...)` 等外部链接格式
- ❌ 不在正文中内嵌冗余设定（设定应放在独立的设定文件中）
- ❌ 不创建指向不存在的节点的链接（先创建节点，再引用）

### 链接密度指南

| 文档类型 | 建议链接密度 | 示例 |
|---------|------------|------|
| 章节正文 | 每 500 字 1-2 个链接 | 关键人物首次出场、重要地点切换时 |
| 角色设定 | 丰富链接到关联角色和所属组织 | 关系网越密越好 |
| 事件设定 | 链接到前因事件和后果事件 | 形成因果链 |

## [[Markdown 编写规范]]

1. **章节标题**：使用 `#` 一级标题
2. **场景切换**：使用 `---` 分隔线 + `##` 二级标题标注新场景
3. **内联标注**：在需要特别提醒守护者的段落前，使用 `<!-- weaveink:constraint 约束描述 -->` HTML 注释
4. **版本控制**：每次修改角色设定时，在文件的 `changelog` 字段中追加修改记录
