## 一、博客页整体定位

这个页面可以叫：

**松果灵感书屋**  
或者更像项目页一点：

**松果书屋 · 项目藏书室**

它承担的是：

> 展示你的项目、实验、文章入口、创作过程，而不是传统博客文章流。

也就是说，它不是“博客列表”，而是一个**个人世界观式的内容空间**。

首页可以继续是星球、宇宙、梦境入口；  
博客页则变成进入内部后的一个小房间：  
从广阔宇宙落进一枚松果里，像钻进某个私人图书馆的夹层。

这个过渡非常妙，有一种“从宏观星球到微观巢穴”的空间缩放感。

## 三、项目入口设计：不要做成卡片，做成“物件”

你 Obsidian 里整理的方向很好。可以进一步规范成 4 类物件：

### 1. 旅行地图册

对应：

- 3D 旅游路线地图
- 杭州巡游计划
- 旅途节点手札

视觉可以是一本摊开的地图册，上面有红线、节点、小旗子、折痕、手绘标注。

点击后打开详情：

```
《3D 旅行路线图册》一套把旅行路线、景点节点和 3D 地图结合起来的实验。
```

适合放在青蛙旁边，像它正在研究下一趟旅行。

---

### 2. 古旧卷轴 / 魔法地图书

对应：

- 现实地图魔改小说世界观
- 小说世界观设定
- 旧城改写计划

视觉可以是半卷开的羊皮纸，有火漆印、墨水标注、地形线条。

这个物件最好放在稍微暗一点的位置，旁边有一盏小灯。  
它的气质是“把现实地图炼成幻想大陆”。

---

### 3. 标签笔记本 / 节点索引册

对应：

- Obsidian 图谱
- 知识整理
- 旅行节点抽取实验

视觉可以是活页夹、标签贴纸、线绳、便签、节点连线。

它可以承担“方法论入口”的作用，展示你如何用 Obsidian、Claudian、节点系统整理知识和项目。

这类不一定特别梦幻，可以稍微偏“工作台”质感。

---

### 4. 技术手册 / 机关书

对应：

- Three.js 视觉实验
- 前端实验
- 交互式地图界面
- 奇怪但有趣的小组件

视觉可以更活泼一点：  
一本弹出式机关书，打开后冒出小型 3D 方块、地图、代码气泡、发光粒子。

它很适合放在右侧或者前景，作为“可点击的技术实验入口”。

## 四、交互建议：先做三档，不要一口吃成松果饼干 🐸

你现在设想了 hover、点击、拖拽、长按。都很有趣，但开发时建议分阶段。

### 第一阶段：静态 + hover + 点击

先完成最核心体验：

- 页面中央是松果屋
- 4 到 6 个项目物件分布在屋里
- hover 时物件轻微抬起、发光、显示标题
- 点击后右侧或弹窗展示项目详情

这是最稳的 MVP。

交互可以这样：

```
hover：  transform: translateY(-8px) rotate(-2deg)  filter: drop-shadow(...)  显示小标签click：  打开项目详情面板
```

这一阶段就已经很有完成度了。

---

### 第二阶段：书本打开动画

点击项目后，不只是弹出普通卡片，而是：

- 地图册摊开
- 卷轴展开
- 笔记本翻开
- 机关书弹出

这一步可以用 Framer Motion 或 CSS 动画实现，不一定马上上 Three.js。

比如点击《3D 旅行路线图册》后，详情面板像一本打开的地图册：

```
左页：项目介绍右页：截图 / 路线示意 / 技术栈
```

---

### 第三阶段：拖拽 / 物理动画

这个阶段再考虑：

- 书本可以被拖起来
- 放手后弹回书架
- 青蛙眼睛跟随鼠标
- 灯光轻微摇晃
- 长按翻页

拖拽很可爱，但也容易变成“交互章鱼锅”：好玩归好玩，页面功能可能被掩盖。所以建议后置。

---

## 五、青蛙角色：别让它变成结构件，要让它成为“馆长”

你这个判断很对：

> 青蛙不是“舌头挂图片”的结构件，而是吉祥物、书屋居民、灵感管理员。

也就是说，青蛙不负责承重，负责“世界观”。

可以给它一些轻交互：

```
默认状态：躺着看书hover 某本书：青蛙抬眼看向那本书点击项目：青蛙举起小牌子介绍页面空闲：青蛙翻页 / 打哈欠 / 眨眼
```

甚至可以有一句小提示：

```
“今天要翻哪本灵感册？”“这本地图册里夹着一条还没走完的路线。”“这卷旧地图有点烫，像刚从现实世界里偷回来。”
```

这种文案会让页面变成你的个人领地，而不是模板站。

## 六、页面信息架构建议

点击某个项目后，详情区可以统一成这个结构：

```
# 项目名称一句话说明：这个项目是做什么的？项目状态：构思中 / 开发中 / 已完成 / 实验中核心内容：- 主要功能- 视觉风格- 技术栈- 当前进度入口：- 查看项目- 查看设计笔记- 查看源码- 查看相关博客
```

比如：

```
# 《3D 旅行路线图册》一句话说明：把 Obsidian 里的旅行节点、路线规划和 3D 地图界面连接起来。项目状态：开发中核心内容：- 景点节点规范化- 路线跨区域展示- Three.js 地图可视化- Obsidian 到网页的数据迁移入口：- 项目说明- 路线样例- 技术记录
```

这套结构可以直接变成你的项目 JSON 数据。

## 九、技术实现路线

建议先别直接上完整 3D。这个页面最适合这样做：

### 方案 A：2.5D 手绘网页，优先推荐

技术：

- React / Next.js
- CSS absolute 定位
- Framer Motion 动画
- SVG 或 PNG 手绘素材
- 项目数据 JSON

优点：

- 容易做出你要的童话感
- 性能稳定
- 适合博客页
- 后期可以逐渐加交互

页面主体可以是一张松果屋大图，然后每个项目物件作为独立 PNG/SVG 叠在上面。

---

### 方案 B：Three.js 3D 松果屋

技术：

- React Three Fiber
- Drei
- 3D 模型 / 平面贴图
- 物理拖拽

优点是酷，缺点是成本高。  
如果你一开始就上 3D，容易把重点从“博客内容”转移到“调模型地狱”。

建议先用 2.5D 做出完整体验，后续再把局部物件升级成 3D。

## 十、下一步最值得做什么

我建议你下一步让 agent 做三件事：

第一，把这个页面抽象成**设计规范文档**，包括页面结构、物件类型、交互规则、项目数据结构。

第二，生成一版**低保真 React 原型**：松果屋背景 + 几个可 hover/click 的项目物件 + 详情面板。

第三，用 image2 分批生成素材，不要一次生成整页。先生成：

```
1. 松果剖面屋背景2. 青蛙管理员3. 旅行地图册4. 古旧卷轴5. Obsidian 标签笔记本6. Three.js 机关书
```

这样素材可控，前端也好组合。

---

整体方向非常可取。  
这个博客页的核心不是“炫技”，而是让访客感觉：  
**他们不是在浏览你的项目列表，而是在翻一只青蛙管理员收藏的灵感书架。** 🐸📚
