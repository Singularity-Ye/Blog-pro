# 一、我建议的命名原则

不要搞成太短的乱码，比如：

```
mb_c.pngib_h.png
```

这种虽然短，但后期你自己都容易看晕。

建议用这种风格：

- **文件夹表达物件类型**
- **文件名只表达状态**

也就是：

```
items/mapbook/closed.pngitems/mapbook/hover.pngitems/mapbook/open.png
```

这样比：

```
travel-mapbook-closed.pngtravel-mapbook-hover.pngtravel-mapbook-open.png
```

简洁很多。

# 六、前端里也要顺手做“别名对象”

这样 Codex 以后就不用到处写长路径了。

## 推荐写法

```
export const BLOG_ASSETS = {
  interior: {
    master: "/images/blog/interior/master.png",
    leftShelf: "/images/blog/interior/left-shelf.png",
    centerDesk: "/images/blog/interior/center-desk.png",
    travelZone: "/images/blog/interior/travel-zone.png",
    rightShelf: "/images/blog/interior/right-shelf.png",
    scrollZone: "/images/blog/interior/scroll-zone.png",
  },
  items: {
    mapbook: {
      closed: "/images/blog/items/mapbook/closed.png",
      hover: "/images/blog/items/mapbook/hover.png",
      open: "/images/blog/items/mapbook/open.png",
    },
    indexBook: {
      closed: "/images/blog/items/index-book/closed.png",
      hover: "/images/blog/items/index-book/hover.png",
      open: "/images/blog/items/index-book/open.png",
    },
    scroll: {
      rolled: "/images/blog/items/scroll/rolled.png",
      hover: "/images/blog/items/scroll/hover.png",
      half: "/images/blog/items/scroll/half.png",
      open: "/images/blog/items/scroll/open.png",
    },
  },
};
```

这样后面代码里用：

```
BLOG_ASSETS.items.mapbook.open
```

而不是每次都写整个路径字符串。

这对 **token、维护性、可读性** 都更友好。
