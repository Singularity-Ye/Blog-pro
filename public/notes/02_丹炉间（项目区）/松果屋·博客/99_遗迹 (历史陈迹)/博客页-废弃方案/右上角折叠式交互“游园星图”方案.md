# 空间长廊 3.6：右上角折叠式交互“游园星图”方案 (Implementation Plan)

我们根据道友的新构想，将全屏地图修改为位于页面右上角的 **实时交互式折叠小地图（游园星图）**。在保留场景双路径探索的同时，提供更贴合游戏实感的常驻雷达导航。

---

## 1. 结构化空间地图双主线 (Corridor Routing Map)

新场景命名为 **`junction`**（前端显示为 **`书脊回廊`**）。空间探索动线定义如下：

* **外线**：`overview` (总览) $\rightarrow$ `outdoor` (叶间书林) $\rightarrow$ `travel` (旅图案台)
* **内线**：`overview` (总览) $\rightarrow$ `indoor` (松果屋内) $\rightarrow$ `junction` (书脊回廊) $\rightarrow$ `archive` (旧札柜) / `workshop` (建站工坊)

---

## 2. 折叠式小地图 (Minimap) 交互与视觉设计

### 🗺️ 游园星图定位与交互：
1. **常驻右上角**：位于页面右上角，浮动层级（`z-index: 999`），使用 Framer Motion 实现平滑的展开/折叠动效。
2. **折叠状态 (Collapsed)**：
   * 展示为一个精巧的黄金罗盘/星徽图标（带自转和呼吸动效）。
   * 悬停时轻微放大并提示 `展开小地图`。
   * 点击后，向左侧平滑滑出/展开为精密的星图面板。
3. **展开状态 (Expanded)**：
   * **背景设计**：设计一幅融合中世纪天文占星图、星座连线、暗金墨水与暗夜靛蓝的羊皮纸贴图（`bg-minimap.png`）作为小地图水印背景（透明度 0.15），呈现复古法阵实感。
   * **连线 (Constellation lines)**：使用 SVG `<line>` 在节点间绘制半透明的轨迹虚线。当前所处的路线会带有彩色脉动流光（SVG `stroke-dashoffset` 动画），象征道友走过的魔法路径。
   * **星徽节点 (Scene Nodes)**：
     * 7个场景在小地图中以发光的星标节点（微型圆形徽章）排列。
     * 每个节点包含简短代称（如 `总览`, `屋内`, `林间`, `回廊`, `旅案`, `旧柜`, `工坊`）。
     * 点击任意节点，角色直接原地穿梭传送至该场景。
   * **道友所在标记 (Current Pointer)**：
     * 道友当前所处的场景星标上，会叠放一个旋转并微微跳动的金色指针星标（如 `✨` 或 `🧙‍♂️`），并带有扩散的金色魔力圈圈。
   * **折叠按钮**：右上角提供 `✕` 按钮或点击星徽图标可将其一键收回，保证不遮挡文章正文阅读。

### 3. 星图节点空间布局坐标 (Grid Coordinates in 280x220 container)：
* `overview` (总览) : `x: 50%`, `y: 15%`
* `indoor` (屋内) : `x: 30%`, `y: 40%`
* `outdoor` (屋外) : `x: 70%`, `y: 40%`
* `travel` (旅案) : `x: 70%`, `y: 75%`
* `junction` (回廊) : `x: 30%`, `y: 65%`
* `archive` (旧柜) : `x: 15%`, `y: 85%`
* `workshop` (工坊) : `x: 45%`, `y: 85%`

## 4. 验证方案 (Verification Plan)

### 自动验证
* 运行项目构建验证以防任何语法死角：
  ```powershell
  npm run build
  ```

### 手动验证
* 访问 `/blog`，在页面右上角检查是否出现常驻的星徽小地图。
* 点击 `🧭 游园星图` 或黄金罗盘，检查是否能丝滑展开星图。
* 点击星图中的“回廊”、“工坊”等节点，验证是否实现原地穿梭传送。
* 观察连线是否呈脉动流光，当前所处节点的跳动星徽是否位置正确。
* 展开阅读博文时，检查折叠小地图是否能够收起，不遮挡正文。
