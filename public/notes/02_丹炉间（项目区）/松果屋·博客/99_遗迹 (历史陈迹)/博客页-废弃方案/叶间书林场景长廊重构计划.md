# 叶间书林场景长廊重构计划 (Implementation Plan)

我们计划将新的博客页面（`/blog-new`）升级为模块化、配置驱动的“场景长廊”架构。这不仅能优化当前已有的三个核心场景，还能为未来扩建的场景（如旅图案台、旧札柜、建站工坊）奠定清晰的扩展基础。

---

## 2. 修改文件列表 (Proposed Modifications)

#### [MODIFY] [BlogNew.jsx](file:///d:/Yhx06/Documents/全栈学习模板/个人博客网站/personal-blog/src/pages/BlogNew.jsx)
* **抽取数据配置**：将 `INDOOR_LEAVES` 和 `OUTDOOR_SCROLLS` 整合进统一的 `BLOG_SCENES` 配置。
* **重构切换方法**：
  * 修改 JSX 渲染部分，使 `BgLayer`、入口热区和二级场景完全由 `BLOG_SCENES` 数据动态 map 生成，缩减模板硬编码，保持逻辑的极简与清晰。
  * 引入 `isUnderConstruction` 容错逻辑：若用户通过长廊误入尚未完工的预留场景，展示一个极具仙气/魔法风格的“建设中”优雅过渡提示卡片。
* **物理防抖交互**：保留已实现的 750ms `isTransitioning` 连点屏蔽与 16:9 视口居中适配。

## 3. 验证方案 (Verification Plan)

### 自动验证 (Automated Tests)
* 运行打包脚本确保代码合规：
  ```powershell
  npm run build
  ```

### 手动验证 (Manual Verification)
* 打开浏览器访问 `/blog-new`。
* 检查场景导航过渡是否正常，文字内容是否渲染正常。
* 验证三个未开放的场景（旅图案台、旧札柜、建站工坊）是否有优美的占位提示，不会报错崩溃。
