# 松果星寰仪：首页设计手札

首页不是传统意义上的"欢迎页"，而是一座被压缩在首屏之中的三维精神枢纽。

它的核心意象是——**太初星宇，灵识之茧。** 博客、图谱、项目、联系、起源五类内容，被转译为星球上的五大生态秘境。读者进入首页，看到的不是"站点入口"，而是一件可以拨动、旋转、感应、传送的松果星寰仪。

---

## 一、五大生态版图映射

导航不是文字列表，而是五种地貌。每个生态定义在 `biomeConfig.js` 中，同时驱动左侧图例卡片和星球表面大陆的高亮与跳转。

**潮汐海 · 起源归位。** 星球的底层母体，不对应凸起大陆而是整个原初海面。点击时指向 `/`——不是"去某个页面"，而是**归位、重整视角、回到主控法阵**。

**叶间书林 · 博客入口。** 对应 `/blog`。灵感碎片、技术心得、凡尘杂感都挂在林间枝叶之间。星球表面落在 `green-chronicle-continent`，纬度 18、经度 -58。

**星砂图原 · 图谱入口。** 对应 `/atlas`。笔记之间的链接关系被转译为荒漠中的星砂棋盘，与图谱页"星砂浑天图谱"的世界观相互咬合。大陆位于星体偏前位置，视觉上更容易被发现。

**幽霜析炼所 · 项目入口。** 对应 `/projects`。代码逻辑被冷凝、析出、铸造成可交互的小法宝。大陆贴图使用雪地纹理，冰晶蓝白，与项目页的"工坊 / 实验 / 造物"气质呼应。

**池畔信亭 · 联系入口。** 对应 `/contact`。水汽氤氲的小亭与精神信标。大陆色彩偏暖橙，像晨曦中的灯塔，与联系页的画卷、水面、信件、飞鸟意象同源。

---

## 二、生态图例：二维界面的水滴水晶拟态

左侧 `PlanetLegend` 是 3D 星体的地面控制台。每个 `LegendItem` 既是导航按钮，也是与星球联动的生态感应器。

鼠标进入图例 → 设置 `activeBiome` → 星体表面对应大陆高亮。离开 → 清空。**左手点名，右侧大陆发光。** 这不是两套 UI，而是一套共用状态的双界面系统。

`LegendSwatch` 的设计尤其精巧——根据 `$biomeKey` 切换不同的 CSS 材质：

- **forest**：树叶晨露，非对称叶滴形
- **snow**：霜冻冰晶，冷蓝半透明
- **ocean**：圆形深海水滴
- **desert**：琥珀砂印
- **city**：晨曦信标晶体

每个图标都用 `radial-gradient`、`backdrop-filter`、`inset box-shadow` 和 `::before` 高光来模拟微型晶体质感。`active` 状态下，图标切换为更强烈的渐变色并对 SVG 做旋转、缩放或偏移。

## 三、传送延迟：点击不是跳走，而是开门

图例点击后不立即跳转。先设 `activeBiome`，延迟 220ms：

```javascript
setActiveBiome(biomeKey);
window.setTimeout(() => navigate(biome.href), 220);
```

星球大陆点击后先触发 `burstRef.current = 1` 让大陆高亮爆发，再延迟 300ms 导航。

这 220-300ms 的停顿不是性能问题——是刻意留出的"仪式间隙"。用户感到自己不是点了链接，而是**激活了某块大陆，随后被传送过去**。交互事件有了因果：触碰 → 发光 → 开门。

---

## 四、萤火虫 CTA 与流光扫面

"关于本站"入口没有做成普通按钮，而是一枚不规则路牌状 CTA——多层径向渐变、蓝绿线稿 SVG 纹理、磨砂背景、内描边和 hover 抬升，形成一种"星图路牌"的质感。

右侧 `Firefly` 是点睛之笔。默认状态下它只是一个沉睡的黄色微光点；当用户 hover CTA 时，触发 `fireflyFlight` 在按钮右侧做不规则飞行，同时三个 `.sparkle` 光尘粒子错时闪烁。同一时间，`CtaBg::before` 执行 `cardShine` 扫光，从左至右掠过按钮表面。

这让"关于本站"不是冷冰冰的说明入口，而像**一只引路微光**——在你靠近时苏醒。

---

## 五、Loading 不是等待，而是 WebGL 构筑仪式

首页预加载背景图并设置最小展示时间 1400ms。这 1.4 秒不是无谓拖延——WebGL 首次进入时需要编译 shader、加载纹理、创建几何、初始化 Canvas。若直接展示，星体可能突然闪现或材质空白。

所以首页用加载屏挡住"星寰仪构筑"的过程。等背景与 WebGL 有足够时间准备，再通过 `AnimatePresence` 让遮罩以 0.65 秒淡出。

> 不让用户看到机器启动，让用户看到星门揭幕。

---

## 六、整体设计价值

松果星寰仪完成了三件事：

**把信息架构视觉化。** 博客、图谱、项目、联系不再是菜单项，而是星体生态。

**把导航变成探索。** 左侧图例、右侧星球、大陆 hover、点击传送、萤火虫 CTA，全部围绕"拨动星体，进入异界"展开。

**把技术服务于叙事。** R3F、shader、Canvas、球面几何、CSS 拟态、Framer Motion、Loading 缓冲——不是各自炫技，而是共同维护一个体验：**读者不是打开网页，而是唤醒一件星寰仪。**
