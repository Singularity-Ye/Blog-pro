# 松果屋发布控制台 (Pinecone Archive)

> 连接 Obsidian Vault 与博客系统的本地可视化发布管理工具。像图书管理员一样挑选哪些笔记可以上架，哪些留在档案室。

---

## 一、构思与定位

Obsidian Vault 里装着大量内容——成品笔记、半成品草稿、模板、私密日记、实验记录。不是所有内容都适合发布到博客。

松果屋发布控制台就是解决这个问题的中间层：

```
Obsidian Vault ──→ 🍍 发布控制台 ──→ 博客图谱
   (全部笔记)      (勾选/排除)      (只发布选中的)
```

不需要手动编辑 JSON 配置文件。打开 Web 界面，鼠标勾选哪些分类、哪些笔记要发布，点击同步，完事。

---

## 二、技术架构

```
┌─────────────────────────────────────────────┐
│              浏览器 (localhost:4177)          │
│          暗色玻璃态 UI (Glassmorphism)        │
├─────────────────────────────────────────────┤
│           后端 (scripts/publish-admin.mjs)    │
│         Node.js 原生 http + fs，零外部依赖     │
├─────────────────────────────────────────────┤
│         配置核心 (publish-manifest.json)      │
│    Vault 路径、排除规则、分类集合、文件清单     │
├─────────────────────────────────────────────┤
│        构建引擎 (scripts/build-graph.mjs)     │
│           读取 manifest → 生成图谱 JSON        │
└─────────────────────────────────────────────┘
```

| 层 | 技术 | 说明 |
|---|------|------|
| 前端 | 内嵌 SPA（HTML/CSS/JS） | 全部内嵌在 `publish-admin.mjs` 的响应中，零额外文件 |
| 后端 | Node.js `http` + `fs` | 原生模块，不依赖 Express 等框架 |
| 配置 | `publish-manifest.json` | Vault 绝对路径、全局排除目录、分类集合、文件发布清单 |
| 构建 | 对接 `build-graph.mjs` | 读取 manifest，抓取 Markdown，生成博客图谱 JSON |

## 三、核心功能

### 3.1 分类管理

- 新增、删除、编辑笔记分类
- 每个分类可配置：
  - **Label**：展示名称（如「建站流程」「Linux 笔记」）
  - **Root**：在 Vault 中的根目录路径
  - **Kind**：专属标识符

### 3.2 可视化文件浏览器

- 树状图展开所选分类下的所有 `.md` 文件
- 状态直观区分：已包含（高亮）vs 已排除（置灰）
- 单个勾选/取消，支持全选/全不选
- 高级用户可在「分类设置」中编写 Glob 表达式的 include / exclude 规则

### 3.3 笔记预览与检索

- 关键字快速过滤当前分类下的笔记
- 侧边栏预览面板：悬浮/点击即可预览正文前 6000 字符
- 确认内容后再决定是否发布，避免误操作

### 3.4 一键同步发布

- 点击「同步发布」→ 后台调用 `build-graph.mjs`
- 内置执行日志面板，实时回传编译进度和输出
- 成功/失败状态即时可见

## 四、使用方法

### 启动

```bash
cd "D:\Yhx06\Documents\仙术工坊——项目集\个人博客网站-松果阁\personal-blog"
npm run publish-admin
```

### 访问

```
http://localhost:4177
```

### 操作流程

```
1. 选择/创建分类
   → 左侧边栏选现有分类，或点「+ 新增分类」

2. 配置分类路径
   → 展开「分类设置」，填入 Obsidian Vault 中对应文件夹名

3. 挑选笔记
   → 勾选要发布的 .md 文件，取消草稿/隐私日记

4. 保存清单
   → 右上角「保存清单」→ 写入 publish-manifest.json

5. 同步发布
   → 左下角「同步发布」→ 调用 build-graph.mjs → 查看执行日志
```

## 五、配置结构 (publish-manifest.json)

```json
{
  "vaultPath": "/path/to/Obsidian Vault",
  "globalExclude": [".obsidian", ".claude", "templates", "_规范"],
  "collections": [
    {
      "id": "build-guide",
      "label": "建站流程",
      "kind": "notes",
      "root": "建站流程指南-静态网页",
      "include": ["**/*.md"],
      "exclude": ["**/draft/**"],
      "files": {
        "个人建站流程总览.md": true,
        "GitHub 项目上传流程.md": false
      }
    }
  ]
}
```

| 字段 | 含义 |
|------|------|
| `vaultPath` | Obsidian Vault 的绝对路径 |
| `globalExclude` | 全局排除的目录（不会被任何分类扫描） |
| `collections[].root` | 该分类在 Vault 中的根目录 |
| `collections[].files` | 逐文件发布开关（true=发布，false=排除） |
