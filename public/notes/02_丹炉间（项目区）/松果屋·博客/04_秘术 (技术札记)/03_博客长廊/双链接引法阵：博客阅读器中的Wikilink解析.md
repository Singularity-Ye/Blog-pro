# 双链接引法阵：博客阅读器中的 Wikilink 解析

博客页阅读弹窗不再是一个孤立的 Markdown 展示器。正文中的 `[[双链]]` 现在会接入 Obsidian 风格的知识网络，在已发布的笔记之间建立真正的可点击连接。

---

## 解析链路

```
正文中出现 [[某篇笔记]]
        ↓
查询 graphData.wikilinkIndex
        ↓
按优先级匹配：同目录 → 同 collection → 标题 → alias → basename
        ↓
匹配成功 → 渲染为 /note/:slug 站内链接
匹配失败或歧义 → 降级为金色虚线提示文本
```

降级不是"报错"——虚线提示保留了 Obsidian 原稿里的知识痕迹，告诉读者"这里本来有一篇笔记被引用了，但它还没发布或路径有歧义"。**既不给读者死链，也不抹掉关系信息。**

---

## 为什么这很重要

之前博客页的 Markdown 正文里，`[[双链]]` 就是纯文本。读者看到"详见 [[杭州旅游攻略]]"，但点不了——它只是一串字。这意味着博客页虽然能展示笔记正文，却**丢掉了 Obsidian 最核心的知识网络层**。

现在双链活了。读者在空间长廊里翻卷轴看正文时，可以顺着链接跳到另一篇笔记。阅读不再是线性的——它变成了图谱式的，和 Obsidian 里的体验一致。
