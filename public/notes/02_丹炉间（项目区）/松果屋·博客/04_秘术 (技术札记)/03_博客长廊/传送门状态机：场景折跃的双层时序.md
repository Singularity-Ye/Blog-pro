# 传送门状态机：场景折跃的双层时序

场景切换不是突兀替换，而是一次精心编排的"空间折跃"。

`changeScene` 维护两层时间：

```javascript
const changeScene = (mode) => {
  if (sceneMode === mode) return;
  setIsTeleporting(true);
  setIsTransitioning(true);
  setTimeout(() => setSceneMode(mode), 300);
  setTimeout(() => {
    setIsTransitioning(false);
    setIsTeleporting(false);
  }, 700);
};
```

300ms 后切换真实场景。700ms 后结束传送状态。中间 400ms 的窗口是 `TeleportFlash` 的表演时间——全屏覆盖，`radial-gradient` + `mix-blend-mode: screen` + `backdrop-filter: blur(12px) brightness(1.5)`，白金强光从屏幕中心向外扩散。

传送门视觉为五层结构：`PortalContainer`（定位）+ `PortalAnchor`（锚点）+ `PortalGlowRing`（波纹）+ `PortalIconWrapper`（符号）+ `PortalTooltip`（悬停叙事）。hover 时光环扩大、图标发光、提示浮现，点击后进入折跃。

> 场景切换不是路由跳转，而是一次空间折跃。读者感到的不是"页面刷新了"，而是"我被传送到了另一间房"。
