# 星沙显影传送门：博客到笔记页的状态往返协议

博客页阅读器标题区新增了"显影于星沙图谱"入口。点击后携带当前博客现场状态跳转到 `/note/:slug`，从空间长廊的一篇文章进入独立笔记页——查看正文、属性、目录、局部图谱与全站关系入口。返回时，博客页自动恢复跳走前的场景。

---

## 去程：携带现场

从 Blog 跳转到 Note 时，把当前博客状态打包进 `blogState`：

```javascript
const blogState = {
  sceneMode,
  selectedArticleSlug,
  isReaderOpen,
  isModalOpen,
  activeScrollTitle,
  modalNotes,
  articleMeta
};
navigate(`/note/${slug}`, { state: { fromBlog: true, blogState } });
```

用户在笔记页看到的不是一篇孤立的文章——他知道自己"从哪片叶子点进来的"。

---

## 回程：恢复现场

Note.jsx 返回按钮触发时，不直接 `navigate('/blog')`，而是带复原数据：

```javascript
navigate('/blog', { replace: true, state: { restoreBlogState: blogState } });
```

Blog.jsx 挂载后检测 `restoreBlogState`，逐一恢复——场景、弹窗状态、阅读器打开、目录列表，然后 `replace` 清理 state 避免浏览器前进后退时重复触发。

---

## 精进方向

**`replace: true` 是必须的。** 如果不用 replace，从 Note 返回 Blog 后按浏览器后退，会回到刚才那篇 Note——形成 Blog → Note → Blog → Note 的死循环。

**布尔值恢复用字段存在判断。** `if (s.isReaderOpen) setIsReaderOpen(s.isReaderOpen)` 只能恢复 true，不能恢复显式 false。应该用 `if ('isReaderOpen' in s)`。

**`blogState` 可以瘦身。** 目前传了完整 `modalNotes`，目录多时 history state 会变重。后续可只传 `modalNoteSlugs`，回到 Blog 后用 `notesData` 重建列表。

**多跳笔记的状态继承。** 从 Note 正文里再点一个双链跳到另一篇 Note，当前的 `blogState` 没有被透传。如果想让"多跳笔记后仍能一键回到博客现场"，需要在 Note 的站内链接上继续携带 `location.state`。

---

> 这套协议解决了最痛的问题——跳出去就丢了现场。补上 replace 和 state 瘦身，就是一套优雅的空间长廊往返协议。
