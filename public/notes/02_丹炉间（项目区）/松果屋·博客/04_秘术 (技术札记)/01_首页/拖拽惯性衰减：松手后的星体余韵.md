# 拖拽惯性衰减：松手后的星体余韵

`BlogPlanet` 是星寰仪的交互核心。星体不是被鼠标机械拖动，而是像真实悬浮物一样——被拨了一下，带着余势缓缓回转。

---

## 拖拽时：实时跟随 + 记录速度

`pointermove` 根据鼠标位移实时改变星体的 `rotation.x` 与 `rotation.y`。同时记录速度：

```javascript
velocityRef.current = {
  x: dy * 0.004,
  y: dx * 0.006,
};
```

`y` 分量比 `x` 分量快 1.5 倍——水平拖拽比垂直拖拽更灵敏。这是经验调出来的手感，不是数学公式。

---

## 松手后：逐帧衰减

```javascript
group.rotation.y += velocityRef.current.y + 0.00045;
group.rotation.x = clamp(
  group.rotation.x + velocityRef.current.x, -0.65, 0.65
);
velocityRef.current.x *= 0.92;
velocityRef.current.y *= 0.92;
```

每帧速度乘以 0.92——衰减很快但不突兀。约 30 帧后速度降至原来的 8%，肉眼已经感觉不到惯性了。

`0.00045` 的基础自转保证即使用户完全不操作，星体也像活物一样慢慢运行。`rotation.x` 被 clamp 在 ±0.65 弧度内，防止用户把星球"掀翻"。

---

## 缩放 lerp 缓动

缩放也不是突变。维护两组变量——目标与当前：

```javascript
currentScaleRef.current = THREE.MathUtils.lerp(
  currentScaleRef.current,
  targetScaleRef.current,
  0.12
);
group.scale.setScalar(currentScaleRef.current);
```

每帧向目标逼近 12%。滚轮改变目标值，星体逐帧追上。急刹车不存在，只有缓缓收放。

---

## 滚轮只在拖拽时生效

更细的一个交互边界：滚轮缩放仅在拖拽状态下触发。页面普通滚动不会被 3D 画布劫持。只有当用户真正"抓住星体"时，滚轮才会被解释为调节星寰仪视距。

> 星体不是被鼠标拖动，是被拨了一下然后自己转。像桌上的一颗低模地球仪——你知道它不是活的，但手感上它是。
