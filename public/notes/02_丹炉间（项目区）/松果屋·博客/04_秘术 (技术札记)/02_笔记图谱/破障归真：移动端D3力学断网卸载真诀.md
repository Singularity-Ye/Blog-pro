# 破障归真：移动端 D3 力学断网卸载真诀

此术看似只是一行 JSX，实则是移动端性能调优里极容易被忽略的一道真诀。

世俗响应式设计常以 CSS 媒体查询遮蔽组件：

```css
@media (max-width: 900px) {
  .sidebar {
    display: none;
  }
}
```

**这只是遮眼法。** 组件虽然不显示，React 生命周期却可能仍在，Canvas 仍可能存在，D3 力学模拟仍可能在后台推演。事件监听、动画循环、内存占用也未必消失。

用户看不见，不代表它不存在。

---

## 图谱页尤其不能犯这个错

MiniGraph 背后不是普通文本块，而是一套 Canvas + D3 力学图谱。若只用 CSS 隐藏，移动端读者打开长文时，右侧那张"不可见星盘"仍可能在后台燃烧 CPU：手机发热、滚动掉帧、电量流失——**皆由这类隐形卡顿业障而起。**

---

## 解法：条件挂载

在 React 状态层引入视口感知：

```javascript
const [isMobile, setIsMobile] = useState(
  () => typeof window !== 'undefined' ? window.innerWidth <= 900 : false
);
```

再在 JSX 层进行条件挂载：

```jsx
{!isMobile && localGraphData && (
  <MiniGraph
    currentSlug={decodedSlug}
    graphData={localGraphData}
    label="局部关系图"
    theme="publishLocal"
    expandHref={`/graph?local=${encodeURIComponent(decodedSlug)}`}
  />
)}
```

这不是让 MiniGraph"藏起来"——**是让它在移动端根本不出生。**

---

## 效果对比

| 方案 | 视觉 | React 生命周期 | Canvas/D3 | CPU |
|------|:--:|:--:|:--:|:--:|
| `display: none` | 看不见 | 仍可能挂载 | 仍可能运行 | 仍可能消耗 |
| JSX 条件卸载 | 看不见 | 直接不挂载 | 不存在 | 基本归零 |

组件不挂载 → Canvas 不创建 → D3 不开阵 → CPU 归于清净。

---

## 用户体验上的考量

这也不是"移动端砍功能"。移动端读者打开笔记，第一需求是**顺滑阅读**，而不是在窄屏里硬塞一张难以操作的力导向图。

图谱入口仍在——需要探索时，再进入全屏大阵。平时阅读，正文轻装上阵。

---

## 可推广到所有高负载组件

此术不限于 MiniGraph。以下组件都可以用同样的逻辑保护移动端：

- Canvas 图表
- D3 force simulation
- Three.js / WebGL 场景
- `requestAnimationFrame` 动画循环
- 定时器轮询
- 视频、音频、地图
- 复杂拖拽与大型虚拟列表
