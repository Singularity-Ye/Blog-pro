## 一、目标

这些交互素材的作用不是当背景装饰，  
而是作为博客页里真正可点击、可展开、可承载内容的**前景物件**。

你现在已经有：

- 松果屋内部总图
- 左书架 / 中央桌面 / 旅行区 / 右书架 / 卷轴区近景图

所以下一步的交互素材，就是从这些区域里“提炼”出核心入口物件。

---

# 二、第一批必须做的 6 个核心交互物件

---

## 1. 旅行地图册

### 对应内容

- 3D 旅游路线地图
- 杭州巡游计划
- 旅途节点手札

### 建议摆放区域

- 下方平台 / 地图区

### 建议状态

- `closed`：合上的地图册
- `hover`：轻微抬起 / 发光
- `open`：摊开的地图册

### 交互形式

- 点击后书本摊开
- 左右页展示项目介绍、路线截图、入口按钮

### 文件建议名

```
travel-mapbook-closed.pngtravel-mapbook-hover.pngtravel-mapbook-open.png
```

### 优先级

**最高**

# 六、每个交互物件至少要做几张图？

为了不让工作量爆炸，我建议第一版按这个标准：

## 基础标准

每个核心物件先做 **2~3 个状态**：

### 最低配置

- closed / default
- hover
- open

### 举例

地图册：

- 合上
- hover 发光
- 摊开

卷轴：

- 卷起
- hover
- 展开

借阅证：

- 正面
- 背面

这样已经足够前端做出完整交互。
