# 设计经验总结

> 从多次实验中沉淀下来的设计判断。

## 1. 不要盲目复刻复杂游戏设计

之前尝试复刻 Broforce 地图风格，发现：

- 靠提示词很难精准复刻游戏美术
- 载具主体（直升机、鸟）增加大量建模和交互负担
- 模型效果不稳定时，反而破坏整体观感

**结论**：不强行复刻游戏，而是吸收其"微缩地图"和"可探索"的优点。

## 2. 先把核心交互做稳

首页星球收敛为拖拽旋转后，反而更稳。

核心交互（拖拽旋转 → hover 标签 → 点击跳转）比复杂载具导航更适合作为个人博客首页。

**原则**：先把最简单的交互做好，再叠加复杂功能。

## 3. image2 贴图 + 3D 小模型很有前景

海洋纹理的成功证明了这条路线：

```
AI 生成纹理贴图 + Three.js 动态材质 + 低模几何体
```

这是一条非常有效的视觉升级路线，后续应继续用于：

- 森林 / 雪山 / 沙漠 / 城市（首页生态区）
- 松果 / 书本 / 木质 / 纸张（松果书屋）
- 书籍封面 / 地图册纹理

## 4. 个人网站要有"可探索感"

网站不只是信息展示，还可以是：

- 一个可以拖拽把玩的小地球仪
- 一个可以逛的松果书屋
- 一个可以点亮路线的地图

**原则**：让用户不只是"阅读"，而是"发现"。

## 5. 规划算法不要过度精炼

路线规划器 v0.3 已经完成了从"生成"到"筛选"的关键跃迁。继续细化只会"抠牙缝"。

**原则**：当算法产出够用时，把精力转向展示与交互。
