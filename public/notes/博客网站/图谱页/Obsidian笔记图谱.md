# 图谱页 - Obsidian 笔记图谱

> 记录和展示 Obsidian 中的知识结构，用节点与连线呈现个人知识网络。

## 设计目标

图谱页负责把 Obsidian 里的笔记结构化，用图谱展示节点关系。支持知识整理、旅游指南、小说编写。

## 可承载的内容

### 知识整理

Obsidian 作为复习目录（不是完整学习路径）：

- 课程知识点
- 学习笔记
- 算法与理论
- 论文阅读
- 考试复习内容

### 旅游指南

重要方向之一。Obsidian 中可整理：

- 景点节点 / 美食节点 / 路线节点
- 活动节点 / 区域节点
- 行程规划
- 路线质量报告 / 推荐路线

未来可与 [[博客网站/项目/3D旅游路线地图|3D 地图项目]] 联动。

### 小说编写

世界观管理：

- 人物 / 地点 / 组织 / 事件
- 世界观设定 / 伏笔 / 时间线
- 关系图谱 / 魔幻地图

未来可与 [[博客网站/项目/现实地图魔改小说世界观|小说地图项目]] 联动。

## 视觉方向

可以接近：

- 节点 + 连线 + 星图
- Obsidian Graph View 风格
- 局部展开 + 点击节点查看详情

不同于首页的低模自然风，图谱页更偏向：

- 信息可视化
- 星图 / 网络图
- 科技感与知识感

## 技术架构

当前手写 ForceGraph 存在 O(N²) 性能瓶颈。已规划架构升级：

→ [[博客网站/图谱页/图谱页架构升级方案]]

核心思路：引入 d3-force (Barnes-Hut O(NlogN) 四叉树) 作为物理内核，react-force-graph-2d 作为 React 渲染框架。保留羊皮纸 UI，赋予节点语义色彩。
