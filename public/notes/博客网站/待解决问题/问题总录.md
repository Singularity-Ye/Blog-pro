 代码审查报告：个人博客网站

  项目概览

  基于 React 18 + CRA 构建的个人博客，集成 Three.js 3D 场景、Rapier 物理引擎、Obsidian
  笔记系统和知识图谱可视化。整体创意和视觉效果出色，但存在一些需要关注的技术问题。

  ---
  高优先级 — Bug / 内存泄漏

  1. ForceGraph 事件监听器永远不会被移除（内存泄漏）
  GraphView.jsx 中 _bindEvents() 用 .bind(this) 创建了新的函数引用，但 destroy() 用原始未绑定方法调用
  removeEventListener。引用不同，监听器永远不会被清除。每次组件重新挂载都会累积泄漏。MiniGraph.jsx 同样受影响。

  2. Canvas DPR 缩放累积
  ForceGraph.resize() 每次窗口 resize 都调用 ctx.scale(dpr, dpr)，canvas 变换会累积，导致图形渲染越来越失真。应改用
  ctx.setTransform(dpr, 0, 0, dpr, 0, 0)。

  3. Lanyard 物理仿真在每个页面都运行
  Lanyard（Rapier 物理 + Three.js 场景）渲染在 Navigation.jsx 中，意味着每个页面都在跑重物理仿真。/graph
  页面同时运行两个 canvas，性能负担很大。建议仅在首页加载或用 IntersectionObserver 控制。

  ---
  中优先级 — 安全 / UX

  4. Markdown 未做 XSS 过滤
  react-markdown v10 默认不清理 HTML。笔记中若含 <script> 标签可执行任意 JS。当前笔记是静态文件风险低，但应加上
  rehype-sanitize。

  5. graph.json 泄露完整 Vault 结构
  274KB 的 graph.json 包含 .claude/skills/ 等内部路径。若公开部署会暴露个人信息。构建脚本应过滤敏感路径。

  6. Obsidian Vault 路径硬编码
  scripts/build-graph.mjs 写死了 C:\Users\Yhx06\OneDrive\文档\Obsidian Vault，换机器或 CI 必然失败。应改为环境变量。

  7. 缺少 favicon.ico 和 logo192.png
  index.html 引用了这两个文件但 public/ 目录下不存在，控制台 404。

  8. 无 React Error Boundary
  Three.js canvas、图谱引擎、Markdown 渲染器任一抛错都会导致白屏崩溃。建议至少在路由层级加一个 ErrorBoundary。

  9. 缺少 404 路由
  App.js 没有 catch-all 路由，访问不存在的路径只显示空白页（仅有导航和页脚）。

  ---
  低优先级 — 代码质量

  10. 未使用的依赖
  - matter-js 和 @types/matter-js 从未被 import，物理引擎用的是 Rapier
  - TextPressure.jsx 从未被任何组件引用，是死代码

  11. 文件扩展名不一致
  App.js、index.js、GlobalStyle.js 用 .js，其他组件用 .jsx。建议统一。

  12. 样式方案混用
  同时使用 styled-components、CSS 文件、内联样式三种方式，不利于维护。

  13. 四个页面是空壳
  /blog、/about、/projects、/contact 只有一行占位文字。

  14. 无无障碍支持
  缺少 ARIA 标签、skip-to-content 链接、图谱焦点管理等。

  15. vite-env.d.ts / global.d.ts 无用
  项目用的是 CRA 不是 Vite，这两个声明文件是残留物。

  ---
  建议优先修复顺序

  16. ForceGraph 事件监听器泄漏 — 存在真实的内存泄漏，组件频繁挂载/卸载时会越来越卡
  17. Canvas DPR 缩放 — 修复只需一行代码
  18. Lanyard 按需加载 — 改为仅首页加载或用懒加载，性能提升最明显
  19. 加 Error Boundary — 防止白屏崩溃
  20. 清理未用依赖和死代码 — npm uninstall matter-js @types/matter-js，删除 TextPressure.jsx
