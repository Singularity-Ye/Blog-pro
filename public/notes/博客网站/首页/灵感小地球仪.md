# 首页 - 灵感小地球仪

> 首页不再是普通 Hero 区域，而是一个可以交互的 3D 小地球仪。

## 设计目标

首页作为整个网站的"第一印象"和"导航入口"，用一个可拖拽旋转的低模小地球仪把所有内容分区可视化。

## 当前方案

- 左侧：文字介绍 + 分类卡片
- 右侧：Three.js 制作的 3D 小地球仪

## 首页文案方向

> **灵感小地球仪**
> 把技术、笔记、随笔和项目折叠成一颗明亮的小星球。
> 拖拽旋转它，找到散落在海洋上的雨林、雪山、荒漠与城市。

## 交互逻辑

| 交互 | 行为 |
|------|------|
| 鼠标拖拽 | 旋转地球仪 |
| 松手 | 带一点惯性继续转 |
| hover 生态区 | 显示标签 |
| 点击生态区 | 进入对应页面 |

### 之前尝试过但放弃的方案

- 多面体水晶
- 星球自动旋转
- 鼠标移到边缘控制旋转
- 直升机 / 鸟作为地图探索主体
- Broforce 风格地图复刻

**收敛原因**：这些方案过于复杂或模型效果难以控制。直接拖拽旋转反而更稳。

## 视觉路线

不追求真实地球，也不追求复刻游戏地图，走：

**低多边形小地球仪 + AI 纹理贴图 + Three.js 小模型**

### 确定的视觉方向

- 明亮、清新
- 蓝绿色自然系
- **不是**星空背景、**不是**深色科幻球体
- 更像一个可拖拽把玩的低模小世界

### 纹理贴图方案

详见 → [[博客网站/首页/image2纹理贴图方案]]

核心发现：用 image2 生成纹理贴图，再结合 Three.js shader / 材质系统，效果远好于纯 vertexColors 或程序色块。

## 核心交互优先级

1. ✅ 拖拽旋转地球仪（已实现）
2. ✅ 海洋纹理贴图（已验证效果好）
3. 🔲 hover 标签优化
4. 🔲 分类跳转稳定化
5. 🔲 其他生态区贴图升级
