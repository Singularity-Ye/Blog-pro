# 博客页空间长廊：七幕游园式交互设计手札

博客页不是文章列表，而是一座可以步入、转场、寻物、开卷的空间长廊。

传统博客页把文章按时间线排成卡片。这里反其道而行——文章、分类、旧方案、旅行手札、技术档案全部化作场景中的可触摸物件。读者不是在滚动列表，而是在松果屋、叶间书林、旅图案台、旧札柜、书脊回廊与建站工坊之间游走，像探秘一样找到不同的笔记入口。

核心心诀：**以空间承载分类，以物件触发数据，以卷轴展开内容。**

---

## 一、七幕场景总览

整座长廊的骨架是 `BLOG_SCENES` 配置，七个场景各有其空间身份：

| 场景 | 意象 | 承载内容 |
|------|------|------|
| **overview** · 松果屋总览 | 树屋大门与林间小径 | 场景入口，两个传送门通往屋内和林间 |
| **indoor** · 碎叶墙 | 暖光木墙，悬挂符纸 | 轻量短篇、随笔感悟（金叶+绿叶） |
| **outdoor** · 叶间书林 | 巨树枝桠，古卷悬垂 | 长文专题、体系化笔记（竹简+玉卷） |
| **travel** · 旅图案台 | 旅行书案，地图摊开 | 旅行笔记、攻略路线（地图册+卷轴） |
| **junction** · 书脊回廊 | 岔路回廊，拱门两侧 | 连接旧札柜与建站工坊的过渡空间 |
| **archive** · 旧札柜 | 阁楼旧柜，抽屉微开 | 旧方案归档、问题追踪、索引检索 |
| **workshop** · 建站工坊 | 魔法工作室，法器陈列 | 技术复盘、架构蓝图、未来构想 |

`sceneMode` 保存当前场景，渲染时根据 `currentScene` 决定背景画卷、传送门位置、可交互物件与弹窗主题。**新增一个场景不是新写一页，而是在长廊地图上新增一间房。**

---

## 二、移步换景：传送门与以太闪光

场景之间的跳转不是路由切换，而是一次空间折跃。

传送门由五层视觉组成——锚点、外层波纹、图标符号、悬停提示、点击强光。`changeScene` 有两个时间层：300ms 后切换真实场景，700ms 后结束传送状态。`TeleportFlash` 覆盖全屏，用 `radial-gradient` + `mix-blend-mode: screen` + `backdrop-filter: blur(12px)` 制造白金强光穿过视口的效果。

场景切换不是突兀替换，而是先亮起传送强光，再在强光中换景。

---

## 三、碎叶墙：摘叶预览的交互仪式

indoor 场景的金叶与绿叶是博客页最有诗意的一组交互。每片叶子绑定了 `collection` 或 `filter`。点击后不是直接打开正文，而是先摘叶、抖叶、浮出摘叶纸条。`getNoteSnippet` 会跳过 frontmatter 和噪音行，根据中文密度与段落长度评分，挑出更像正文的片段。用户读完纸条再决定是否展开完整阅读。

交互链路：点击叶片 → 叶片抖动 → filter/collection 找到笔记 → fetch Markdown → parseFrontmatter + getNoteSnippet → 浮出摘叶纸条 → 点击"展开读卷"进入正文。

---

## 四、叶间书林：分类接引

outdoor 场景的竹简和玉卷绑定多个 collections。建站流程、Linux 与嵌入式、织墨灵卷、编译原理，从不同卷轴进入。点击后筛选 `notesData`，结果放入统一卷轴弹窗。场景物件不保存文章内容——只保存如何找到文章。数据来自 `notes-index.json`，物件只是入口、筛选器和叙事壳。

---

## 五、旅图案台与旧札柜工坊

travel 场景的地图册和卷轴依赖**透明抠图 alpha 命中检测**——不是矩形热区，而是预计算图片 alpha 通道，只在非透明区域触发 hover。archive 的旧方案抽屉绑定高级过滤函数，workshop 的星盘和蓝图指向特定笔记 slug。旧柜、蓝图、魔药瓶不是静态图标，而是 Obsidian 数据筛选器的可视化外壳。

---

## 六、全局粒子气候与卷轴弹窗

全屏 `CanvasBackground` 用原生 Canvas + rAF 绘制浮光粒子，颜色随场景变化——屋内暖金、室外深青、总览左侧金右侧绿。粒子读取 `rawMouseRef`（用 ref 而非 state，避免 React 高频渲染）。卷轴弹窗根据场景切换背景：林间卷轴、旅案海图、旧柜抽屉索引、通用羊皮纸。

---

## 七、整体设计价值

三层融合：**视觉叙事**——分类变成空间，读者凭视觉记忆找到内容；**交互物件化**——叶子、竹简、地图册、抽屉、蓝图、药水瓶，每种对应不同信息密度；**数据动态化**——所有物件最终连接 `notes-index.json` 与 Markdown，笔记更新则物件背后的内容随之变化。

画卷负责召唤，索引负责寻路，卷轴负责展开。
