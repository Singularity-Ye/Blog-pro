# 联系页 — 三幕虚拟画卷交互设计

> 基于 Framer Motion 的虚拟滚动吸附系统，用电影分镜的方式组织联系页面的浏览体验。

---

## 一、设计理念

联系页不只是一张表单。它被设计成一幅纵向展开的三幕画卷：用户轻拨滚轮，画面如电影镜头般推拉对焦，从问候过渡到名片，再滑入表单。每一步都是完整的满屏沉浸。

核心目标：**比肩 Apple 官网 / 顶级创意工作室页面的交互质感**。

---

## 二、技术架构总览

```
┌──────────────────────────────────────────┐
│           ContactWrapper                  │
│  overflow: hidden   (锁定原生滚动)         │
│  ┌────────────────────────────────────┐  │
│  │       Sections 容器                 │  │
│  │  pageTranslationY = -simulatedScrollY│  │
│  │  ┌──────┐ ┌──────┐ ┌──────┐        │  │
│  │  │第 0 幕│ │第 1 幕│ │第 2 幕│        │  │
│  │  │问候  │ │名片  │ │表单  │        │  │
│  │  └──────┘ └──────┘ └──────┘        │  │
│  └────────────────────────────────────┘  │
│                          ┌─────────┐     │
│                          │ DotNav  │     │
│                          │ ● ○ ○  │     │
│                          └─────────┘     │
└──────────────────────────────────────────┘
```

| 层次 | 技术 | 解决的问题 |
|------|------|-----------|
| 手势捕获 | `onWheel` + `onTouchStart/End` | 识别翻页意图，防止误触发 |
| 虚拟滚动轴 | `useMotionValue(0)` + `animate()` | 脱离浏览器原生滚动物理差异 |
| 视觉投影 | `useTransform(val => -val)` | 驱动整幅画卷纵向平移 |
| 镜头动效 | `scale` / `blur` 动态映射 | 翻页时画面虚焦退出，落位时对焦清晰 |

## 三、核心系统详解

### 1. 虚拟滚动吸附 (Virtual Snapping)

**问题**：原生 CSS `scroll-snap` 在不同平台、不同浏览器的滚动物理特性不一致，吸附反弹生硬。

**方案**：完全接管滚动行为。

| 步骤 | 实现 |
|------|------|
| 锁定视口 | `ContactWrapper: height: 100vh; overflow: hidden` |
| 隐藏滚动条 | `useEffect` 中设置 `html/body { overflow: hidden }` |
| 虚拟位置 | `simulatedScrollY = useMotionValue(0)` |
| 翻页触发 | `onWheel` 捕获，累计 delta 超过阈值后判定翻页方向 |
| 平滑过渡 | `animate(simulatedScrollY, target, { duration: 1.1, ease: [0.76, 0, 0.24, 1] })` |
| 容器平移 | `pageTranslationY = useTransform(simulatedScrollY, val => -val)` |

**缓动曲线**：`easeInOutQuart` — 贝塞尔 `[0.76, 0, 0.24, 1]`，1.1 秒过渡：
- 前段极缓启动（像推一扇沉重的门）
- 中段快速平滑推进（滑行）
- 尾端渐进式柔和回弹（落定，不晃动）

### 2. 镜头焦点呼吸 (Focus Breathe)

翻页时不是粗暴地切走画面，而是模拟光学镜头的「拉焦 → 聚焦」过程。

```
触发翻页 →
  StageContent scale: 1.0 → 0.93
  StageContent blur:  0px → 8px
  (画面推开、虚化 —— 像镜头拉开)

吸附落位 →
  StageContent scale: 0.93 → 1.0
  StageContent blur:  8px → 0px
  (画面弹回、清晰 —— 像镜头对焦)
```

**效果**：视觉上的过渡缓冲完全抵消了物理停止的生硬感，呈现出电影运镜般的质感。

### 3. 背景暗场交叉淡化 (Crossfade Backdrop)

三段背景图的 `opacity` 绑定在 `simulatedScrollY` 上分段映射：

```
simulatedScrollY
  0vh ─────── 第 0 幕背景 opacity: 1
       ├── 第 1 幕背景 opacity: 0
  100vh ───── 第 0 幕背景 opacity: 0
       ├── 第 1 幕背景 opacity: 1
       ├── 第 2 幕背景 opacity: 0
  200vh ───── 第 1 幕背景 opacity: 0
       ├── 第 2 幕背景 opacity: 1
```

幕间空白区间：前幕背景淡出 → 露底纯黑 → 后幕背景浮现。黑场过渡取代硬切边界。

### 4. 悬浮圆点导航 (DotNav)

| 特性 | 实现 |
|------|------|
| 位置 | 页面右侧固定悬浮 |
| 状态 | 当前幕圆点高亮 + 缩放 |
| 点击 | `animate(simulatedScrollY, section * vh)` 跨屏跳转 |
| 视觉 | 精致小型圆点，不抢主画面 |

### 5. 3D 挂绳卡片层级优化

第二幕（名片幕）包含一个 Canvas 3D 交互区域。需要在不影响其他幕的前提下正确处理 z-index。

| 状态 | z-index | 行为 |
|------|:--:|------|
| `activeSection === 1` | 25 | 拦截鼠标事件，3D 拖拽畅通 |
| 其他幕 | 1 | 完全透明于 HTML 事件，不阻塞表单点击 |

> 废弃了原有的 `isLanyardInteractive` 状态和 `window.scrollY` 轮询。改用一个 `activeSection` 条件直接控制。

## 四、三幕内容布局

| 幕 | 内容 | 交互 |
|:--:|------|------|
| 第 0 幕 | 问候语 / 引导文案 | 纯展示，氛围入场 |
| 第 1 幕 | 3D 挂绳卡片（青蛙 + 名片 + 拖拽） | Canvas 交互，松果名片体验 |
| 第 2 幕 | 表单与联系方式 | 输入、提交、Hover 动效 |
