
## 结论：挂绳卡片适合放博客页，但要改身份

它不应该再叫普通的 lanyard card，而应该变成：

**书屋访客牌 / 灵感通行证 / 青蛙馆长借阅证**

也就是说，它从首页的装饰物，变成博客页里的一个**功能性入口**。

比如：

```
松果书屋借阅证Pinecone Library Pass
```

卡片上可以放：

- 青蛙头像
- 你的昵称 / 站点名
- 当前状态：OBE ONLINE
- 一个小二维码
- 一句短文案

这样它就不只是“挂着玩”，而是有语义了。

## 二维码指向什么？不建议直接放微信 / QQ

我不太建议把二维码直接指向微信或 QQ。

原因有三个：

第一，隐私太硬。访客一扫就进入私人联系方式，像突然把书屋后门钥匙塞给路人。

第二，维护麻烦。微信二维码可能会变，QQ 也不一定适合公开挂在网页主视觉里。

第三，它会削弱“联系”页面。你说得对，如果卡片二维码就是联系方式，那“联系”页就变成摆设了。

更好的方案是：

## 二维码指向“联系页”或“个人名片页”

也就是二维码不要直接指向微信，而是指向：

```
/contact
```

或者：

```
/card/about-card/guest-pass
```

这个页面里再放：

- 微信二维码
- QQ
- 邮箱
- GitHub
- Bilibili / 小红书 / 其他平台
- 留言方式
- 合作说明

这样挂绳卡片是**入口**，联系页是**内容容器**。

它们不是互相替代，而是分工：

```
挂绳卡片：我是谁？扫这里进入我的名片。联系页面：你可以通过哪些方式找到我？
```

这就像书屋门口挂了一张借阅证，扫一下不是直接闯进馆长卧室，而是打开访客登记册。🐸📇

---

## 最推荐的设计：二维码指向“个人名片页”

我建议新建一个页面：

```
/guest-card
```

中文名可以叫：

```
青蛙馆长名片松果书屋访客牌灵感书屋通行证
```

这个页面可以比普通联系页更有趣，像一张电子名片：

```
叶浩祥 / 呆猫松果灵感书屋管理员研究方向：- Obsidian 图谱- Three.js 视觉实验- 旅行地图- 小说世界观地图- AI 辅助创作联系方式：- 邮箱- GitHub- QQ- 微信
```

然后导航栏里的“联系”仍然保留，指向正式的 `/contact` 页面。  
挂绳卡片二维码可以指向 `/guest-card`，而 `/guest-card` 里面再引导到 `/contact`。

这样层级就很清楚：

```
挂绳卡片二维码        ↓个人名片页 / guest-card        ↓正式联系页 / contact
```

## 放在博客页哪里最合适？

不要像首页那样从顶部导航栏吊下来。博客页可以这样放：

### 方案 A：挂在松果书屋入口处

像进入书屋前的访客牌：

```
[松果屋]   |   |-- 门口挂着一张小卡片
```

用户 hover 卡片时，它轻轻晃动；点击后放大，显示二维码。

适合当作“书屋入口身份牌”。

---

### 方案 B：挂在青蛙管理员旁边

青蛙躺着读书，旁边挂一张小小借阅证：

```
青蛙馆长旁边：Pinecone Library Pass
```

这样卡片和青蛙绑定，不会孤零零飘在页面上。

---

### 方案 C：作为博客页右下角的浮动名片

不挡内容，用户需要时才展开：

```
右下角小挂件点击展开成完整卡片再次点击收起
```

这个最实用，也最不容易破坏布局。

---

## 卡片上的青蛙图要不要换成二维码？

我建议不要完全替换。  
最好的设计是：**正面青蛙，背面二维码。**

交互可以这样：

```
默认：青蛙卡面hover：卡片微微翻转点击：翻到背面，显示二维码
```

这样青蛙依然是视觉角色，二维码也有功能。  
如果一开始就把卡片正面做成二维码，会很工具化，童话感会被一块黑白方阵吃掉。

卡片可以设计成：

```
正面：青蛙头像PINECONE LIBRARYOBE ONLINE叶浩祥 / Daimao背面：二维码Scan to visit my guest card
```
